2.4.2 Tytuł strony

Poziom A

Oficjalna dokumentacja W3C

Opis kryterium

Strony internetowe mają tytuły opisujące temat lub cel strony.

Intencja

Celem tego kryterium sukcesu jest pomoc użytkownikom w znajdowaniu treści oraz orientowaniu się w niej poprzez zapewnienie, że każda strona ma opisowy tytuł. Tytuły identyfikują bieżące położenie bez konieczności czytania lub interpretowania zawartości strony.

Tytuły stron są szczególnie ważne dla:

  • Użytkowników czytników ekranu - tytuł jest pierwszą informacją odczytywaną na stronie
  • Osób z ograniczeniami poznawczymi - pomagają w orientacji i zrozumieniu kontekstu
  • Wszystkich użytkowników - są widoczne w kartach przeglądarki, zakładkach i historii
  • Wyszukiwarek - tytuły są kluczowe dla SEO i prezentowania wyników

Gdy użytkownicy nawigują między różnymi stronami, tytuł pomaga im zrozumieć gdzie się znajdują i co mogą znaleźć na danej stronie.

Przykłady

Zły przykład - nieopisowe i problematyczne tytuły

Przykłady tytułów, które nie spełniają kryterium 2.4.2:

❌ Problematyczne tytuły:
1. Brak tytułu:
<title></title> lub brak elementu title
2. Ogólny tytuł:
<title>Strona główna</title>
3. Tylko nazwa firmy:
<title>Acme Corp</title>
4. Nieaktualne tytuły:
<title>Strona w budowie</title>
5. Techniczne nazwy:
<title>page_01.html</title>
Dlaczego to problematyczne?
  • Użytkownik nie wie na jakiej stronie się znajduje
  • Trudno rozróżnić karty w przeglądarce
  • Problemy z zakładkami i historią
  • Słaba dostępność dla czytników ekranu
  • Negatywny wpływ na SEO
<!-- ZŁE PRZYKŁADY - nieopisowe tytuły -->

<!-- 1. Brak tytułu -->
<head>
    <meta charset="UTF-8">
    <!-- Brak elementu title -->
</head>

<!-- 2. Pusty tytuł -->
<head>
    <title></title>
</head>

<!-- 3. Nieopisowy tytuł -->
<head>
    <title>Strona</title>
</head>

<!-- 4. Tylko nazwa firmy -->
<head>
    <title>Acme Corporation</title>
</head>

<!-- 5. Techniczne nazwy -->
<head>
    <title>index.html</title>
</head>

<!-- 6. Nieaktualne informacje -->
<head>
    <title>Strona w budowie | Proszę wrócić później</title>
</head>

Problem: Te tytuły nie dostarczają użytkownikowi wystarczających informacji o treści lub celu strony. Użytkownicy, szczególnie korzystający z czytników ekranu, nie mogą skutecznie identyfikować i nawigować między stronami.

Dobry przykład - opisowe tytuły stron

Przykłady dobrze napisanych tytułów dla różnych typów stron:

✅ Dobre przykłady tytułów:
Strona główna:
"Najlepsze produkty ekologiczne - EcoShop | Naturalny sklep online"

Zawiera: cel strony + nazwę firmy + opis działalności

Strona produktu:
"Laptop Dell XPS 13 - specyfikacja, cena, opinie | TechStore"

Zawiera: nazwę produktu + kluczowe informacje + sklep

Artykuł/blog:
"Jak dbać o rośliny domowe zimą - 10 praktycznych porad | Garden Blog"

Zawiera: temat artykułu + wartość + nazwę bloga

Formularz kontaktowy:
"Skontaktuj się z nami - formularz kontaktowy | Firma ABC"

Zawiera: cel strony + typ treści + firmę

Strona błędu:
"Błąd 404 - Strona nie została znaleziona | Pomocne linki | NaszaSerwis"

Zawiera: typ błędu + informację pomocniczą + serwis

Zalety dobrych tytułów:
  • Natychmiastowa identyfikacja - użytkownik od razu wie gdzie jest
  • Łatwa nawigacja - rozróżnianie kart i zakładek
  • Lepsza dostępność - czytniki ekranu ogłaszają tytuł
  • SEO - lepsze pozycjonowanie w wyszukiwarkach
  • UX - lepsza orientacja w serwisie

✨ Kluczowe cechy dynamicznych tytułów:
  • Automatyczna aktualizacja - tytuł zmienia się przy zmianie widoku
  • Informacje kontekstowe - np. liczba nieprzeczytanych wiadomości
  • Status operacji - np. "Zapisywanie..." lub "Błąd połączenia"
  • Breadcrumbs w tytule - odzwierciedlenie ścieżki nawigacji
  • Accessibility - czytniki ekranu są powiadamiane o zmianach 
// DOBRY PRZYKŁAD - zarządzanie dynamicznymi tytułami

class PageTitleManager {
    constructor(appName = '') {
        this.appName = appName;
        this.defaultTitle = appName;
        this.titleStack = [];
    }

    // Ustawia tytuł strony z automatycznym formatowaniem
    setTitle(pageTitle, context = '', showNotifications = true) {
        let fullTitle = '';
        
        if (pageTitle) {
            fullTitle = pageTitle;
            
            // Dodaj kontekst jeśli istnieje
            if (context) {
                fullTitle += ` - ${context}`;
            }
            
            // Dodaj nazwę aplikacji
            if (this.appName) {
                fullTitle += ` | ${this.appName}`;
            }
        } else {
            fullTitle = this.defaultTitle;
        }
        
        // Aktualizuj tytuł
        document.title = fullTitle;
        
        // Powiadom czytniki ekranu o zmianie (opcjonalne)
        if (showNotifications) {
            this.announcePageChange(pageTitle);
        }
        
        console.log(`Tytuł zmieniony na: ${fullTitle}`);
    }

    // Tytuł z dodatkowymi informacjami (np. liczniki)
    setTitleWithBadge(pageTitle, badgeCount, context = '') {
        let titleWithBadge = pageTitle;
        
        if (badgeCount && badgeCount > 0) {
            titleWithBadge = `(${badgeCount}) ${pageTitle}`;
        }
        
        this.setTitle(titleWithBadge, context);
    }

    // Tymczasowy tytuł ze statusem (np. ładowanie, błąd)
    setTemporaryTitle(status, originalTitle) {
        const tempTitle = `${status} - ${originalTitle}`;
        this.setTitle(tempTitle, '', false);
    }

    // Przywróć poprzedni tytuł
    restoreTitle() {
        if (this.titleStack.length > 0) {
            const previousTitle = this.titleStack.pop();
            document.title = previousTitle;
        }
    }

    // Powiadom czytniki ekranu o zmianie strony
    announcePageChange(pageTitle) {
        // Twórz ukryty element dla czytników ekranu
        const announcement = document.createElement('div');
        announcement.setAttribute('aria-live', 'polite');
        announcement.setAttribute('aria-atomic', 'true');
        announcement.className = 'sr-only';
        announcement.textContent = `Przeszedłeś do sekcji: ${pageTitle}`;
        
        document.body.appendChild(announcement);
        
        // Usuń po krótkim czasie
        setTimeout(() => {
            document.body.removeChild(announcement);
        }, 1000);
    }

    // Formatowanie dla różnych typów stron
    formatTitle(type, data) {
        switch(type) {
            case 'dashboard':
                return this.setTitle('Panel główny', 'Przegląd aktywności');
            
            case 'profile':
                const userName = data?.name || 'użytkownika';
                return this.setTitle(`Profil ${userName}`, 'Ustawienia konta');
            
            case 'messages':
                const unreadCount = data?.unread || 0;
                return this.setTitleWithBadge('Wiadomości', unreadCount, 'Skrzynka odbiorcza');
            
            case 'settings':
                const section = data?.section || '';
                const context = section ? `Sekcja: ${section}` : 'Konfiguracja';
                return this.setTitle('Ustawienia', context);
            
            case 'error':
                const errorCode = data?.code || '404';
                return this.setTitle(`Błąd ${errorCode}`, 'Strona nie została znaleziona');
            
            case 'loading':
                return this.setTemporaryTitle('Ładowanie', data?.target || 'treści');
            
            default:
                return this.setTitle(data?.title || 'Strona główna');
        }
    }
}

// Inicjalizacja managera tytułów
const titleManager = new PageTitleManager('MyApp - System zarządzania');

// Przykłady użycia w SPA
function navigateToPage(route, data = {}) {
    // Symulacja nawigacji w SPA
    console.log(`Navigating to: ${route}`);
    
    // Aktualizuj tytuł zgodnie z nową stroną
    titleManager.formatTitle(route, data);
    
    // Tutaj normalmente byłaby logika renderowania treści
    updatePageContent(route, data);
}

// Funkcja pomocnicza dla demo

    const pageData = {
        dashboard: { },
        profile: { name: 'Jan Kowalski' },
        settings: { section: 'Powiadomienia' },
        messages: { unread: 3 }
    };
    
    navigateToPage(page, pageData[page]);
}

Rozwiązanie: Dynamiczne zarządzanie tytułami w aplikacjach SPA zapewnia, że użytkownicy zawsze wiedzą gdzie się znajdują, nawet gdy strona się nie przeładowuje. System automatycznie aktualizuje tytuły, obsługuje powiadomienia dla czytników ekranu i formatuje tytuły zgodnie z kontekstem.

Testowanie zgodności z 2.4.2

Procedura testowa

  1. Sprawdzenie obecności tytułu
    • Sprawdź czy każda strona ma element <title>
    • Upewnij się, że tytuł nie jest pusty
    • Sprawdź czy tytuł jest w sekcji <head>
  2. Ocena opisowości tytułu
    • Sprawdź czy tytuł opisuje temat lub cel strony
    • Oceń czy tytuł jest wystarczająco szczegółowy
    • Sprawdź czy tytuł nie jest zbyt ogólny
  3. Test z czytnikiem ekranu
    • Otwórz stronę z czytnikiem ekranu (NVDA, JAWS)
    • Sprawdź czy tytuł jest pierwszą informacją
    • Oceń czy tytuł jest zrozumiały bez kontekstu
  4. Test nawigacji
    • Otwórz kilka kart z różnymi stronami
    • Sprawdź czy można łatwo rozróżnić karty
    • Przetestuj dodawanie do zakładek

Narzędzia testowe

  • axe DevTools - sprawdza obecność i jakość tytułów
  • WAVE - analizuje tytuły stron
  • Lighthouse - ocenia SEO i dostępność tytułów
  • Browser DevTools - inspekcja elementu title
  • HeadingsMap - pokazuje strukturę nagłówków i tytuł

Checklist zgodności

✅ Kryteria do spełnienia:
  • ☐ Każda strona ma element <title>
  • ☐ Tytuł nie jest pusty
  • ☐ Tytuł opisuje temat lub cel strony
  • ☐ Tytuł jest wystarczająco szczegółowy
  • ☐ Tytuł pozwala rozróżnić strony
  • ☐ Tytuł jest zrozumiały bez kontekstu
  • ☐ W aplikacjach SPA tytuł aktualizuje się dynamicznie
  • ☐ Tytuł odzwierciedla bieżący stan aplikacji

Wytyczne implementacyjne

Najlepsze praktyki:

  • Konkretność - opisuj konkretną treść, nie ogólniki
  • Unikalność - każda strona powinna mieć unikalny tytuł
  • Długość - optymalna długość 30-60 znaków dla SEO
  • Struktura - używaj spójnego wzorca w całym serwisie
  • Kluczowe słowa - umieść najważniejsze informacje na początku
  • Aktualizacje - w SPA aktualizuj tytuł przy zmianie widoku

Wzorce tytułów:

1. Strona główna

3. Strony funkcjonalne

4. Strony błędów
[Typ błędu] - [Pomocna informacja] | [Serwis]
