2.4.1 Pomijanie bloków

Poziom A

Oficjalna dokumentacja W3C

Opis kryterium

Dostępny jest mechanizm pomijania bloków treści, które są powtarzane na wielu stronach internetowych.

Intencja

Celem tego kryterium sukcesu jest umożliwienie osobom nawigującym sekwencyjnie treścią szybszego dostępu do głównej treści strony internetowej. Strony i aplikacje często zawierają treści, które pojawiają się na wielu stronach lub ekranach. Przykłady takich powtarzających się bloków treści to:

  • Linki nawigacyjne - główne menu strony
  • Bannery i reklamy - powtarzające się na każdej stronie
  • Menu boczne - nawigacja sekcji
  • Formularze wyszukiwania - obecne w nagłówku
  • Stopki - informacje kontaktowe i linki

Dla osób korzystających z czytników ekranu lub nawigacji klawiaturą przechodzenie przez te same elementy na każdej stronie może być męczące i czasochłonne.

Przykłady

Zły przykład - brak mechanizmów pomijania

Strona bez linków pomijających ani właściwej struktury nagłówków:

Logo Strony

Strona główna O nas Produkty Usługi Blog Kontakt

Problem: Użytkownik musi przejść przez 6 linków nawigacji + formularz wyszukiwania na każdej stronie, zanim dotrze do treści głównej.

Dla osoby korzystającej z klawiatury oznacza to minimum 8 naciśnięć Tab na każdej stronie!

<!-- ZŁY PRZYKŁAD - brak mechanizmów pomijania -->
<!DOCTYPE html>
<html>
<head>
    <title>Strona bez dostępności</title>
</head>
<body>
    <!-- Brak linku pomijającego -->
    
    <header>
        <h1>Logo Strony</h1>
        <nav>
            <a href="/">Strona główna</a>
            <a href="/o-nas">O nas</a>
            <a href="/produkty">Produkty</a>
            <a href="/uslugi">Usługi</a>
            <a href="/blog">Blog</a>
            <a href="/kontakt">Kontakt</a>
        </nav>
        <form>
            <input type="search" placeholder="Szukaj...">
            <button type="submit">Szukaj</button>
        </form>
    </header>
    
    <!-- Brak oznaczenia treści głównej -->
    <div>
        <p>Tutaj jest treść główna...</p>
    </div>
</body>
</html>

Problem: Użytkownicy muszą przechodzić przez te same elementy nawigacyjne na każdej stronie. Brak jest mechanizmu pozwalającego pominąć powtarzające się treści i przejść bezpośrednio do głównej zawartości.

Dobry przykład - link pomijający do treści głównej

Implementacja podstawowego linku pomijającego:

Demo: Kliknij w tym obszarze i naciśnij Tab, aby zobaczyć link pomijający

Przejdź do treści głównej
Logo Strony
Home About Products Contact
✅ Treść główna (można od razu tutaj przeskoczyć!)

Tu jest główna zawartość strony, do której użytkownik może przejść bezpośrednio.

<!-- DOBRY PRZYKŁAD - podstawowy link pomijający -->
<!DOCTYPE html>
<html lang="pl">
<head>
    <title>Dostępna strona</title>
</head>
<body>
    <!-- Link pomijający - pierwszy element focusable -->
    <a href="#main" class="skip-link">Przejdź do treści głównej</a>
    
    <header>
        <h1>Logo Strony</h1>
        <nav aria-label="Nawigacja główna">
            <ul>
                <li><a href="/">Strona główna</a></li>
                <li><a href="/o-nas">O nas</a></li>
                <li><a href="/produkty">Produkty</a></li>
                <li><a href="/kontakt">Kontakt</a></li>
            </ul>
        </nav>
        <form role="search">
            <label for="search">Szukaj</label>
            <input type="search" id="search" name="search">
            <button type="submit">Szukaj</button>
        </form>
    </header>
    
    <!-- Oznaczona treść główna -->
    <main id="main">
        <h1>Tytuł strony</h1>
        <p>Tutaj jest treść główna strony...</p>
    </main>
</body>
</html>
/* CSS dla linku pomijającego */
.skip-link {
    position: absolute;
    top: -40px;
    left: 6px;
    background: #000;
    color: #fff;
    padding: 8px;
    text-decoration: none;
    border-radius: 0 0 4px 4px;
    font-weight: bold;
    z-index: 1000;
}

.skip-link:focus {
    top: 0;
}

/* Alternatywnie - zawsze widoczny link */
.skip-link-visible {
    display: block;
    background: #007bff;
    color: white;
    padding: 8px 16px;
    text-decoration: none;
    text-align: center;
    margin-bottom: 10px;
}

.skip-link-visible:hover,
.skip-link-visible:focus {
    background: #0056b3;
}

Rozwiązanie: Link pomijający pozwala użytkownikom pominąć powtarzające się elementy nawigacyjne i przejść bezpośrednio do głównej treści. Jest to szczególnie ważne dla osób korzystających z czytników ekranu i nawigacji klawiaturowej.

Dobry przykład - wielokrotne linki pomijające

Zaawansowany system linków pomijających do różnych sekcji:

Zaawansowane linki pomijające
Treść główna Menu boczne Wyszukiwarka Stopka
📄 Treść główna

Tu jest główna zawartość artykułu lub strony.

📋 Menu boczne

Dodatkowe linki i nawigacja sekcji.

🔍 Sekcja wyszukiwania

Zaawansowane opcje wyszukiwania i filtry.

📞 Stopka

Informacje kontaktowe i dodatkowe linki.

<!-- DOBRY PRZYKŁAD - wielokrotne linki pomijające -->
<!DOCTYPE html>
<html lang="pl">
<head>
    <title>Zaawansowane pomijanie bloków</title>
</head>
<body>
    <!-- Panel linków pomijających -->
    <nav aria-label="Linki pomijające" class="skip-links">
        <a href="#main">Przejdź do treści głównej</a>
        <a href="#sidebar">Przejdź do menu bocznego</a>
        <a href="#search">Przejdź do wyszukiwarki</a>
        <a href="#footer">Przejdź do stopki</a>
    </nav>
    
    <header>
        <h1>Logo Witryny</h1>
        <nav aria-label="Nawigacja główna">
            <!-- Główne menu -->
        </nav>
    </header>
    
    <div class="layout">
        <main id="main">
            <h1>Główna treść strony</h1>
            <!-- Zawartość główna -->
        </main>
        
        <aside id="sidebar" aria-label="Menu boczne">
            <h2>Dodatkowe opcje</h2>
            <!-- Menu boczne -->
        </aside>
    </div>
    
    <section id="search" aria-label="Wyszukiwanie">
        <h2>Zaawansowane wyszukiwanie</h2>
        <!-- Formularz wyszukiwania -->
    </section>
    
    <footer id="footer">
        <h2>Informacje kontaktowe</h2>
        <!-- Treść stopki -->
    </footer>
</body>
</html>

Rozwiązanie: Zaawansowany system linków pomijających pozwala użytkownikom przejść bezpośrednio do różnych sekcji strony. Jest to szczególnie przydatne na złożonych stronach z wieloma obszarami treści.

Dobry przykład - struktura nagłówków jako mechanizm pomijania

Wykorzystanie prawidłowej hierarchii nagłówków do nawigacji:

Struktura nagłówków pozwala czytnikowi ekranu:
  • Przeskakiwać między sekcjami (H key w NVDA/JAWS)
  • Nawigować po poziomach nagłówków (1-6 keys)
  • Generować spis treści (Elements List)
  • Orientować się w strukturze dokumentu

H2: Nawigacja główna

Użytkownik może pominąć tę sekcję

H2: Treść główna

Użytkownik może skoczyć tutaj z H2

H3: Podsekcja

Dalsze sekcje

H2: Menu boczne

Rozwiązanie: Prawidłowa struktura nagłówków (H1-H6) tworzy naturalny mechanizm pomijania bloków. Użytkownicy czytników ekranu mogą nawigować między sekcjami używając skrótów klawiaturowych i pomijać nieistotne części.

Dobry przykład - rozwijane sekcje (collapsible)

Mechanizm zwijania/rozwijania jako sposób na pomijanie bloków:

Strona główna O nas Produkty Kontakt
📄 Treść główna

Użytkownik może od razu przejść do tej sekcji, pomijając zwinięte bloki powyżej.

<!-- DOBRY PRZYKŁAD - rozwijane sekcje -->
<!DOCTYPE html>
<html lang="pl">
<head>
    <title>Strona z rozwijalnymi sekcjami</title>
</head>
<body>
    <header>
        <h1>Logo Strony</h1>
        
        <!-- Rozwijalna nawigacja -->
        <div class="collapsible-section">
            <button type="button" 
                    aria-expanded="false" 
                    aria-controls="main-nav"
                    class="collapsible-trigger">
                Nawigacja główna
                <span aria-hidden="true">▼</span>
            </button>
            <nav id="main-nav" class="collapsible-content" hidden>
                <ul>
                    <li><a href="/">Strona główna</a></li>
                    <li><a href="/o-nas">O nas</a></li>
                    <li><a href="/produkty">Produkty</a></li>
                    <li><a href="/kontakt">Kontakt</a></li>
                </ul>
            </nav>
        </div>
        
        <!-- Rozwijalne wyszukiwanie -->
        <div class="collapsible-section">
            <button type="button" 
                    aria-expanded="false" 
                    aria-controls="search-form"
                    class="collapsible-trigger">
                Zaawansowane wyszukiwanie
                <span aria-hidden="true">▼</span>
            </button>
            <div id="search-form" class="collapsible-content" hidden>
                <form role="search">
                    <label for="search-query">Wyszukiwana fraza:</label>
                    <input type="search" id="search-query" name="q">
                    
                    <label for="search-category">Kategoria:</label>
                    <select id="search-category" name="category">
                        <option value="">Wszystkie</option>
                        <option value="articles">Artykuły</option>
                        <option value="products">Produkty</option>
                    </select>
                    
                    <button type="submit">Szukaj</button>
                </form>
            </div>
        </div>
    </header>
    
    <main>
        <h1>Główna treść strony</h1>
        <p>Ta treść jest od razu dostępna, użytkownik nie musi 
           rozwijać powyższych sekcji aby do niej dotrzeć.</p>
    </main>
</body>
</html>
/* CSS dla rozwijanych sekcji */
.collapsible-trigger {
    width: 100%;
    padding: 12px 16px;
    background: #007bff;
    color: white;
    border: none;
    text-align: left;
    cursor: pointer;
    font-size: 1em;
    border-radius: 4px;
    margin-bottom: 1px;
    display: flex;
    justify-content: space-between;
    align-items: center;
}

.collapsible-trigger:hover,
.collapsible-trigger:focus {
    background: #0056b3;
    outline: 2px solid #fff;
    outline-offset: -2px;
}

.collapsible-trigger[aria-expanded="true"] {
    border-radius: 4px 4px 0 0;
}

.collapsible-trigger[aria-expanded="true"] span[aria-hidden] {
    transform: rotate(180deg);
}

.collapsible-content {
    background: #f8f9fa;
    border: 1px solid #dee2e6;
    border-top: none;
    border-radius: 0 0 4px 4px;
    padding: 16px;
}

.collapsible-content[hidden] {
    display: none;
}

/* Animacja rozwijania (opcjonalna) */
.collapsible-content {
    transition: max-height 0.3s ease;
    overflow: hidden;
}

/* Wskaźnik stanu */
.collapsible-trigger span[aria-hidden] {
    transition: transform 0.2s ease;
    font-size: 0.8em;
}
// JavaScript dla rozwijanych sekcji
document.addEventListener('DOMContentLoaded', function() {
    const triggers = document.querySelectorAll('.collapsible-trigger');
    
    triggers.forEach(trigger => {
        trigger.addEventListener('click', function() {
            const isExpanded = this.getAttribute('aria-expanded') === 'true';
            const targetId = this.getAttribute('aria-controls');
            const targetContent = document.getElementById(targetId);
            
            // Przełącz stan
            this.setAttribute('aria-expanded', !isExpanded);
            
            if (isExpanded) {
                targetContent.setAttribute('hidden', '');
            } else {
                targetContent.removeAttribute('hidden');
            }
            
            // Zaktualizuj tekst przycisku dla czytników ekranu
            const statusText = isExpanded ? 'zwinięte' : 'rozwinięte';
            this.setAttribute('aria-label', 
                this.textContent.trim() + ' - ' + statusText);
        });
    });
    
    // Dodaj obsługę klawiatury
    triggers.forEach(trigger => {
        trigger.addEventListener('keydown', function(event) {
            // Enter i Space działają jak click
            if (event.key === 'Enter' || event.key === ' ') {
                event.preventDefault();
                this.click();
            }
        });
    });
});

Rozwiązanie: Rozwijane sekcje pozwalają użytkownikom pomijać niepotrzebne bloki treści poprzez pozostawienie ich w stanie zwiniętym. Wszystkie elementy są dostępne z klawiatury i właściwie komunikują swój stan czytnikowi ekranu.

Testowanie zgodności z 2.4.1

Procedura testowa

  1. Identyfikacja powtarzających się bloków
    • Znajdź elementy powtarzające się na wielu stronach
    • Oceń czy stanowią przeszkodę w dotarciu do głównej treści
    • Sprawdź czy są to pierwsze elementy w kolejności tabulacji
  2. Test nawigacji klawiaturowej
    • Rozpocznij nawigację od początku strony (Tab)
    • Sprawdź czy dostępne są mechanizmy pomijania
    • Przetestuj czy linki pomijające działają poprawnie
  3. Test z czytnikiem ekranu
    • Użyj NVDA, JAWS lub VoiceOver
    • Sprawdź nawigację po nagłówkach (H key)
    • Przetestuj linki pomijające
    • Oceń strukturę dokumentu (Elements List)
  4. Weryfikacja skuteczności
    • Zmierz liczbę kroków do głównej treści
    • Porównaj z i bez mechanizmów pomijania
    • Sprawdź różne ścieżki nawigacji

Narzędzia testowe

  • axe DevTools - wykrywa brakujące mechanizmy pomijania
  • WAVE - analizuje strukturę nagłówków
  • HeadingsMap - wizualizuje hierarchię nagłówków
  • Lighthouse - sprawdza podstawowe problemy dostępności
  • Keyboard Navigation Tester - symuluje nawigację klawiaturową

Checklist zgodności

✅ Kryteria do spełnienia:
  • ☐ Dostępne są linki pomijające do głównej treści
  • ☐ Linki pomijające są pierwszymi elementami focusable
  • ☐ Linki pomijające działają poprawnie (przenoszą fokus)
  • ☐ Struktura nagłówków jest logiczna i hierarchiczna
  • ☐ Można nawigować tylko po nagłówkach
  • ☐ Główne sekcje mają nagłówki lub landmarks
  • ☐ Mechanizmy pomijania są dostępne z klawiatury
  • ☐ Alternatywne metody (collapsible) działają poprawnie

Wytyczne implementacyjne

Najlepsze praktyki:

  • Umieść linki pomijające na początku - jako pierwsze elementy focusable
  • Używaj opisowych tekstów - "Przejdź do treści głównej", nie "Pomiń"
  • Zapewnij wizualny feedback - pokaż linki przy focus
  • Testuj z rzeczywistymi użytkownikami - szczególnie z osobami niewidomymi
  • Oferuj multiple opcje - linki pomijające + struktura nagłówków
  • Zachowaj spójność - te same mechanizmy na całej stronie

Popularne wzorce:

1. Klasyczny link pomijający
<a href="#main" class="skip-link">Skip to main content</a>
2. Wielokrotne linki pomijające
<nav aria-label="Skip links">
  <a href="#main">Main content</a>
  <a href="#sidebar">Sidebar</a>
</nav>
3. Struktura nagłówków
<h1>Page title</h1>
<h2>Navigation</h2>
<h2>Main content</h2>
<h2>Sidebar</h2>
4. ARIA landmarks
<main>...</main>
<nav aria-label="Main">...</nav>
<aside>...</aside>
