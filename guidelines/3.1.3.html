Przejdź do treści głównej

3.1.3 Nietypowe słowa (AAA)

Mechanizm identyfikacji specyficznych definicji słów lub fraz użytych w nietypowy lub ograniczony sposób, włączając idiomy i żargon, jest dostępny.

Opis kryterium

Gdy strona zawiera słowa lub wyrażenia używane w sposób nietypowy, specjalistyczny lub jako żargon, musi być dostępny mechanizm wyjaśniający ich znaczenie. Pomaga to użytkownikom zrozumieć treść, zwłaszcza gdy słowa są używane w kontekście innym niż powszechnie przyjęty.

Kto odnosi korzyść

  • Osoby z ograniczeniami poznawczymi - łatwiejsze zrozumienie specjalistycznej terminologii
  • Osoby z dysleksją - dodatkowe wsparcie w rozumieniu tekstów
  • Osoby uczące się języka - pomoc w rozumieniu idiomów i żargonu
  • Młodzi użytkownicy - wyjaśnienie skomplikowanych pojęć
  • Osoby starsze - objaśnienie nowoczesnego slangu i terminologii technicznej
  • Specjaliści z innych dziedzin - zrozumienie branżowego żargonu

Wymagania

  • Mechanizm definiowania nietypowych słów musi być dostępny
  • Definicje muszą być jasne i zrozumiałe
  • Mechanizm może obejmować różne sposoby dostarczania definicji
  • Wyjaśnienia muszą być dostępne dla technologii wspomagających

Definicje

Nietypowe słowa
Słowa używane w sposób odmienny od powszechnie przyjętego znaczenia lub w specjalistycznym kontekście.
Idiomy
Wyrażenia, których znaczenie nie wynika bezpośrednio z znaczenia poszczególnych słów (np. "leje jak z cebra").
Żargon
Specjalistyczne słownictwo używane w określonej grupie zawodowej lub społecznej.
Mechanizm definicji
Sposób dostarczania wyjaśnień - może to być słowniczek, linki do definicji, tooltips, lub inne rozwiązania.
Użycie w ograniczony sposób
Słowa używane ze znaczeniem specyficznym dla danego kontekstu, dokumentu lub dziedziny.

Przykłady

❌ Nieprawidłowe implementacje

Brak wyjaśnienia żargonu technicznego

<p>Nasz API wykorzystuje RESTful endpoints z autentykacją JWT 
dla seamless integration z Vašymi systemami.</p>

Problem: Specjalistyczne terminy (API, RESTful, JWT, seamless integration) nie są wyjaśnione.

Idiomy bez wyjaśnienia

<p>Ten projekt to już przeszłość, teraz trzeba iść z duchem czasu 
i nie sprzedawać skóry na niedźwiedziu.</p>

Problem: Idiomy mogą być niezrozumiałe dla osób uczących się języka lub młodszych użytkowników.

Branżowy slang bez kontekstu

<p>Frontend developer powinien znać responsive design, 
cross-browser compatibility i progressive enhancement.</p>

Problem: Branżowe terminy IT nie są wyjaśnione dla osób spoza branży.

✅ Prawidłowe implementacje

Definicje w elemencie dfn

<p>Nasz <dfn title="Application Programming Interface - interfejs programistyczny">API</dfn> 
wykorzystuje <dfn title="Architektura oprogramowania zgodna z zasadami REST">RESTful endpoints</dfn> 
z autentykacją <dfn title="JSON Web Token - standard tokenu dostępu">JWT</dfn>.</p>

Dlaczego działa: Każdy termin techniczny ma wyjaśnienie dostępne przez atrybut title.

Słowniczek terminów

<p>Podczas tworzenia aplikacji używamy metodologii 
<a href="#agile-def">Agile</a> i praktyk <a href="#devops-def">DevOps</a>.</p>

<section id="glossary">
    <h2>Słowniczek</h2>
    <dl>
        <dt id="agile-def">Agile</dt>
        <dd>Metodologia tworzenia oprogramowania oparta na iteracyjnym 
            rozwoju i ścisłej współpracy z klientem.</dd>
        
        <dt id="devops-def">DevOps</dt>
        <dd>Zestaw praktyk łączących rozwój oprogramowania (Dev) 
            z operacjami IT (Ops) w celu skrócenia cyklu życia systemu.</dd>
    </dl>
</section>

Dlaczego działa: Terminy są linkowane do szczegółowych definicji w słowniczku.

Wyjaśnienia idiomów

<p>Ten projekt to już przeszłość, teraz trzeba 
<span class="idiom" title="dostosować się do aktualnych trendów">iść z duchem czasu</span> 
i nie <span class="idiom" title="nie planować czegoś przedwcześnie">sprzedawać skóry na niedźwiedziu</span>.</p>

Dlaczego działa: Idiomy mają wyjaśnienia pokazujące ich rzeczywiste znaczenie.

Interaktywne definicje

<p>Implementujemy <button type="button" class="term-button" 
data-term="microservices">mikrousługi</button> z wykorzystaniem 
<button type="button" class="term-button" 
data-term="containers">kontenerów</button>.</p>

<div class="definitions-panel" hidden>
    <div id="def-microservices">
        <strong>Mikrousługi:</strong> Architektura aplikacji jako zbioru 
        małych, niezależnych usług komunikujących się przez API.
    </div>
    <div id="def-containers">
        <strong>Kontenery:</strong> Lekkie, przenośne jednostki oprogramowania 
        zawierające aplikację i wszystkie jej zależności.
    </div>
</div>

Dlaczego działa: Interaktywne elementy pozwalają na dostęp do definicji na żądanie.

Sposoby implementacji

1. Atrybut title

Zastosowanie: Proste, krótkie definicje

Zalety: Łatwe w implementacji, dostępne dla czytników ekranu

Wady: Ograniczona długość, nie wszyscy użytkownicy wiedzą o hover

<p>Używamy <span title="Rozwój oprogramowania oparty na testach">TDD</span> 
w naszych projektach.</p>

2. Element dfn z linkami

Zastosowanie: Pierwszy raz użyte terminy z definicjami

Zalety: Semantycznie poprawne, linkuje do szczegółowych wyjaśnień

Wady: Wymaga dodatkowej strony lub sekcji z definicjami

<p>Metodologia <dfn><a href="glossary.html#scrum">Scrum</a></dfn> 
jest popularna w zespołach programistycznych.</p>

3. Słowniczek terminów

Zastosowanie: Strony z wieloma specjalistycznymi terminami

Zalety: Centralny punkt z wszystkimi definicjami

Wady: Użytkownik musi przechodzić do innej sekcji

<section id="glossary">
    <h2>Słowniczek</h2>
    <dl>
        <dt id="ajax">AJAX</dt>
        <dd>Asynchronous JavaScript and XML - technika tworzenia 
            interaktywnych aplikacji webowych.</dd>
    </dl>
</section>

4. Tooltips i popovery

Zastosowanie: Interaktywne wyjaśnienia na żądanie

Zalety: Nie zakłócają czytania, dostępne na żądanie

Wady: Wymagają JavaScript, muszą być dostępne dla klawiatury

<button type="button" class="term-toggle" 
        aria-describedby="tooltip-rest">REST</button>
<div id="tooltip-rest" role="tooltip" hidden>
    Representational State Transfer - architektura dla usług webowych
</div>

5. Sekcje wyjaśnień

Zastosowanie: Szczegółowe wyjaśnienia skomplikowanych konceptów

Zalety: Miejsce na obszerne wyjaśnienia z przykładami

Wady: Mogą przerwać czytanie głównej treści

<p>Implementujemy architekturę <a href="#microservices-explanation">mikrousług</a>.</p>

<aside id="microservices-explanation">
    <h3>Czym są mikrousługi?</h3>
    <p>Mikrousługi to podejście do budowy aplikacji jako zestawu 
       małych usług, z których każda działa w swoim procesie...</p>
</aside>

Narzędzie do wykrywania terminów bez definicji:


            
// Wykrywanie potencjalnych terminów technicznych
function detectTechnicalTerms() {
    const technicalPatterns = [
        // Akronimy (3+ wielkie litery)
        /\b[A-Z]{3,}\b/g,
        // Terminy IT
        /\b(API|REST|JSON|XML|HTTP|CSS|HTML|JavaScript|React|Vue|Angular|Node\.js|Docker|Kubernetes|AWS|Azure|GCP)\b/gi,
        // Terminy biznesowe
        /\b(ROI|KPI|B2B|B2C|CRM|ERP|SaaS|PaaS|IaaS)\b/gi,
        // Metodologie
        /\b(Agile|Scrum|DevOps|CI\/CD|TDD|BDD)\b/gi
    ];
    
    const pageText = document.body.textContent;
    const foundTerms = new Set();
    
    technicalPatterns.forEach(pattern => {
        const matches = pageText.match(pattern);
        if (matches) {
            matches.forEach(match => foundTerms.add(match));
        }
    });
    
    return Array.from(foundTerms);
}

// Sprawdzanie czy terminy mają definicje
function checkTermDefinitions() {
    const technicalTerms = detectTechnicalTerms();
    const undefinedTerms = [];
    
    technicalTerms.forEach(term => {
        // Sprawdź czy termin ma definicję
        const hasDefinition = 
            document.querySelector(`[title*="${term}"]`) ||
            document.querySelector(`dfn:contains("${term}")`) ||
            document.querySelector(`#glossary [id*="${term.toLowerCase()}"]`) ||
            document.querySelector(`[data-term="${term.toLowerCase()}"]`);
            
        if (!hasDefinition) {
            undefinedTerms.push(term);
        }
    });
    
    return {
        totalTerms: technicalTerms.length,
        undefinedTerms: undefinedTerms,
        definitionCoverage: ((technicalTerms.length - undefinedTerms.length) / technicalTerms.length * 100).toFixed(1)
    };
}

// Walidacja dostępności definicji
function validateDefinitionAccessibility() {
    const issues = [];
    
    // Sprawdź tooltips
    const tooltips = document.querySelectorAll('[role="tooltip"]');
    tooltips.forEach(tooltip => {
        if (!tooltip.id) {
            issues.push('Tooltip bez ID: nie może być powiązany z elementem wyzwalającym');
        }
        
        const trigger = document.querySelector(`[aria-describedby="${tooltip.id}"]`);
        if (!trigger) {
            issues.push(`Tooltip #${tooltip.id} nie ma elementu wyzwalającego`);
        }
    });
    
    // Sprawdź interaktywne elementy
    const termButtons = document.querySelectorAll('.term-button, .term-toggle');
    termButtons.forEach(button => {
        if (!button.getAttribute('aria-describedby') && !button.getAttribute('aria-expanded')) {
            issues.push('Przycisk terminu bez odpowiednich atrybutów ARIA');
        }
    });
    
    return issues;
}

// Uruchom wszystkie testy
const termsAnalysis = checkTermDefinitions();
const accessibilityIssues = validateDefinitionAccessibility();

console.log('Analiza terminów:', termsAnalysis);
console.log('Problemy z dostępnością:', accessibilityIssues);

// Shortcode dla definicji terminów
function glossary_term_shortcode($atts, $content = null) {
    $atts = shortcode_atts(array(
        'term' => '',
        'definition' => '',
        'type' => 'title' // title, tooltip, link
    ), $atts);
    
    switch($atts['type']) {
        case 'tooltip':
            return sprintf(
                '<button type="button" class="term-button" aria-describedby="tooltip-%s">%s</button>
                <div id="tooltip-%s" role="tooltip" hidden>%s</div>',
                sanitize_title($atts['term']),
                $content ?: $atts['term'],
                sanitize_title($atts['term']),
                esc_html($atts['definition'])
            );
        case 'link':
            return sprintf(
                '<a href="#def-%s">%s</a>',
                sanitize_title($atts['term']),
                $content ?: $atts['term']
            );
        default:
            return sprintf(
                '<dfn title="%s">%s</dfn>',
                esc_attr($atts['definition']),
                $content ?: $atts['term']
            );
    }
}
add_shortcode('term', 'glossary_term_shortcode');

React Component

            
import React, { useState } from 'react';

const GlossaryTerm = ({ 
    term, 
    definition, 
    type = 'title', 
    children 
}) => {
    const [showTooltip, setShowTooltip] = useState(false);
    
    const termId = `term-${term.toLowerCase().replace(/\s+/g, '-')}`;
    const tooltipId = `tooltip-${termId}`;
    
    switch(type) {
        case 'tooltip':
            return (
                <>
                    <button
                        type="button"
                        className="term-button"
                        aria-describedby={tooltipId}
                        onMouseEnter={() => setShowTooltip(true)}
                        onMouseLeave={() => setShowTooltip(false)}
                        onFocus={() => setShowTooltip(true)}
                        onBlur={() => setShowTooltip(false)}
                    >
                        {children || term}
                    </button>
                    {showTooltip && (
                        <div
                            id={tooltipId}
                            role="tooltip"
                            className="tooltip"
                        >
                            {definition}
                        </div>
                    )}
                </>
            );
        
        case 'link':
            return (
                <a href={`#def-${termId}`}>
                    {children || term}
                </a>
            );
        
        default:
            return (
                <dfn title={definition}>
                    {children || term}
                </dfn>
            );
    }
};

export default GlossaryTerm;

// GlossaryTerm.vue
<template>
    <dfn 
        v-if="type === 'title'"

            :id="tooltipId"
            role="tooltip"
            class="tooltip"
        >
            {{ definition }}
        </div>
    </div>
    
    <a v-else :href="`#def-${termSlug}`">

Automatyczne wykrywanie terminów

            
class AutoGlossary {
    constructor(glossaryData) {
        this.glossary = new Map(Object.entries(glossaryData));
        this.processed = new Set();
    }
    
    // Automatycznie dodaj definicje do pierwszego wystąpienia terminu
    autoAnnotate(container = document.body) {
        this.glossary.forEach((definition, term) => {
            if (this.processed.has(term)) return;
            
            const regex = new RegExp(`\\b${this.escapeRegex(term)}\\b`, 'i');
            const walker = document.createTreeWalker(
                container,
                NodeFilter.SHOW_TEXT,
                null,
                false
            );
            
            let node;
            while (node = walker.nextNode()) {
                if (regex.test(node.textContent)) {
                    this.wrapTerm(node, term, definition);
                    this.processed.add(term);
                    break;
                }
            }
        });
    }
    
    wrapTerm(textNode, term, definition) {
        const parent = textNode.parentNode;
        const text = textNode.textContent;
        const regex = new RegExp(`\\b(${this.escapeRegex(term)})\\b`, 'i');
        
        if (regex.test(text)) {
            const newHTML = text.replace(regex, 
                `<dfn title="${this.escapeHtml(definition)}">$1</dfn>`
            );
            
            const wrapper = document.createElement('span');
            wrapper.innerHTML = newHTML;
            parent.replaceChild(wrapper, textNode);
        }
    }
    
    escapeRegex(string) {
        return string.replace(/[.*+?^${}()|[\]\\]/g, '\\$&');
    }
    
    escapeHtml(string) {
        return string.replace(/[&<>"']/g, (match) => {
            const escapeMap = { '&': '&', '<': '<', '>': '>', '"': '"', "'": ''' };
            return escapeMap[match];
        });
    }
}

// Użycie
const glossary = new AutoGlossary({
    'API': 'Application Programming Interface - interfejs programistyczny',
    'REST': 'Representational State Transfer - architektura usług webowych',
    'JSON': 'JavaScript Object Notation - format wymiany danych'
});

glossary.autoAnnotate();


            
System kontekstowych definicji

            
class ContextualGlossary {
    constructor() {
        this.definitions = new Map();
        this.contexts = new Map();
    }
    
    // Dodaj definicję z kontekstem
    addDefinition(term, definition, context = 'general') {
        if (!this.definitions.has(term)) {
            this.definitions.set(term, new Map());
        }
        this.definitions.get(term).set(context, definition);
    }
    
    // Pobierz definicję w kontekście
    getDefinition(term, context = 'general') {
        const termDefs = this.definitions.get(term);
        if (!termDefs) return null;
        
        // Sprawdź czy jest definicja dla tego kontekstu
        if (termDefs.has(context)) {
            return termDefs.get(context);
        }
        
        // Fallback do definicji ogólnej
        return termDefs.get('general') || null;
    }
    
    // Ustaw kontekst dla sekcji
    setContext(element, context) {
        element.setAttribute('data-context', context);
        
        // Zaktualizuj wszystkie terminy w tej sekcji
        const terms = element.querySelectorAll('[data-term]');
        terms.forEach(term => {
            const termName = term.getAttribute('data-term');
            const definition = this.getDefinition(termName, context);
            if (definition) {
                term.setAttribute('title', definition);
            }
        });
    }
}

// Przykład użycia
const contextGlossary = new ContextualGlossary();

// Różne definicje w różnych kontekstach
contextGlossary.addDefinition('bank', 'Instytucja finansowa', 'finance');
contextGlossary.addDefinition('bank', 'Brzeg rzeki', 'geography');
contextGlossary.addDefinition('bank', 'Miejsce składowania danych', 'it');
