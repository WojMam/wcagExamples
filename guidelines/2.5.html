2.5 Sposoby wprowadzania danych

Wytyczna ta zapewnia, że użytkownicy mogą w prosty sposób wprowadzać dane za pomocą różnych urządzeń i metod wejścia.

Opis wytycznej

Nowoczesne urządzenia oferują wiele sposobów wprowadzania danych - od tradycyjnej klawiatury i myszy, przez ekrany dotykowe, po sterowanie gestami i ruchem. Użytkownicy muszą być w stanie:

  • Używać różnych metod wprowadzania - nie tylko gestów czy ruchu
  • Kontrolować swoje działania - móc anulować przypadkowe aktywacje
  • Rozpoznawać elementy interfejsu - etykiety zgodne z tekstem mówionym
  • Unikać przypadkowych aktywacji - przez niewłaściwe ruchy urządzeniem
  • Trafić w obszary dotyku - odpowiednio duże cele
  • Łączyć różne metody wejścia - jednocześnie używać klawiatury, myszy i dotyku

Te elementy są szczególnie istotne dla osób:

  • Z ograniczeniami motorycznymi - które mają trudności z precyzyjnymi gestami
  • Z drżeniem rąk - które mogą przypadkowo aktywować elementy
  • Używających technologii wspomagających - które mogą inaczej interpretować gesty
  • Z ograniczeniami poznawczymi - które potrzebują spójnych nazw elementów
  • Starszych - które mogą mieć problemy z nowymi metodami wprowadzania

Kryteria sukcesu

Poziom A (podstawowy)

  • 2.5.1 Gesty wskazujące (A) - alternatywy dla wielopunktowych gestów
  • 2.5.2 Anulowanie wskazania (A) - możliwość cofnięcia działań wskaźnika
  • 2.5.3 Etykieta w nazwie (A) - zgodność widocznego tekstu z nazwą dostępną
  • 2.5.4 Aktywacja ruchem (A) - alternatywy dla funkcji sterowanych ruchem

Poziom AAA (zaawansowany)

  • 2.5.5 Rozmiar obszaru dotyku (AAA) - minimalne rozmiary celów dotyku 44×44 piksele
  • 2.5.6 Mechanizmy wprowadzania współbieżnego (AAA) - obsługa wszystkich dostępnych mechanizmów wprowadzania

Kluczowe zasady wprowadzania danych

👆 Elastyczność gestów

Interfejs powinien oferować:

  • Alternatywy dla złożonych gestów
  • Opcje anulowania działań
  • Prostsze metody aktywacji
  • Wsparcie dla różnych urządzeń

🎯 Precyzja i kontrola

Użytkownicy muszą mieć:

  • Kontrolę nad przypadkowymi aktywacjami
  • Możliwość cofnięcia działań
  • Odpowiednio duże obszary dotyku
  • Wyraźne granice elementów interaktywnych

🏷️ Spójność nazewnictwa

Etykiety elementów powinny:

  • Być zgodne z nazwami programowymi
  • Odzwierciedlać tekst mówiony
  • Używać zrozumiałego języka
  • Pozostawać spójne w całym interfejsie

Metody wprowadzania danych

1. Wprowadzanie wskazujące

  • Mysz - tradycyjne klikanie i przeciąganie
  • Ekran dotykowy - dotyk, przeciąganie, gesty wielopunktowe
  • Rysik/stylus - precyzyjne wskazywanie
  • Trackpad - gesty wielopalcowe

2. Wprowadzanie ruchem

  • Przyspieszeniomierz - pochylanie urządzenia
  • Żyroskop - obrót urządzenia
  • Kamera - śledzenie ruchu ciała
  • Mikrofon - komenty głosowe

3. Wprowadzanie alternatywne

  • Klawiatura - nawigacja klawiszami
  • Przełączniki - dla osób z ograniczeniami motorycznymi
  • Śledzenie wzroku - kontrola oczami
  • Sterowanie głosem - komendy mówione

Funkcje dostępności w wprowadzaniu danych

Alternatywy dla gestów

<!-- Przycisk z alternatywą dla gestu -->
<div class="image-gallery">
    <button id="prev" aria-label="Poprzednie zdjęcie">‹</button>
    <img src="image.jpg" alt="Opis zdjęcia">
    <button id="next" aria-label="Następne zdjęcie">›</button>
</div>

<!-- Zamiast tylko gestu swipe -->
<script>
// Obsługa gestów z alternatywą
element.addEventListener('click', handleNavigation);
element.addEventListener('touchstart', handleGesture);
</script>

Anulowanie działań

// Anulowanie na up event zamiast down
button.addEventListener('pointerdown', function(e) {
    // Wizualna informacja o naciśnięciu
    this.classList.add('pressed');
});

button.addEventListener('pointerup', function(e) {
    this.classList.remove('pressed');
    // Faktyczne działanie dopiero tutaj
    performAction();
});

// Możliwość anulowania przez przeciągnięcie poza element
button.addEventListener('pointerleave', function(e) {
    this.classList.remove('pressed');
    // Anuluj działanie
});

Dostępne obszary dotyku

/* Minimalne rozmiary obszarów dotyku */
.touch-target {
    min-width: 44px;
    min-height: 44px;
    padding: 8px;
}

/* Dodatkowy obszar kliknięcia */
.small-icon {
    position: relative;
}

.small-icon::after {
    content: '';
    position: absolute;
    top: -8px;
    left: -8px;
    right: -8px;
    bottom: -8px;
}

Testowanie sposobów wprowadzania

Testy manualne

  • Różne urządzenia - testuj na telefonie, tablecie, laptopie
  • Tylko jeden palec - sprawdź czy można używać bez gestów wielopunktowych
  • Tylko klawiatura - czy wszystkie funkcje są dostępne
  • Symulacja drżenia - testuj z nieustabilizowanymi ruchami
  • Różne rozmiary palców - czy obszary dotyku są wystarczające

Narzędzia automatyczne

  • axe-core - sprawdza rozmiary obszarów dotyku
  • WAVE - analizuje etykiety i nazwy
  • Lighthouse - audit dostępności mobilnej
  • DevTools - symulacja różnych urządzeń

Checklist wprowadzania danych

✅ Sprawdź czy:

  • ☐ Złożone gesty mają alternatywy
  • ☐ Można anulować przypadkowe aktywacje
  • ☐ Etykiety zawierają tekst mówiony
  • ☐ Ruch urządzenia nie jest jedynym sposobem aktywacji
  • ☐ Obszary dotyku mają minimum 44×44 piksele
  • ☐ Można używać różnych metod wprowadzania jednocześnie
  • ☐ Interfejs działa bez gestów wielopunktowych
  • ☐ Wszystkie funkcje dostępne są z klawiatury
