1.4.10 Zawijanie tekstu

Poziom AA

Oficjalna dokumentacja W3C

Opis kryterium

Treść może być prezentowana bez utraty informacji lub funkcjonalności oraz bez konieczności przewijania w dwóch wymiarach dla:

  • Treści przewijanej w pionie przy szerokości równoważnej 320 CSS pikseli;
  • Treści przewijanej w poziomie przy wysokości równoważnej 256 CSS pikseli.

Z wyjątkiem części treści, które wymagają układu dwuwymiarowego ze względu na sposób użytkowania lub znaczenie.

Intencja

Celem tego kryterium jest zapewnienie, że treść może być oglądana i obsługiwana na małych ekranach lub przez użytkowników, którzy potrzebują powiększać tekst. Kryterium "Zawijanie tekstu" gwarantuje, że treść strony dostosuje się do dostępnej przestrzeni, bez konieczności przewijania w poziomie, co znacznie utrudnia czytanie i nawigację.

To kryterium jest szczególnie istotne dla użytkowników:

  • korzystających z urządzeń mobilnych
  • z wadami wzroku, którzy muszą powiększać treść
  • z zaburzeniami poznawczymi, dla których przewijanie w dwóch wymiarach może być dezorientujące

Przykłady

Zły przykład - Strona z ustaloną szerokością

Strona zaprojektowana z ustaloną szerokością, która wymaga przewijania poziomego na wąskich ekranach:

<style>
.fixed-width-layout {
    width: 900px; /* Stała szerokość */
    margin: 0 auto;
}

.content-column {
    width: 600px;
    float: left;
}

.sidebar {
    width: 280px;
    float: right;
}

table {
    width: 580px;
    border-collapse: collapse;
}
</style>

<div class="fixed-width-layout">
    <div class="content-column">
        <h1>Nasza oferta</h1>
        <p>Lorem ipsum dolor sit amet, consectetur adipiscing elit. Sed at purus a quam dignissim condimentum.</p>
        
        <table border="1">
            <tr>
                <th>Usługa</th>
                <th>Opis</th>
                <th>Cena</th>
            </tr>
            <tr>
                <td>Usługa 1</td>
                <td>Szczegółowy opis usługi 1</td>
                <td>100 zł</td>
            </tr>
            <tr>
                <td>Usługa 2</td>
                <td>Szczegółowy opis usługi 2</td>
                <td>200 zł</td>
            </tr>
        </table>
    </div>
    
    <div class="sidebar">
        <h2>Aktualności</h2>
        <p>Najnowsze informacje o naszej firmie.</p>
    </div>
</div>

Problem: Układ o stałej szerokości 900px nie dostosuje się do małych ekranów (np. 320px). Użytkownik musi przewijać poziomo, aby zobaczyć całą zawartość, co znacznie utrudnia korzystanie ze strony.

Problemy w tym przykładzie:

  • Układ strony ma ustaloną szerokość 900px, która nie dostosowuje się do mniejszych ekranów
  • Dwukolumnowy układ z ustalonymi szerokościami nie zawijają się na wąskich ekranach
  • Tabela z ustaloną szerokością wychodzi poza granice ekranu mobilnego
  • Użytkownik musi przewijać w poziomie, aby zobaczyć całą zawartość
  • Osoby powiększające treść (np. ze względu na problemy ze wzrokiem) będą miały trudności z korzystaniem ze strony

Dobry przykład - Responsywny układ strony

Responsywny układ, który dostosowuje się do szerokości ekranu:

<style>
.responsive-layout {
    width: 100%;
    max-width: 1200px;
    margin: 0 auto;
    box-sizing: border-box;
    padding: 0 15px;
}

.row::after {
    content: "";
    clear: both;
    display: table;
}

.content-column {
    width: 100%;
    box-sizing: border-box;
}

.sidebar {
    width: 100%;
    box-sizing: border-box;
    margin-top: 20px;
}

table {
    width: 100%;
    border-collapse: collapse;
    table-layout: fixed;
}

th, td {
    word-wrap: break-word;
}

/* Dla większych ekranów - układ dwukolumnowy */
@media (min-width: 768px) {
    .content-column {
        width: 65%;
        float: left;
        padding-right: 15px;
    }
    
    .sidebar {
        width: 35%;
        float: right;
        padding-left: 15px;
        margin-top: 0;
    }
}

/* Na bardzo małych ekranach upraszczamy tabelę */
@media (max-width: 480px) {
    table, thead, tbody, th, td, tr {
        display: block;
    }
    
    thead tr {
        position: absolute;
        top: -9999px;
        left: -9999px;
    }
    
    tr {
        border: 1px solid #ccc;
        margin-bottom: 10px;
    }
    
    td {
        border: none;
        border-bottom: 1px solid #eee;
        position: relative;
        padding-left: 50%;
    }
    
    td:before {
        position: absolute;
        top: 6px;
        left: 6px;
        width: 45%;
        padding-right: 10px;
        white-space: nowrap;
        font-weight: bold;
    }
    
    td:nth-of-type(1):before { content: "Usługa"; }
    td:nth-of-type(2):before { content: "Opis"; }
    td:nth-of-type(3):before { content: "Cena"; }
}
</style>

</div>

Aktualności

Najnowsze informacje o naszej firmie.

Rozwiązanie: Responsywny układ dostosowuje się do szerokości ekranu. Na wąskich ekranach kolumny układają się jedna pod drugą, a tabela jest przekształcana w format bardziej przyjazny dla urządzeń mobilnych, eliminując potrzebę przewijania poziomego.

Zalety tego rozwiązania:

  • Układ stosuje szerokość względną (100%) i dostosowuje się do dostępnej przestrzeni
  • Media queries są używane do zmiany układu w zależności od rozmiaru ekranu
  • Na mniejszych ekranach dwukolumnowy układ zmienia się w jednokolumnowy (kolumny układają się jedna pod drugą)
  • Tabela jest przekształcana na wąskich ekranach, aby zapobiec przewijaniu poziomemu
  • Zawartość można łatwo powiększyć do 200% bez potrzeby przewijania poziomego
  • Model pudełkowy (box-sizing: border-box) gwarantuje, że padding i border są wliczane w całkowitą szerokość elementów

Zły przykład - Obrazy o stałych wymiarach

Strona z dużym obrazem o ustalonej szerokości, który nie dopasowuje się do ekranu:

    <h1>Nasza galeria</h1>
    <p>Zobacz zdjęcia z naszego ostatniego wydarzenia:</p>

</div>

Problem: Obraz o stałej szerokości 800px wymusza przewijanie poziome na wąskich ekranach (np. 320px). Użytkownik musi przewijać, aby zobaczyć cały obraz i tekst.

Problemy w tym przykładzie:

  • Obraz ma ustawione sztywne wymiary (width="800" height="500"), które nie dopasowują się do szerokości ekranu
  • Na urządzeniach mobilnych lub przy dużym powiększeniu, obraz wymusza przewijanie poziome
  • Brak technik responsywnego wyświetlania obrazów

Dobry przykład - Responsywne obrazy

Strona z obrazem, który dostosowuje się do szerokości ekranu:

<style>
.responsive-content {
    width: 100%;
    max-width: 1200px;
    margin: 0 auto;
    padding: 0 15px;
    box-sizing: border-box;
}

.responsive-image {
    max-width: 100%;
    height: auto;
    display: block;
}

/* Opcjonalnie: Zastosowanie art direction dla różnych rozmiarów ekranu */
.image-container {
    width: 100%;
    margin: 20px 0;
}

figure {
    margin: 0;
}

figcaption {
    font-style: italic;
    margin-top: 8px;
    font-size: 0.9em;
}
</style>

<div class="responsive-content">
    <h1>Nasza galeria</h1>
    <p>Zobacz zdjęcia z naszego ostatniego wydarzenia:</p>
    
    <div class="image-container">
        <figure>
            <!-- Responsywny obraz -->
            <picture>
                <source media="(max-width: 480px)" srcset="event-photo-mobile.jpg">
                <source media="(max-width: 768px)" srcset="event-photo-tablet.jpg">
                <img src="event-photo.jpg" alt="Zdjęcie z wydarzenia" class="responsive-image">
            </picture>
            <figcaption>Uczestnicy naszego ostatniego spotkania branżowego.</figcaption>
        </figure>
    </div>
    
    <p>Spotkanie odbyło się w dniach 10-12 maja i wzięło w nim udział ponad 100 specjalistów z branży.</p>
</div>
Podgląd (wizualizacja na wąskim ekranie):

Nasza galeria

Zobacz zdjęcia z naszego ostatniego wydarzenia:

Spotkanie odbyło się w dniach 10-12 maja i wzięło w nim udział ponad 100 specjalistów z branży.

Rozwiązanie: Obraz automatycznie dostosowuje się do szerokości ekranu dzięki właściwości max-width: 100%. Dodatkowo, element picture pozwala na dostarczenie różnych wersji obrazu w zależności od rozmiaru ekranu.

Zalety tego rozwiązania:

  • Obraz dostosowuje się do szerokości kontenera dzięki właściwości max-width: 100%; height: auto;
  • Element <picture> umożliwia zastosowanie różnych wersji obrazu dla różnych rozmiarów ekranu (art direction)
  • Na małych ekranach można dostarczyć mniejsze, bardziej przycięte wersje obrazów, co poprawia użyteczność i wydajność
  • Obraz nie wymusza przewijania poziomego, nawet na bardzo wąskich ekranach
  • Strona zachowuje czytelność przy dowolnym poziomie powiększenia, co jest kluczowe dla użytkowników z wadami wzroku

Wskazówki i techniki

  • Wykorzystuj responsywny design - Projektuj strony z myślą o różnych rozmiarach ekranów, stosując podejście "mobile first"
  • Używaj względnych jednostek - Stosuj procentowe szerokości, jednostki em, rem zamiast stałych pikseli
  • Dodawaj metatag viewport - <meta name="viewport" content="width=device-width, initial-scale=1.0"> zapewnia prawidłowe skalowanie na urządzeniach mobilnych
  • Używaj Media Queries - Dostosowuj układ strony do różnych szerokości ekranów za pomocą CSS Media Queries
  • Stosuj elastyczne obrazy - Używaj max-width: 100%; height: auto; dla obrazów, aby automatycznie dopasowywały się do szerokości kontenera
  • Responsywne tabele - Przekształcaj tabele na wąskich ekranach (np. zmieniając je w karty lub listy) lub implementuj poziome przewijanie tylko dla tabeli, nie dla całej strony
  • Zapewniaj odpowiednie odstępy dotykowe - Elementy klikalne powinny mieć wystarczającą wielkość (min. 44x44px) i odpowiednie odstępy na ekranach dotykowych
  • Testuj na różnych urządzeniach - Sprawdzaj swoją stronę na różnych rozmiarach ekranów i poziomach powiększenia
  • Używaj CSS Flexbox i Grid - Te nowoczesne techniki CSS ułatwiają tworzenie elastycznych układów
  • Uwzględniaj wyjątki - Pamiętaj, że niektóre elementy, jak mapy czy skomplikowane wykresy, mogą wymagać dwuwymiarowego układu
