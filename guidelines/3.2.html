3.2 Przewidywalność

Wytyczna ta zapewnia, że strony internetowe pojawiają się i działają w przewidywalny sposób.

Opis wytycznej

Przewidywalność w interfejsach użytkownika oznacza, że użytkownicy mogą przewidzieć, co się stanie w odpowiedzi na ich działania. Wytyczna 3.2 koncentruje się na:

  • Stabilności interfejsu - elementy nie zmieniają się niespodziewanie
  • Spójności zachowań - podobne elementy działają podobnie
  • Kontroli użytkownika - zmiany następują na żądanie
  • Jasnej komunikacji - użytkownik wie, czego się spodziewać

Te elementy są szczególnie istotne dla osób:

  • Z ograniczeniami poznawczymi - które potrzebują przewidywalnych wzorców
  • Używających czytników ekranu - które polegają na spójnej strukturze
  • Z niepełnosprawnościami motorycznymi - które mają trudności z nawigacją
  • Z zaburzeniami uwagi - które mogą być rozpraszane przez niespodziewane zmiany
  • Starszych - które preferują znane wzorce interakcji
  • Początkujących użytkowników - które uczą się obsługi interfejsu

Kryteria sukcesu

Poziom A (podstawowy)

  • 3.2.1 Po otrzymaniu fokusa (A) - komponenty nie zmieniają kontekstu przy otrzymaniu fokusa
  • 3.2.2 Podczas wpisywania (A) - zmiana ustawień nie powoduje automatycznej zmiany kontekstu

Poziom AA (standardowy)

  • 3.2.3 Spójna nawigacja (AA) - mechanizmy nawigacji są spójne na wszystkich stronach
  • 3.2.4 Spójna identyfikacja (AA) - komponenty o tej samej funkcjonalności są identyfikowane spójnie

Poziom AAA (zaawansowany)

  • 3.2.5 Zmiana na żądanie (AAA) - zmiany kontekstu inicjowane są tylko przez użytkownika lub dostępny jest mechanizm ich wyłączenia

Kluczowe zasady przewidywalności

🎯 Kontrola użytkownika

Użytkownik powinien mieć kontrolę nad:

  • Kiedy następują zmiany w interfejsie
  • Czy i jak są otwierane nowe okna
  • Czy formularze są automatycznie wysyłane
  • Kiedy odtwarzane są multimedia

🔄 Spójność

Elementy interfejsu powinny być:

  • Umieszczone w tych samych miejscach
  • Oznaczone w ten sam sposób
  • Zachowywać się podobnie
  • Mieć przewidywalne funkcje

⚡ Stabilność

Interfejs powinien:

  • Nie zmieniać się niespodziewanie
  • Ostrzegać przed ważnymi zmianami
  • Dawać możliwość cofnięcia akcji
  • Zachowywać stan po błędach

📢 Komunikacja

Użytkownik powinien wiedzieć:

  • Co się stanie po kliknięciu
  • Gdzie zostanie przeniesiony
  • Jakie dane są wymagane
  • Czy akcja jest nieodwracalna

Typy zmian kontekstu

Zmiany kontekstu to:

Otwarcie nowego okna/karty
Przeniesienie użytkownika do innej aplikacji lub strony bez jego wyraźnej zgody
Przeniesienie fokusa
Automatyczne przeniesienie fokusa do innego elementu bez działania użytkownika
Zmiana treści
Istotna modyfikacja zawartości strony, która zmienia znaczenie lub funkcjonalność
Reorganizacja layoutu
Przesunięcie, ukrycie lub pokazanie głównych elementów nawigacji

Co NIE jest zmianą kontekstu:

  • Rozwinięcie menu dropdown na tym samym miejscu
  • Pokazanie tooltipa lub pomocy kontekstowej
  • Zmiana zawartości w wydzielonym obszarze (np. tab panel)
  • Dodanie komunikatów o błędach lub sukcesie
  • Progresywne ładowanie treści (infinite scroll)

Przykłady

❌ Nieprawidłowe implementacje

Automatyczne otwieranie nowego okna

<!-- Nieprawidłowe - otwiera nowe okno bez ostrzeżenia -->
<a href="external-site.com" target="_blank">Zobacz więcej</a>

Problem: Użytkownik nie wie, że link otworzy nowe okno.

Automatyczne wysyłanie formularza

<!-- Nieprawidłowe - natychmiastowa submisja po zmianie -->
<select onchange="this.form.submit()">
    <option>Opcja 1</option>
    <option>Opcja 2</option>
</select>

Problem: Formularz się wysyła bez możliwości sprawdzenia innych opcji.

Niespójne umiejscowienie nawigacji

<!-- Strona 1 - menu u góry -->
<header>
    <nav>Menu główne</nav>
</header>

<!-- Strona 2 - menu z boku -->
<div class="sidebar">
    <nav>Menu główne</nav>
</div>

Problem: Menu znajduje się w różnych miejscach na różnych stronach.

✅ Prawidłowe implementacje

Ostrzeżenie o nowym oknie

<!-- Prawidłowe - informuje o nowym oknie -->
<a href="external-site.com" target="_blank" rel="noopener">
    Zobacz więcej 
    <span class="sr-only">(otwiera się w nowym oknie)</span>
    <span aria-hidden="true">↗</span>
</a>

Dlaczego działa: Użytkownik jest informowany o otwarciu nowego okna.

Kontrolowane wysyłanie formularza

<!-- Prawidłowe - zmiana nie powoduje submisji -->
<form>
    <label for="category-select">Wybierz kategorię:</label>
    <select id="category-select" name="category">
        <option>Kategoria 1</option>
        <option>Kategoria 2</option>
    </select>
    
    <button type="submit">Zastosuj filtr</button>
</form>

Dlaczego działa: Użytkownik musi świadomie wysłać formularz przyciskiem.

Spójna nawigacja

<!-- Każda strona ma tę samą strukturę nawigacji -->
<header class="site-header">
    <nav class="main-navigation" aria-label="Nawigacja główna">
        <ul>
            <li><a href="/">Strona główna</a></li>
            <li><a href="/produkty">Produkty</a></li>
            <li><a href="/kontakt">Kontakt</a></li>
        </ul>
    </nav>
</header>

Dlaczego działa: Nawigacja jest zawsze w tym samym miejscu i ma tę samą strukturę.

Kontrolowana zmiana kontekstu

<!-- Ostrzeżenie o przekierowaniu -->
<div class="redirect-notice">
    <p>Za 10 sekund zostaniesz przekierowany do nowej strony.</p>
    <button onclick="redirectNow()">Przejdź teraz</button>
    <button onclick="cancelRedirect()">Anuluj przekierowanie</button>
</div>

<script>
let redirectTimer;

function startRedirectTimer() {
    redirectTimer = setTimeout(() => {
        window.location.href = '/new-page';
    }, 10000);
}

function cancelRedirect() {
    clearTimeout(redirectTimer);
    document.querySelector('.redirect-notice').style.display = 'none';
}

function redirectNow() {
    clearTimeout(redirectTimer);
    window.location.href = '/new-page';
}
</script>

Dlaczego działa: Użytkownik jest informowany o przekierowaniu i może je kontrolować.

Wzorce spójności

Spójna nawigacja

  • Położenie - menu w tym samym miejscu na każdej stronie
  • Kolejność - elementy menu w takiej samej kolejności
  • Nazewnictwo - te same nazwy dla tych samych sekcji
  • Wygląd - spójne style i zachowania

Spójna identyfikacja

  • Ikony - ta sama ikona oznacza tę samą funkcję
  • Etykiety - podobne funkcje mają podobne nazwy
  • Kolory - spójne znaczenie kolorów (np. czerwony = błąd)
  • Zachowania - podobne elementy reagują podobnie

Kontrola zmian

  • Ostrzeżenia - informowanie o nadchodzących zmianach
  • Potwierdzenia - wymóg akceptacji ważnych akcji
  • Anulowanie - możliwość cofnięcia zmian
  • Status - komunikowanie stanu operacji

Testowanie przewidywalności

Testy manualne

  • Nawigacja klawiaturą - sprawdzenie czy fokus nie skacze niespodziewanie
  • Test różnych przeglądarek - czy zachowania są spójne
  • Porównanie stron - czy nawigacja jest jednakowa
  • Test użytkowników - czy interfejs jest przewidywalny

Testy automatyczne

// Test spójności nawigacji
function testNavigationConsistency() {
    const currentPageNav = document.querySelector('nav');
    const navItems = Array.from(currentPageNav.querySelectorAll('a')).map(a => ({
        text: a.textContent.trim(),
        href: a.getAttribute('href'),
        order: Array.from(a.parentElement.children).indexOf(a)
    }));
    
    // Zapisz strukturę nawigacji
    localStorage.setItem('expectedNavigation', JSON.stringify(navItems));
    
    return {
        items: navItems.length,
        structure: navItems
    };
}

// Test niespodziewanych zachowań
function testUnexpectedBehaviors() {
    const issues = [];
    
    // Sprawdź elementy z onchange
    document.querySelectorAll('[onchange]').forEach(element => {
        const handler = element.getAttribute('onchange');
        if (handler.includes('submit') || handler.includes('location')) {
            issues.push({
                element: element,
                issue: 'Potencjalna niespodziewana zmiana kontekstu'
            });
        }
    });
    
    // Sprawdź linki target="_blank" bez ostrzeżenia
    document.querySelectorAll('a[target="_blank"]').forEach(link => {
        const hasWarning = link.textContent.includes('nowym oknie') ||
                          link.getAttribute('aria-label')?.includes('nowym oknie');
        if (!hasWarning) {
            issues.push({
                element: link,
                issue: 'Link otwiera nowe okno bez ostrzeżenia'
            });
        }
    });
    
    return issues;
}

// Test fokusa
function testFocusManagement() {
    let focusTraps = 0;
    let unexpectedFocusChanges = 0;
    
    document.addEventListener('focus', function(e) {
        console.log('Focus moved to:', e.target);
        // Monitoruj nieoczekiwane zmiany fokusa
    }, true);
    
    return {
        focusTraps: focusTraps,
        unexpectedChanges: unexpectedFocusChanges
    };
}

Checklist przewidywalności

✅ Sprawdź czy:

  • ☐ Otrzymanie fokusa nie zmienia kontekstu
  • ☐ Zmiana wartości nie powoduje automatycznych akcji
  • ☐ Nawigacja jest spójna między stronami
  • ☐ Podobne elementy są identyfikowane podobnie
  • ☐ Linki otwierające nowe okna są oznaczone
  • ☐ Użytkownik kontroluje ważne zmiany
  • ☐ Błędy nie powodują utraty danych
  • ☐ Stan interfejsu jest jasno komunikowany
