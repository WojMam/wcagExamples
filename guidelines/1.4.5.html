1.4.5 Obrazy tekstu

Poziom AA

Oficjalna dokumentacja W3C

Opis kryterium

Jeśli wykorzystywane technologie mogą przedstawiać treść wizualnie, do przekazywania informacji wykorzystuje się tekst, a nie obrazy tekstu, z wyjątkiem następujących przypadków:

  • Możliwość dostosowania: Obraz tekstu może być dostosowany wizualnie do wymagań użytkownika;
  • Istotność: Konkretna prezentacja tekstu jest niezbędna dla przekazania informacji.

Intencja

Celem tego kryterium jest zachęcenie do używania tekstu zamiast obrazów tekstu, kiedy technologia na to pozwala. Tekst może być przedstawiany w sposób, który spełnia potrzeby większości użytkowników bez konieczności stosowania obrazów.

Rzeczywisty tekst HTML (w przeciwieństwie do obrazów zawierających tekst) ma wiele zalet:

  • Może być łatwo skalowany bez utraty jakości
  • Wygląd może być dostosowany do potrzeb użytkownika
  • Kontrast między tekstem a tłem może być dostosowany
  • Jest dostępny dla technologii wspomagających
  • Może być wyszukiwany, zaznaczany i kopiowany

Przykłady

Zły przykład - Nagłówek jako obraz tekstu

Strona używająca obrazu do wyświetlenia nagłówka, zamiast użycia tekstu HTML:

<!-- Nagłówek jako obraz bez możliwości dostosowania -->
<header>
    <img src="header-title.png" alt="Nowoczesne Rozwiązania IT" width="400" height="80">
    <p>Profesjonalne usługi informatyczne dla firm</p>
</header>

<main>
    <h2>Nasza oferta</h2>
    <p>Treść strony...</p>
</main>
Podgląd:
[Obraz: Nowoczesne Rozwiązania IT]

Profesjonalne usługi informatyczne dla firm

Nasza oferta

Treść strony...

Problem: Nagłówek jest obrazem, który nie może być dostosowany do preferencji użytkownika. Nie można zmienić jego rozmiaru, kontrastu czy kroju pisma.

Problem: Użycie obrazu do wyświetlenia nagłówka strony powoduje następujące problemy:

  • Tekst na obrazie nie może być powiększony niezależnie od reszty strony
  • Użytkownicy nie mogą dostosować kroju pisma ani innych aspektów wizualnych
  • Tekst na obrazie nie zmienia się przy zmianie kontrastu czy kolorów przez użytkownika
  • Obraz może tracić jakość przy powiększaniu strony
  • Może wyglądać rozmazany na ekranach o wysokiej gęstości pikseli

Dobry przykład - Nagłówek jako tekst HTML

Ta sama strona, ale z nagłówkiem jako tekst HTML, który może być dostosowany:

<style>
.site-title {
    font-size: 2.5rem;
    font-weight: bold;
    color: #3f51b5;
    margin-bottom: 0.5rem;
}

.site-subtitle {
    font-size: 1.2rem;
    color: #555;
}
</style>

<header>
    <h1 class="site-title">Nowoczesne Rozwiązania IT</h1>
    <p class="site-subtitle">Profesjonalne usługi informatyczne dla firm</p>
</header>

<main>
    <h2>Nasza oferta</h2>
    <p>Treść strony...</p>
</main>
Podgląd - Normalny:

Nowoczesne Rozwiązania IT

Profesjonalne usługi informatyczne dla firm

Nasza oferta

Treść strony...

Podgląd - Dostosowany (Większa czcionka, inny kolor):

Nowoczesne Rozwiązania IT

Profesjonalne usługi informatyczne dla firm

Nasza oferta

Treść strony...

Rozwiązanie: Tekst HTML może być dostosowany przez użytkownika - powiększony, zmieniony kolorystycznie i sformatowany według preferencji.

Rozwiązanie: Użycie prawdziwego tekstu HTML zamiast obrazu przynosi wiele korzyści:

  • Tekst może być powiększony razem z resztą strony
  • Użytkownicy mogą zmienić krój pisma, kolor i inne właściwości za pomocą swoich preferencji
  • Czytniki ekranu mogą poprawnie odczytać tekst
  • Tekst jest ostry i czytelny na wszystkich urządzeniach, w tym na ekranach o wysokiej gęstości pikseli
  • Zmniejsza rozmiar pliku strony (tekst HTML jest mniejszy niż obrazy)

Zły przykład - Cytat jako obraz

Strona wyświetlająca cytat jako obraz, bez możliwości dostosowania:

<!-- Cytat jako obraz -->
<div class="testimonial">
    <img src="quote.png" alt="Korzystanie z usług tej firmy to najlepsza decyzja biznesowa, jaką podjęliśmy w tym roku. - Jan Kowalski, Dyrektor Marketingu" width="600" height="150">
</div>
Podgląd:
"Korzystanie z usług tej firmy to najlepsza decyzja biznesowa, jaką podjęliśmy w tym roku."
- Jan Kowalski, Dyrektor Marketingu

Problem: Ten cytat jest obrazem, więc użytkownicy nie mogą dostosować jego tekstu ani wyglądu.

Problem: Umieszczenie cytatu jako obrazu ogranicza dostępność treści, ponieważ:

  • Użytkownicy nie mogą dostosować rozmiaru, koloru czy kroju pisma tekstu
  • Cytowany tekst nie może być wyszukiwany
  • Cytat nie może być zaznaczony i skopiowany
  • Technologie wspomagające mogą mieć problem z prawidłowym odczytaniem długiego tekstu alternatywnego

Dobry przykład - Cytat jako tekst HTML

Ten sam cytat, ale przedstawiony jako tekst HTML z formatowaniem CSS:

<style>
.testimonial {
    background-color: #f5f5f5;
    padding: 30px;
    border-radius: 5px;
    margin-bottom: 2rem;
    position: relative;
}

.testimonial blockquote {
    font-style: italic;
    font-size: 1.2rem;
    color: #333;
    margin: 0 0 1rem 0;
    padding: 0 2rem;
    position: relative;
}

.testimonial blockquote::before,
.testimonial blockquote::after {
    content: '"';
    font-size: 3rem;
    color: #ddd;
    position: absolute;
}

.testimonial blockquote::before {
    left: 0;
    top: -10px;
}

.testimonial blockquote::after {
    right: 0;
    bottom: -30px;
}

.testimonial cite {
    font-style: normal;
    font-weight: bold;
    display: block;
    text-align: right;
}
</style>

<div class="testimonial">
    <blockquote>
        Korzystanie z usług tej firmy to najlepsza decyzja biznesowa, jaką podjęliśmy w tym roku.
        <cite>Jan Kowalski, Dyrektor Marketingu</cite>
    </blockquote>
</div>
Podgląd - Normalny:
" Korzystanie z usług tej firmy to najlepsza decyzja biznesowa, jaką podjęliśmy w tym roku. " Jan Kowalski, Dyrektor Marketingu
Podgląd - Dostosowany (Większa czcionka, inny kontrast):
" Korzystanie z usług tej firmy to najlepsza decyzja biznesowa, jaką podjęliśmy w tym roku. " Jan Kowalski, Dyrektor Marketingu

Rozwiązanie: Cytat jako tekst HTML może być dostosowany przez użytkownika - powiększony, zmieniony kolorystycznie i sformatowany według preferencji.

Rozwiązanie: Użycie znaczników HTML (blockquote, cite) z formatowaniem CSS zamiast obrazu z cytatem:

  • Pozwala użytkownikom na dostosowanie wyglądu tekstu
  • Umożliwia powiększanie i zmianę kontrastu zgodnie z preferencjami
  • Używa semantycznych znaczników HTML, co poprawia dostępność
  • Pozwala na wyszukiwanie i kopiowanie tekstu
  • Używa efektów CSS do uzyskania atrakcyjnego wyglądu bez ograniczania dostępności

Dopuszczalny przykład - Logo i znaki towarowe

Zgodnie z wyjątkiem "istotności", logo i znaki towarowe mogą być przedstawiane jako obrazy tekstu:

<!-- Logo i znak towarowy mogą być obrazami tekstu -->
<header>
    <img src="company-logo.png" alt="Firma XYZ" width="200" height="80">
    
    <nav>
        <!-- Menu nawigacyjne jako tekst HTML -->
        <ul>
            <li><a href="#">Strona główna</a></li>
            <li><a href="#">O nas</a></li>
            <li><a href="#">Kontakt</a></li>
        </ul>
    </nav>
</header>
Podgląd:
[Logo Firma XYZ]
  • Strona główna
  • O nas
  • Kontakt

Akceptowalne: Logo firmy może być obrazem tekstu, ponieważ konkretna prezentacja wizualna jest istotna dla identyfikacji marki.

Wyjaśnienie: Logo i znaki towarowe są objęte wyjątkiem "istotności" z następujących powodów:

  • Konkretna prezentacja wizualna jest niezbędna dla identyfikacji marki
  • Logo często zawiera specjalnie zaprojektowane kroje pisma i elementy graficzne, które są częścią tożsamości marki
  • Zgodnie z WCAG, prezentacja tekstu jest "istotna", gdy zmiana jej wizualnych aspektów zmieniłaby znaczenie przekazu

Ważne jest jednak, aby reszta strony (nawigacja, treść) używała rzeczywistego tekstu HTML, który może być dostosowany.

Wskazówki i techniki

  • Używaj tekstu HTML - Zawsze kiedy to możliwe, używaj tekstu HTML zamiast umieszczać tekst na obrazach
  • Wykorzystuj CSS - Nowoczesne możliwości CSS (web fonts, text-shadow, gradients) pozwalają na tworzenie atrakcyjnych wizualnie tekstów
  • SVG dla tekstu graficznego - Jeśli potrzebujesz graficznego tekstu, rozważ użycie SVG, który jest skalowany bez utraty jakości
  • Ogranicz użycie obrazów tekstu - Używaj ich tylko w sytuacjach, gdy konkretna prezentacja wizualna jest niezbędna
  • Zapewnij alternatywy - Jeśli musisz użyć obrazu tekstu, zapewnij odpowiedni tekst alternatywny
  • Używaj modern web fonts - Dzięki nim możesz używać niestandardowych fontów bez konieczności tworzenia obrazów tekstowych
  • Testuj responsywność - Upewnij się, że tekst dobrze wygląda na różnych urządzeniach i przy różnych rozmiarach ekranu
  • Unikaj tekstu na złożonych tłach - Jeśli tekst musi być na złożonym tle, użyj raczej HTML+CSS z tłem, a nie obrazu z tekstem
