1.3.4 Orientacja Poziom AA

Wyświetlanie i obsługa treści nie jest ograniczona tylko do jednej orientacji wyświetlania, takiej jak pionowa lub pozioma, chyba że konkretna orientacja wyświetlania jest niezbędna.

Intencja

Celem tego kryterium jest zapewnienie, że treści mogą być obsługiwane i wyświetlane w orientacji preferowanej przez użytkownika. Niektórzy użytkownicy mają swoje urządzenia zamontowane w stałej orientacji (np. przymocowane do podłokietnika wózka inwalidzkiego), a inni mogą mieć trudności z obracaniem urządzenia lub zmianą jego pozycji. Strony internetowe i aplikacje, które wymuszają określoną orientację (np. tylko poziomą) mogą uniemożliwić tym osobom wygodne korzystanie z treści.

Przykłady

Zły przykład - wymuszanie orientacji poziomej

Aplikacja lub strona internetowa, która blokuje wyświetlanie w orientacji pionowej i wyświetla komunikat nakazujący obrócenie urządzenia do pozycji poziomej:

@media screen and (orientation: portrait) {
    .app-content {
        display: none;
    }
    
    .rotate-message {
        display: flex;
        height: 100vh;
        align-items: center;
        justify-content: center;
        text-align: center;
        background-color: #f0f0f0;
    }
}

@media screen and (orientation: landscape) {
    .app-content {
        display: block;
    }
    
    .rotate-message {
        display: none;
    }
}
<div class="rotate-message">
    <div>
        <img src="../images/rotate-device.svg" alt="Ikona obracania urządzenia">
        <h2>Proszę obrócić urządzenie</h2>
        <p>Ta aplikacja działa tylko w trybie poziomym.</p>
    </div>
</div>

<div class="app-content">
    <!-- Zawartość aplikacji widoczna tylko w trybie poziomym -->
</div>

W tym przykładzie cała zawartość aplikacji jest ukryta, gdy urządzenie jest w orientacji pionowej. Zamiast tego wyświetlany jest komunikat nakazujący użytkownikowi obrócenie urządzenia. Jest to problem dla osób, które nie mogą zmienić orientacji swojego urządzenia.

Dobry przykład - responsywny design obsługujący obie orientacje

Strona internetowa zaprojektowana tak, aby dostosowywała się do obu orientacji, używając responsywnego designu:

/* Style dla obu orientacji */
.app-content {
    display: block;
    max-width: 1200px;
    margin: 0 auto;
    padding: 20px;
}

/* Dostosowania dla orientacji pionowej */
@media screen and (orientation: portrait) {
    .dashboard-grid {
        display: grid;
        grid-template-columns: 1fr;
        gap: 20px;
    }
}

/* Dostosowania dla orientacji poziomej */
@media screen and (orientation: landscape) {
    .dashboard-grid {
        display: grid;
        grid-template-columns: 1fr 1fr 1fr;
        gap: 20px;
    }
}
<div class="app-content">
    <h1>Panel główny</h1>
    
    <div class="dashboard-grid">
        <div class="dashboard-card">
            <h2>Statystyki</h2>
            <!-- Zawartość karty -->
        </div>
        
        <div class="dashboard-card">
            <h2>Nowe wiadomości</h2>
            <!-- Zawartość karty -->
        </div>
        
        <div class="dashboard-card">
            <h2>Ostatnie aktywności</h2>
            <!-- Zawartość karty -->
        </div>
    </div>
</div>

W tym przykładzie strona używa responsywnego designu, aby dostosować układ w zależności od orientacji urządzenia. W orientacji poziomej karty są wyświetlane w trzech kolumnach, a w orientacji pionowej jedna pod drugą. Wszystkie treści są dostępne niezależnie od orientacji.

Przykład wyjątku - aplikacja fortepianowa

Aplikacja do nauki gry na fortepianie, która sugeruje orientację poziomą, ale nadal umożliwia korzystanie w trybie pionowym:

/* Style dla obu orientacji */
.piano-app {
    display: block;
    max-width: 1200px;
    margin: 0 auto;
    padding: 20px;
}

.orientation-suggestion {
    background-color: #fff3cd;
    border: 1px solid #ffeeba;
    padding: 10px;
    margin-bottom: 20px;
    border-radius: 4px;
    display: none;
}

/* W trybie pionowym pokaż sugestię i zmniejsz klawiaturę */
@media screen and (orientation: portrait) {
    .orientation-suggestion {
        display: block;
    }
    
    .piano-keyboard {
        transform: scale(0.7);
        transform-origin: left top;
        overflow-x: auto;
    }
}

/* W trybie poziomym ukryj sugestię i pokaż pełną klawiaturę */
@media screen and (orientation: landscape) {
    .piano-keyboard {
        width: 100%;
    }
}
<div class="piano-app">
    <div class="orientation-suggestion">
        <p><strong>Sugestia:</strong> Aby zobaczyć pełną klawiaturę, obróć urządzenie do pozycji poziomej.</p>
    </div>
    
    <h1>Wirtualny fortepian</h1>
    
    <div class="piano-keyboard">
        <!-- Klawisze fortepianu -->
    </div>
</div>

Ten przykład przedstawia aplikację, której naturalnym układem jest orientacja pozioma (ze względu na kształt klawiatury fortepianu). Jednak zamiast całkowicie blokować dostęp w orientacji pionowej, aplikacja wyświetla sugestię i dostosowuje interfejs, aby był nadal użyteczny. Jest to zgodne z wytycznymi, ponieważ nie wymusza konkretnej orientacji, a jedynie sugeruje tę bardziej optymalną.

Wyzwania i wskazówki

  • Unikaj blokowania orientacji - Nie używaj metaznaczników viewport, JavaScript ani CSS do blokowania orientacji wyświetlania.
  • Implementuj responsywny design - Zaprojektuj interfejs, który płynnie dostosowuje się do różnych orientacji i rozmiarów ekranów.
  • Testuj w obu orientacjach - Podczas testowania sprawdź, czy wszystkie funkcje i treści są dostępne zarówno w orientacji pionowej, jak i poziomej.
  • Sugeruj, nie wymuszaj - Jeśli pewna orientacja jest bardziej optymalna dla konkretnej treści, możesz zasugerować zmianę orientacji, ale zawsze zapewnij dostęp do treści niezależnie od orientacji.
  • Uwzględnij wyjątki - Pamiętaj, że są pewne przypadki, gdy konkretna orientacja może być niezbędna dla funkcjonalności (np. niektóre gry, specjalistyczne aplikacje), ale nawet wtedy staraj się zapewnić jak najwięcej dostępności.
