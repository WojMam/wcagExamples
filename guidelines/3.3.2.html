3.3.2 Etykiety lub instrukcje (A)

Etykiety lub instrukcje są dostępne, gdy zawartość wymaga wprowadzenia danych przez użytkownika.

Opis kryterium

To kryterium poziomu A zapewnia, że użytkownicy wiedzą, jakie informacje należy wprowadzić w formularzach i kontrolkach. Kluczowe jest, aby:

  • Każde pole wymagające danych miało etykietę lub instrukcję
  • Etykiety były opisowe i zrozumiałe
  • Instrukcje były dostępne gdy potrzebne
  • Formaty danych były jasno określone
  • Wymagane pola były oznaczone

Etykiety lub instrukcje powinny pomagać użytkownikom w zrozumieniu:

  • Jakiego typu dane należy wprowadzić
  • W jakim formacie wprowadzić dane
  • Które pola są wymagane
  • Jakie są reguły walidacji
  • Jak używać kontrolek interfejsu

Kto odnosi korzyść

  • Osoby z ograniczeniami poznawczymi - otrzymują jasne wskazówki co robić
  • Osoby niewidome - mogą poznać przeznaczenie pól przez czytniki ekranu
  • Osoby z trudnościami językowymi - otrzymują jasne instrukcje
  • Osoby z dysleksją - mają dostęp do formatów i przykładów
  • Seniorzy - otrzymują dodatkowe wsparcie przy nowych technologiach
  • Wszyscy użytkownicy - mogą szybciej i dokładniej wypełnić formularze

Definicje

Etykieta
Tekst lub inny komponent z alternatywą tekstową prezentowany użytkownikowi w celu identyfikacji komponentu w treści internetowej
Instrukcja
Wskazówka lub wyjaśnienie dotyczące sposobu wprowadzania danych lub używania kontrolki
Kontrolka formularza
Element interfejsu, który pozwala użytkownikowi na wprowadzenie danych lub dokonanie wyboru
Dane wprowadzane przez użytkownika
Informacje, które użytkownik musi wprowadzić w celu ukończenia zadania
Format danych
Określony wzorzec lub struktura, w jakiej dane powinny być wprowadzone

Przykłady

❌ Nieprawidłowe implementacje

Brak etykiet w polach

<!-- Nieprawidłowe - pola bez etykiet -->
<form>
    <input type="text" placeholder="Wpisz tutaj..." />
    <input type="email" placeholder="email@example.com" />
    <input type="tel" />
    <textarea></textarea>
    <button type="submit">Wyślij</button>
</form>

<!-- Placeholder nie zastępuje etykiety -->
<input type="text" placeholder="Imię i nazwisko" />

Problem: Użytkownicy nie wiedzą, jakiego typu danych się od nich oczekuje.

Niejasne etykiety

<!-- Zbyt ogólne etykiety -->
<label for="data">Data:</label>
<input type="text" id="data" />

<label for="nazwa">Nazwa:</label>
<input type="text" id="nazwa" />

<!-- Brak instrukcji dla skomplikowanych formatów -->
<label for="phone">Telefon:</label>
<input type="tel" id="phone" />

Problem: Nie ma jasności co do oczekiwanego formatu lub typu danych.

Brak oznaczenia wymaganych pól

<!-- Nie wiadomo które pola są wymagane -->
<form>
    <div>
        <label for="first-name">Imię</label>
        <input type="text" id="first-name" required />
    </div>
    <div>
        <label for="last-name">Nazwisko</label>
        <input type="text" id="last-name" required />
    </div>
    <div>
        <label for="middle-name">Drugie imię</label>
        <input type="text" id="middle-name" />
    </div>
</form>

Problem: Użytkownik nie wie które pola musi wypełnić.

✅ Prawidłowe implementacje

Jasne etykiety z oznaczeniem wymaganych pól

<!-- Prawidłowe etykiety z oznaczeniem wymaganych pól -->
<form>
    <div class="form-group">
        <label for="first-name">
            Imię: <span aria-label="wymagane">*</span>
        </label>
        <input type="text" 
               id="first-name" 
               aria-required="true" 
               required />
    </div>
    
    <div class="form-group">
        <label for="last-name">
            Nazwisko: <span aria-label="wymagane">*</span>
        </label>
        <input type="text" 
               id="last-name" 
               aria-required="true" 
               required />
    </div>
    
    <div class="form-group">
        <label for="middle-name">Drugie imię: (opcjonalne)</label>
        <input type="text" id="middle-name" />
    </div>
    
    <p class="form-note">
        <span aria-hidden="true">*</span> Pola oznaczone gwiazdką są wymagane
    </p>
</form>

Zalety: Jasne oznaczenie wymaganych pól, dostępne dla technologii asystujących.

Instrukcje formatów danych

<!-- Pola z instrukcjami formatów -->
<div class="form-group">
    <label for="birth-date">Data urodzenia: *</label>
    <input type="date" 
           id="birth-date" 
           aria-describedby="birth-date-help" 
           aria-required="true" 
           required />
    <div id="birth-date-help" class="help-text">
        Format: DD/MM/RRRR (np. 15/03/1990)
    </div>
</div>

<div class="form-group">
    <label for="phone">Numer telefonu: *</label>
    <input type="tel" 
           id="phone" 
           aria-describedby="phone-help" 
           aria-required="true" 
           pattern="[0-9]{3}-[0-9]{3}-[0-9]{3}" 
           required />
    <div id="phone-help" class="help-text">
        Format: 123-456-789 (cyfry z myślnikami)
    </div>
</div>

<div class="form-group">
    <label for="postal-code">Kod pocztowy: *</label>
    <input type="text" 
           id="postal-code" 
           aria-describedby="postal-help" 
           aria-required="true" 
           pattern="[0-9]{2}-[0-9]{3}" 
           required />
    <div id="postal-help" class="help-text">
        Format: 00-000 (np. 30-001)
    </div>
</div>

Zalety: Jasne instrukcje formatów, przykłady, powiązanie z polami przez aria-describedby.

Grupowanie powiązanych pól

<!-- Grupowanie adresu -->
<fieldset>
    <legend>Adres zamieszkania: *</legend>
    
    <div class="form-group">
        <label for="street">Ulica i numer: *</label>
        <input type="text" 
               id="street" 
               aria-required="true" 
               required />
    </div>
    
    <div class="form-row">
        <div class="form-group">
            <label for="city">Miasto: *</label>
            <input type="text" 
                   id="city" 
                   aria-required="true" 
                   required />
        </div>
        
        <div class="form-group">
            <label for="postal">Kod pocztowy: *</label>
            <input type="text" 
                   id="postal" 
                   aria-describedby="postal-format"
                   aria-required="true" 
                   pattern="[0-9]{2}-[0-9]{3}"
                   required />
            <div id="postal-format" class="help-text">Format: 00-000</div>
        </div>
    </div>

<!-- Grupowanie danych kontaktowych -->
<fieldset>
    <legend>Dane kontaktowe</legend>
    
    <div class="form-group">
        <label for="email">Adres email: *</label>
        <input type="email" 
               id="email" 
               aria-describedby="email-help"
               aria-required="true" 
               required />
        <div id="email-help" class="help-text">
            Wprowadź poprawny adres email (nazwa@domena.pl)
        </div>
    </div>

Zalety: Logiczne grupowanie pól, jasne legendy, różne formaty instrukcji.

Kontrolki wyboru z jasnymi etykietami

<!-- Grupa przycisków radio -->
<fieldset>
    <legend>Preferowany sposób kontaktu: *</legend>
    <div class="radio-group">
        <input type="radio" 
               id="contact-email" 
               name="contact-method" 
               value="email" 
               required />
        <label for="contact-email">Email</label>
    </div>
    <div class="radio-group">
        <input type="radio" 
               id="contact-phone" 
               name="contact-method" 
               value="phone" 
               required />
        <label for="contact-phone">Telefon</label>
    </div>
    <div class="radio-group">
        <input type="radio" 
               id="contact-sms" 
               name="contact-method" 
               value="sms" 
               required />
        <label for="contact-sms">SMS</label>
    </div>
</fieldset>

<!-- Lista wyboru z instrukcją -->
<div class="form-group">
    <label for="country">Kraj: *</label>
    <select id="country" 
            aria-describedby="country-help"
            aria-required="true" 
            required>
        <option value="">-- Wybierz kraj --</option>
        <option value="pl">Polska</option>
        <option value="de">Niemcy</option>
        <option value="fr">Francja</option>

    </select>
    <div id="country-help" class="help-text">
        Wybierz kraj, w którym obecnie mieszkasz
    </div>
</div>

<!-- Pole wyboru z jasną etykietą -->
<div class="form-group">
    <input type="checkbox" 
           id="newsletter" 
           name="newsletter" 
           value="yes" />
    <label for="newsletter">
        Chcę otrzymywać newsletter z informacjami o nowych produktach 
        (maksymalnie raz w tygodniu)
    </label>
</div>

Zalety: Jasne etykiety dla wszystkich opcji, dodatkowe instrukcje gdzie potrzebne.

Formularz z instrukcjami wprowadzającymi

<form>
    <div class="form-instructions">
        <h2>Formularz rejestracji</h2>
        <p>Wypełnij wszystkie pola oznaczone gwiazdką (*). 
           Formularz składa się z trzech sekcji: dane osobowe, 
           adres i preferencje kontaktu.</p>
        <ul>
            <li>Pola obowiązkowe są oznaczone gwiazdką (*)</li>
            <li>Hasło musi mieć minimum 8 znaków</li>
            <li>Data urodzenia w formacie DD/MM/RRRR</li>
            <li>Po wypełnieniu kliknij "Zarejestruj się"</li>
        </ul>
    </div>

    <!-- Sekcja danych osobowych -->
    <fieldset>
        <legend>Dane osobowe</legend>
        
        <div class="form-group">
            <label for="username">Nazwa użytkownika: *</label>
            <input type="text" 
                   id="username" 
                   aria-describedby="username-help"
                   aria-required="true" 
                   required />
            <div id="username-help" class="help-text">
                3-20 znaków, tylko litery, cyfry i podkreślenia
            </div>
        </div>
        
        <div class="form-group">
            <label for="password">Hasło: *</label>
            <input type="password" 
                   id="password" 
                   aria-describedby="password-requirements"
                   aria-required="true" 
                   required />
            <div id="password-requirements" class="help-text">
                Minimum 8 znaków, w tym co najmniej jedna wielka litera, 
                jedna mała litera i jedna cyfra
            </div>
        </div>
    </fieldset>
    
    <div class="form-actions">
        <button type="submit">Zarejestruj się</button>
        <button type="reset">Wyczyść formularz</button>
    </div>
</form>

Zalety: Instrukcje wprowadzające, jasne sekcje, szczegółowe wymagania.

Rodzaje etykiet i instrukcji

Etykiety wizualne

Zastosowanie: Standardowe etykiety widoczne dla wszystkich użytkowników

Implementacja: Element <label> powiązany z polem przez for

Zalety: Uniwersalne, intuicyjne, dostępne

Instrukcje kontekstowe

Zastosowanie: Dodatkowe wskazówki dotyczące formatów lub reguł

Implementacja: Powiązane z polami przez aria-describedby

Zalety: Szczegółowe informacje bez zaśmiecania interfejsu

Etykiety grupowe

Zastosowanie: Opisywanie grup powiązanych pól

Implementacja: <fieldset> z <legend>

Zalety: Logiczne grupowanie, kontekst dla całej sekcji

Instrukcje globalne

Zastosowanie: Ogólne wskazówki dotyczące całego formularza

Implementacja: Na początku formularza lub sekcji

Zalety: Przygotowanie użytkownika, zmniejszenie błędów

Implementacja techniczna

HTML - podstawowe etykiety

<!-- Standardowa etykieta -->
<label for="field-id">Nazwa pola</label>
<input type="text" id="field-id" />

<!-- Etykieta z instrukcją -->
<label for="email">Adres email:</label>
<input type="email" 
       id="email" 
       aria-describedby="email-help" />
<div id="email-help">Format: nazwa@domena.pl</div>

ARIA - dodatkowe powiązania

<!-- Powiązanie z wieloma opisami -->
<input type="password" 
       id="password" 
       aria-labelledby="password-label"
       aria-describedby="password-help password-requirements" />
<label id="password-label">Hasło</label>
<div id="password-help">Wprowadź bezpieczne hasło</div>
<div id="password-requirements">Min. 8 znaków, litery i cyfry</div>

<!-- Aria-label dla przypadków specjalnych -->
<input type="search" 
       aria-label="Wyszukaj produkty" 
       placeholder="Wpisz nazwę produktu..." />

CSS - stylowanie etykiet

/* Podstawowe stylowanie etykiet */
label {
    display: block;
    margin-bottom: 0.25rem;
    font-weight: bold;
    color: #333;
}

/* Oznaczenie wymaganych pól */
.required::after {
    content: " *";
    color: #d32f2f;
    font-weight: bold;
}

/* Instrukcje pomocnicze */
.help-text {
    font-size: 0.875rem;
    color: #666;
    margin-top: 0.25rem;
    line-height: 1.4;
}

/* Legendy dla fieldset */
legend {
    font-weight: bold;
    margin-bottom: 1rem;
    padding: 0 0.5rem;
}

/* Etykiety dla radio i checkbox */
input[type="radio"] + label,
input[type="checkbox"] + label {
    display: inline;
    margin-left: 0.5rem;
    font-weight: normal;
}

Najlepsze praktyki

Tworzenie etykiet

  • Używaj jasnych, opisowych etykiet
  • Umieszczaj etykiety przed polami (nad lub z lewej strony)
  • Unikaj stosowania placeholder jako jedynej etykiety
  • Oznaczaj wymagane pola w sposób dostępny
  • Grupuj powiązane pola w fieldset z legend

Pisanie instrukcji

  • Podawaj konkretne przykłady formatów danych
  • Używaj prostego, zrozumiałego języka
  • Umieszczaj instrukcje blisko pól których dotyczą
  • Zawieraj informacje o ograniczeniach i regułach
  • Testuj instrukcje z prawdziwymi użytkownikami

Dostępność

  • Zawsze wiąż etykiety z polami przez id
  • Używaj aria-describedby dla dodatkowych instrukcji
  • Sprawdzaj dostępność z czytnikami ekranu
  • Testuj nawigację tylko klawiaturą
  • Zapewniaj odpowiedni kontrast kolorów

Testowanie

Metody testowania

  • Test z czytnikiem ekranu - sprawdź czy etykiety są poprawnie odczytywane
  • Test nawigacji klawiaturą - upewnij się o dostępności wszystkich pól
  • Test z wyłączonymi stylami - sprawdź czy etykiety są nadal widoczne
  • Test z użytkownikami - obserwuj rzeczywiste trudności
  • Walidacja HTML - sprawdź poprawność powiązań etykiet

Lista kontrolna

  • Czy każde pole ma odpowiednią etykietę?
  • Czy etykiety są jasne i opisowe?
  • Czy wymagane pola są właściwie oznaczone?
  • Czy instrukcje formatów są dostępne gdy potrzebne?
  • Czy grupy pól mają odpowiednie legendy?
  • Czy etykiety są powiązane z polami programowo?
  • Czy instrukcje są zrozumiałe dla użytkowników?

Narzędzia pomocne w testowaniu

  • NVDA/JAWS - czytniki ekranu do testowania etykiet
  • axe DevTools - automatyczna analiza dostępności formularzy
  • WAVE - walidator etykiet i powiązań
  • Lighthouse - audit dostępności formularzy
  • W3C Markup Validator - sprawdzenie poprawności HTML
