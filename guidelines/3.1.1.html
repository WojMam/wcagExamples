Przejdź do treści głównej

3.1.1 Język strony (A)

Główny ludzki język każdej strony internetowej może być określony programowo.

Opis kryterium

Strona internetowa musi mieć określony główny język, aby technologie wspomagające mogły prawidłowo interpretować i wypowiadać treść. Język strony określa się za pomocą atrybutu lang w elemencie <html>.

Kto odnosi korzyść

  • Użytkownicy czytników ekranu - poprawna wymowa tekstu
  • Osoby z dysleksją - wsparcie syntez mowy
  • Użytkownicy tłumaczy automatycznych - rozpoznanie języka źródłowego
  • Osoby uczące się języków - pomoc w rozumieniu treści
  • Wyszukiwarki - lepsze indeksowanie dla różnych języków

Wymagania

  • Element <html> musi mieć atrybut lang
  • Wartość atrybutu musi być prawidłowym kodem języka (BCP 47)
  • Kod języka musi odpowiadać głównemu językowi treści strony
  • Atrybut musi być obecny na każdej stronie

Definicje

Główny język strony
Język, w którym napisana jest większość treści strony. Jeśli treść jest w równym stopniu w dwóch językach, można wybrać jeden z nich.
BCP 47
Standard określający kody języków (Best Current Practice 47). Przykłady: "pl" (polski), "en" (angielski), "en-US" (angielski amerykański).
Technologie wspomagające
Oprogramowanie lub urządzenia pomagające osobom z niepełnosprawnościami, takie jak czytniki ekranu, syntezatory mowy.

Przykłady

❌ Nieprawidłowe implementacje

Brak atrybutu lang

<!DOCTYPE html>
<html> <!-- Brak atrybutu lang -->
<head>
    <title>Moja strona internetowa</title>
</head>
<body>
    <h1>Witamy na naszej stronie</h1>
    <p>Oferujemy najlepsze usługi w branży.</p>
</body>
</html>

Problem: Technologie wspomagające nie wiedzą, w jakim języku przeczytać treść.

Nieprawidłowy kod języka

<!DOCTYPE html>
<html lang="polish"> <!-- Nieprawidłowy format -->
<head>
    <title>Strona firmowa</title>
</head>
<body>
    <h1>O naszej firmie</h1>
</body>
</html>

Problem: "polish" nie jest prawidłowym kodem języka według BCP 47.

Błędny język dla treści

<!DOCTYPE html>
<html lang="en"> <!-- Angielski, ale treść po polsku -->
<head>
    <title>Nasza oferta</title>
</head>
<body>
    <h1>Zapraszamy do zapoznania się z ofertą</h1>
    <p>Jesteśmy liderem na polskim rynku.</p>
</body>
</html>

Problem: Czytnik ekranu będzie próbował przeczytać polską treść z angielską wymową.

✅ Prawidłowe implementacje

Strona w języku polskim

<!DOCTYPE html>
<html lang="pl">
<head>
    <meta charset="UTF-8">
    <title>Przykładowa strona firmowa</title>
</head>
<body>
    <h1>Witamy w naszej firmie</h1>
    <p>Specjalizujemy się w tworzeniu dostępnych stron internetowych.</p>
</body>
</html>

Dlaczego działa: Jasno określony język polski (pl) odpowiada treści strony.

Strona w języku angielskim

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Our Company Website</title>
</head>
<body>
    <h1>Welcome to Our Company</h1>
    <p>We specialize in creating accessible web solutions.</p>
</body>
</html>

Dlaczego działa: Kod "en" prawidłowo określa język angielski.

Specyficzna odmiana języka

<!DOCTYPE html>
<html lang="en-US">
<head>
    <meta charset="UTF-8">
    <title>American English Website</title>
</head>
<body>
    <h1>Color and Flavor</h1>
    <p>We use American spelling conventions.</p>
</body>
</html>

Dlaczego działa: "en-US" określa amerykańską odmianę języka angielskiego.

Strona wielojęzyczna z przeważającym językiem

<!DOCTYPE html>
<html lang="pl">
<head>
    <meta charset="UTF-8">
    <title>Firma międzynarodowa</title>
</head>
<body>
    <h1>Nasza międzynarodowa firma</h1>
    <p>Działamy na rynku polskim od 20 lat. Oferujemy usługi 
       również w języku angielskim.</p>
    
    <!-- Fragment w innym języku oznaczony osobno -->
    <p>Our mission: <span lang="en">Innovation for everyone</span>.</p>
</body>
</html>

Dlaczego działa: Główny język to polski, fragmenty angielskie są oznaczone osobno.

Testowanie

Testy automatyczne

Użyj następującego kodu JavaScript do sprawdzenia obecności atrybutu lang:

// Test obecności atrybutu lang
function testPageLanguage() {
    const htmlElement = document.documentElement;
    const langAttr = htmlElement.getAttribute('lang');
    
    const results = {
        hasLangAttribute: !!langAttr,
        languageCode: langAttr,
        isValidCode: false,
        issues: []
    };
    
    if (!langAttr) {
        results.issues.push('Brak atrybutu lang w elemencie html');
        return results;
    }
    
    // Podstawowa walidacja kodu języka (BCP 47)
    const langPattern = /^[a-z]{2,3}(-[A-Z]{2})?(-[a-z]{4})?(-[A-Z]{2})?$/;
    results.isValidCode = langPattern.test(langAttr);
    
    if (!results.isValidCode) {
        results.issues.push('Nieprawidłowy format kodu języka');
    }
    
    return results;
}

// Uruchom test
const testResult = testPageLanguage();
console.log('Test języka strony:', testResult);

if (testResult.issues.length === 0) {
    console.log('✅ Test przeszedł pomyślnie');
} else {
    console.warn('❌ Znaleziono problemy:', testResult.issues);
}

Testy manualne

  • Sprawdź kod źródłowy - czy element <html> ma atrybut lang
  • Zweryfikuj kod języka - czy odpowiada głównej treści strony
  • Test z czytnikiem ekranu - czy wymowa jest prawidłowa
  • Walidator HTML - sprawdź poprawność kodu

Narzędzia

  • WAVE Web Accessibility Evaluator - wykrywa brak atrybutu lang
  • axe DevTools - sprawdza obecność i poprawność języka
  • W3C Markup Validator - walidacja HTML
  • Lighthouse - audyt dostępności

Lista kontrolna

  • ☐ Element <html> zawiera atrybut lang
  • ☐ Kod języka jest w formacie BCP 47
  • ☐ Kod odpowiada głównemu językowi treści
  • ☐ Atrybut jest obecny na wszystkich stronach
  • ☐ Czytnik ekranu prawidłowo interpretuje język
  • ☐ Walidator HTML nie zgłasza błędów

Systemy CMS

WordPress

<!-- W pliku functions.php -->
function custom_html_lang_attribute() {
    return 'pl'; // lub inny kod języka
}
add_filter('language_attributes', 'custom_html_lang_attribute');

<!-- W szablonie theme -->
<html <?php language_attributes(); ?>>

Drupal

<!-- W konfiguracji witryny -->
// admin/config/regional/language
// Ustaw domyślny język witryny

<!-- W szablonie Twig -->
<html{{ html_attributes.setAttribute('lang', language.getId()) }}>

Frameworki JavaScript

React

// W komponencie głównym lub App.js
useEffect(() => {
    document.documentElement.lang = 'pl';
}, []);

// Lub w index.html
<html lang="pl">

Vue.js

// W main.js lub komponencie głównym
mounted() {
    document.documentElement.setAttribute('lang', 'pl');
}

// Lub dynamicznie
computed: {
    currentLanguage() {
        return this.$i18n.locale;
    }
},
watch: {
    currentLanguage(newLang) {
        document.documentElement.lang = newLang;
    }
}

Angular

// W app.component.ts
import { DOCUMENT } from '@angular/common';
import { Inject, Component } from '@angular/core';

@Component({...})
export class AppComponent {
    constructor(@Inject(DOCUMENT) private document: Document) {
        this.document.documentElement.lang = 'pl';
    }
}

Dynamiczna zmiana języka

Dla aplikacji wielojęzycznych, które pozwalają użytkownikowi zmieniać język:

// Funkcja zmiany języka
function changePageLanguage(newLanguage) {
    // Zmień atrybut lang
    document.documentElement.lang = newLanguage;
    
    // Opcjonalnie: zmień kierunek tekstu dla języków RTL
    if (['ar', 'he', 'fa'].includes(newLanguage)) {
        document.documentElement.dir = 'rtl';
    } else {
        document.documentElement.dir = 'ltr';
    }
    
    // Powiadom technologie wspomagające o zmianie
    announceLanguageChange(newLanguage);
}

// Funkcja powiadamiania o zmianie języka
function announceLanguageChange(language) {
    const announcement = document.createElement('div');
    announcement.setAttribute('aria-live', 'polite');
    announcement.setAttribute('aria-atomic', 'true');
    announcement.style.position = 'absolute';
    announcement.style.left = '-10000px';
    announcement.textContent = `Język strony zmieniony na ${language}`;
    
    document.body.appendChild(announcement);
    
    setTimeout(() => {
        document.body.removeChild(announcement);
    }, 1000);
}

// Przykład użycia
document.getElementById('lang-selector').addEventListener('change', function(e) {
    changePageLanguage(e.target.value);
});

Częste błędy

❌ Używanie niepoprawnych kodów

<!-- Nieprawidłowe -->
<html lang="polish">
<html lang="POL">
<html lang="PL">

<!-- Prawidłowe -->
<html lang="pl">

❌ Umieszczanie lang w niewłaściwym miejscu

<!-- Nieprawidłowe -->
<html>
<head>
    <meta lang="pl">
</head>
<body lang="pl">

<!-- Prawidłowe -->
<html lang="pl">

❌ Brak zgodności treści z językiem

<!-- Nieprawidłowe -->
<html lang="en">
<body>
    <h1>Witamy na stronie</h1> <!-- Polska treść z angielskim lang -->
</body>

<!-- Prawidłowe -->
<html lang="pl">
<body>
    <h1>Witamy na stronie</h1>
</body>
