Przejdź do treści głównej

3.2.2 Podczas wpisywania (A)

Zmiana ustawień komponentu interfejsu użytkownika nie powoduje automatycznie zmiany kontekstu, chyba że użytkownik został poinformowany o tym zachowaniu przed użyciem komponentu.

Opis kryterium

To kryterium wymaga, aby wprowadzanie danych lub zmiana ustawień w elementach interfejsu (takich jak pola formularza, listy rozwijane, przyciski radio) nie powodowało automatycznych zmian kontekstu bez uprzedzenia użytkownika. Oznacza to, że:

  • Wybór opcji z listy rozwijanej nie powinien automatycznie wysyłać formularza
  • Zmiana wartości pola nie powinna przekierowywać na inną stronę
  • Zaznaczenie checkboxa nie może otwierać nowego okna
  • Wprowadzanie tekstu nie powinno automatycznie przesyłać danych

Wyjątek stanowią sytuacje, gdzie użytkownik został wcześniej poinformowany o automatycznym zachowaniu systemu.

Kto odnosi korzyść

  • Osoby niewidome - używające czytników ekranu, które potrzebują przewidywalności
  • Osoby z ograniczeniami poznawczymi - które mogą być zdezorientowane niespodziewanymi zmianami
  • Osoby z ograniczeniami motorycznymi - które mogą przypadkowo zmienić wartości
  • Użytkownicy z ADHD - którzy są wrażliwi na rozpraszające zmiany
  • Początkujący użytkownicy - którzy uczą się obsługi interfejsu
  • Użytkownicy urządzeń mobilnych - którzy mogą przypadkowo dotknąć elementy
  • Starsi użytkownicy - którzy preferują przewidywalny interfejs

Definicje

Zmiana kontekstu (Change of context)
Znaczące zmiany w treści strony internetowej, które mogą dezorientować użytkowników, jeśli nie są oni świadomi, że zmiana może wystąpić. Obejmuje to zmiany:
  • Programu użytkownika (przeglądarki)
  • Okna wyświetlania (viewport)
  • Fokusa
  • Treści, która zmienia znaczenie strony
Komponent interfejsu użytkownika
Część treści, która jest postrzegana przez użytkowników jako pojedynczy element kontrolny dla określonej funkcji (np. przycisk, pole tekstowe, lista rozwijana).
Wprowadzanie danych (Input)
Dane podawane do systemu przez użytkownika poprzez działania takie jak: pisanie na klawiaturze, zaznaczanie opcji, klikanie przycisków.
Ustawienia komponentu
Konfiguracja lub stan elementu interfejsu, taki jak zaznaczenie checkboxa, wybrana opcja z listy, wprowadzony tekst.

<script>
function confirmLanguageChange(select) {
    const confirmed = confirm(
        'Czy chcesz zmienić język strony? Zostaniesz przekierowany na nową stronę.'
    );
    
    if (confirmed) {
        window.location.href = '/' + select.value + '/';
    } else {
        // Przywróć poprzednią wartość
        select.selectedIndex = 0;
    }
}
</script>

Dlaczego działa: Użytkownik jest ostrzeżony i musi potwierdzić zmianę.

Wyszukiwanie z podpowiedziami

<!-- Prawidłowe - podpowiedzi bez zmiany kontekstu -->
<form method="get" action="/search">
    <label for="search">Wyszukaj produkt:</label>
    <div class="search-container">
        <input type="text" id="search" name="q"
               aria-describedby="search-suggestions"
               oninput="showSuggestions(this.value)">
        
        <div id="search-suggestions" class="suggestions" role="listbox">
            <!-- Podpowiedzi będą tu dodawane dynamicznie -->
        </div>
    </div>
    
    <button type="submit">Szukaj</button>
</form>

<script>
function showSuggestions(query) {
    const suggestionsDiv = document.getElementById('search-suggestions');
    
    if (query.length < 2) {
        suggestionsDiv.innerHTML = '';
        return;
    }
    
    // Symulacja podpowiedzi
    const suggestions = ['laptop', 'lampa', 'lalka'].filter(item => 
        item.includes(query.toLowerCase())
    );
    
    suggestionsDiv.innerHTML = suggestions.map(suggestion => 
        `<div class="suggestion" onclick="selectSuggestion('${suggestion}')">
            ${suggestion}
        </div>`
    ).join('');
}

function selectSuggestion(text) {
    document.getElementById('search').value = text;
    document.getElementById('search-suggestions').innerHTML = '';
}
</script>

Dlaczego działa: Podpowiedzi pojawiają się lokalnie, nie zmieniając głównej treści.

Wytyczne implementacyjne

Co MOŻNA robić przy wprowadzaniu danych:

  • Pokazywanie podpowiedzi - sugestie, autocompletowanie
  • Walidacja w czasie rzeczywistym - sprawdzanie formatu bez przesyłania
  • Lokalne filtrowanie - ukrywanie/pokazywanie opcji na tej samej stronie
  • Aktualizacja powiązanych pól - np. lista miast po wyborze województwa
  • Pokazywanie dodatkowych opcji - rozwijanie sekcji formularza
  • Przygotowanie danych w tle - ładowanie bez zmiany interfejsu

Czego NIE MOŻNA robić przy wprowadzaniu danych:

  • Automatyczne wysyłanie formularzy
  • Przekierowywanie na inne strony
  • Otwieranie nowych okien
  • Przenoszenie fokusa
  • Zmiana głównej treści strony
  • Usuwanie lub ukrywanie aktywnych elementów

// ✅ Prawidłowe - podpowiedzi bez zmiany kontekstu
function showSuggestions(inputElement) {
    const query = inputElement.value;
    const suggestionsContainer = document.getElementById('suggestions');
    
    if (query.length > 2) {
        fetch(`/api/suggestions?q=${encodeURIComponent(query)}`)
            .then(response => response.json())
            .then(suggestions => {
                suggestionsContainer.innerHTML = suggestions.map(item => 
                    `
${item}
`
                ).join('');
            });
    } else {
        suggestionsContainer.innerHTML = '';
    }
}

// ✅ Prawidłowe - walidacja bez przesyłania
function validateField(field) {
    const errorDiv = document.getElementById(field.id + '-error');
    let isValid = true;
    let message = '';
    
    // Wykonaj walidację
    if (field.type === 'email' && !isValidEmail(field.value)) {
        isValid = false;
        message = 'Nieprawidłowy format email';
    }
    
    // Pokaż/ukryj komunikat błędu lokalnie
    if (errorDiv) {
        errorDiv.textContent = message;
        errorDiv.style.display = isValid ? 'none' : 'block';
    }
    
    return isValid;
}

// ✅ Prawidłowe - kontrolowane przesłanie
function handleFormSubmit(event) {
    event.preventDefault(); // Zatrzymaj domyślne przesłanie
    
    const form = event.target;
    const isValid = validateForm(form);
    
    if (isValid) {
        // Użytkownik kontroluje kiedy formularz jest wysłany
        if (confirm('Czy chcesz wysłać formularz?')) {
            form.submit();
        }
    }
}

Testowanie kryterium 3.2.2

Testy manualne

  • Test każdego elementu formularza - sprawdź czy zmiana wartości nie powoduje niespodziewanych akcji
  • Test nawigacji klawiaturą - czy przechodzenie przez pola nie powoduje automatycznych zmian
  • Test z czytnikami ekranu - czy zmiany są komunikowane odpowiednio
  • Test różnych przeglądarek - weryfikuj spójność zachowań
  • Test na urządzeniach mobilnych - sprawdź przypadkowe dotknięcia

Testy automatyczne

// Test wykrywania automatycznych akcji
function auditInputHandlers() {
    const issues = [];
    const inputElements = document.querySelectorAll('input, select, textarea');
    
    inputElements.forEach(element => {
        // Sprawdź onchange
        const changeHandler = element.getAttribute('onchange');
        if (changeHandler) {
            const problematicPatterns = [
                { pattern: /\.submit\(\)/, issue: 'Automatyczne wysłanie formularza' },
                { pattern: /location\.href/, issue: 'Przekierowanie' },
                { pattern: /window\.open/, issue: 'Otwieranie nowego okna' },
                { pattern: /\.focus\(\)/, issue: 'Przeniesienie fokusa' }
            ];
            
            problematicPatterns.forEach(({ pattern, issue }) => {
                if (pattern.test(changeHandler)) {
                    issues.push({
                        element: element,
                        selector: getElementSelector(element),
                        issue: issue,
                        handler: changeHandler,
                        event: 'onchange'
                    });
                }
            });
        }
        
        // Sprawdź oninput
        const inputHandler = element.getAttribute('oninput');
        if (inputHandler) {
            if (/\.submit\(\)|location\.href/.test(inputHandler)) {
                issues.push({
                    element: element,
                    selector: getElementSelector(element),
                    issue: 'Automatyczna akcja podczas pisania',
                    handler: inputHandler,
                    event: 'oninput'
                });
            }
        }
    });
    
    return issues;
}

// Test formularzy bez przycisków submit
function auditFormsWithoutSubmit() {
    const issues = [];
    const forms = document.querySelectorAll('form');
    
    forms.forEach(form => {
        const submitButtons = form.querySelectorAll(
            'button[type="submit"], input[type="submit"]'
        );
        
        if (submitButtons.length === 0) {
            // Sprawdź czy jest jakiś inny sposób wysłania
            const hasAutoSubmit = Array.from(form.elements).some(element => {
                const changeHandler = element.getAttribute('onchange');
                return changeHandler && changeHandler.includes('submit');
            });
            
            if (hasAutoSubmit) {
                issues.push({
                    form: form,
                    issue: 'Formularz bez przycisku submit używa automatycznego wysyłania',
                    severity: 'high'
                });
            }
        }
    });
    
    return issues;
}

// Test symulujący wprowadzanie danych
function simulateInputChanges() {
    const results = [];
    const inputElements = document.querySelectorAll('input, select');
    
    inputElements.forEach((element, index) => {
        const initialURL = window.location.href;
        const initialWindowCount = window.length;
        const initialFocus = document.activeElement;
        
        try {
            // Symuluj zmianę wartości
            if (element.type === 'checkbox') {
                element.checked = !element.checked;
            } else if (element.tagName === 'SELECT') {
                if (element.options.length > 1) {
                    element.selectedIndex = 1;
                }
            } else {
                element.value = 'test input';
            }
            
            // Wywołaj zdarzenie change
            const changeEvent = new Event('change', { bubbles: true });
            element.dispatchEvent(changeEvent);
            
            // Sprawdź czy nastąpiły niespodziewane zmiany
            setTimeout(() => {
                const urlChanged = window.location.href !== initialURL;
                const windowCountChanged = window.length !== initialWindowCount;
                const focusChanged = document.activeElement !== initialFocus && 
                                  document.activeElement !== element;
                
                if (urlChanged || windowCountChanged || focusChanged) {
                    results.push({
                        element: element,
                        index: index,
                        urlChanged: urlChanged,
                        windowCountChanged: windowCountChanged,
                        focusChanged: focusChanged
                    });
                }
            }, 100);
            
        } catch (error) {
            console.warn(`Nie można przetestować elementu ${index}:`, error);
        }
    });
    
    return results;
}

Częste błędy

❌ Automatyczne wysłanie formularza

Problem: Formularz wysyła się przy zmianie wartości w polu

<select onchange="this.form.submit()">

Rozwiązanie: Dodaj przycisk submit

<select name="category"></select>
<button type="submit">Zastosuj filtr</button>

❌ Przekierowanie przy wyborze opcji

Problem: Lista język automatycznie przekierowuje

<select onchange="location.href='/'+this.value">

Rozwiązanie: Ostrzeż użytkownika i daj możliwość anulowania

<select onchange="confirmLanguageChange(this)">
<script>
function confirmLanguageChange(select) {
    if (confirm('Zmienić język strony?')) {
        location.href = '/' + select.value;
    } else {
        select.selectedIndex = 0;
    }
}
</script>

❌ Utrata danych przy błędach

Problem: Automatyczne wysłanie nie pozwala na poprawkę błędów

Rozwiązanie: Waliduj lokalnie przed wysłaniem

<form onsubmit="return validateBeforeSubmit(this)">
    <input required oninput="validateField(this)">
    <button type="submit">Wyślij</button>
</form>
