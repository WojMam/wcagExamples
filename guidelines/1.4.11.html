1.4.11 Kontrast elementów nietekstowych

Poziom AA

Oficjalna dokumentacja W3C

Opis kryterium

Prezentacja wizualna następujących elementów ma współczynnik kontrastu wynoszący co najmniej 3:1 w stosunku do sąsiadujących kolorów:

  • Komponenty interfejsu użytkownika: Informacje wizualne wymagane do identyfikacji komponentów interfejsu użytkownika i stanów, z wyjątkiem nieaktywnych komponentów lub gdy wygląd komponentu jest określony przez przeglądarkę użytkownika i nie jest modyfikowany przez autora;
  • Obiekty graficzne: Części grafiki wymagane do zrozumienia treści, z wyjątkiem sytuacji, gdy konkretna prezentacja grafiki jest niezbędna do przekazania informacji.

Intencja

Celem tego kryterium jest zapewnienie, że wszystkie ważne elementy nietekstowe (takie jak ikony, przyciski, pola formularzy, wykresy) są widoczne dla osób z obniżoną zdolnością widzenia, bez konieczności polegania na dodatkowych technologiach wspomagających.

Odpowiedni kontrast elementów nietekstowych jest kluczowy dla użytkowników z różnymi typami zaburzeń widzenia, w tym daltonizmem, osłabioną percepcją kontrastu lub innymi wadami wzroku. Dotyczy to zarówno komponentów interfejsu (przyciski, pola formularzy, przełączniki) jak i elementów graficznych przekazujących informacje (wykresy, diagramy, ikony informacyjne).

Przykłady

Zły przykład - Przyciski o niskim kontraście

Przyciski z obramowaniem o niskim kontraście w stosunku do tła:

<style>
.button-container {
    background-color: #f8f8f8;
    padding: 20px;
    border-radius: 5px;
}

.outline-button {
    padding: 10px 20px;
    background: transparent;
    border: 2px solid #e0e0e0; /* Bardzo jasny szary, słaby kontrast z tłem #f8f8f8 */
    border-radius: 4px;
    color: #333;
    font-weight: bold;
    cursor: pointer;
    margin-right: 10px;
}
</style>

<div class="button-container">
    <button class="outline-button">Anuluj</button>
    <button class="outline-button">Zapisz</button>
</div>
Podgląd:

Problem: Obramowanie przycisków (#e0e0e0) ma zbyt niski kontrast w stosunku do tła (#f8f8f8). Współczynnik kontrastu wynosi tylko około 1.1:1, znacznie poniżej wymaganego 3:1.

Problemy w tym przykładzie:

  • Obramowanie przycisków jest bardzo słabo widoczne ze względu na niski kontrast
  • Użytkownicy z osłabioną percepcją kontrastu mogą mieć trudności z rozpoznaniem granic przycisków
  • Brakuje wyraźnego odróżnienia elementów interaktywnych od tła
  • Granica między przyciskami a tłem jest prawie niewidoczna przy niektórych ustawieniach ekranu lub w różnych warunkach oświetleniowych

Dobry przykład - Przyciski o odpowiednim kontraście

Przyciski z obramowaniem o wystarczającym kontraście:

<style>
.button-container {
    background-color: #f8f8f8;
    padding: 20px;
    border-radius: 5px;
}

.outline-button {
    padding: 10px 20px;
    background: transparent;
    border: 2px solid #767676; /* Ciemniejszy szary, kontrast 4.5:1 z tłem #f8f8f8 */
    border-radius: 4px;
    color: #333;
    font-weight: bold;
    cursor: pointer;
    margin-right: 10px;
}

.filled-button {
    padding: 10px 20px;
    background-color: #0056b3; /* Niebieski z kontrastem >3:1 z tłem */
    border: none;
    border-radius: 4px;
    color: white;
    font-weight: bold;
    cursor: pointer;
}
</style>

<div class="button-container">
    <button class="outline-button">Anuluj</button>
    <button class="filled-button">Zapisz</button>
</div>
Podgląd:

Rozwiązanie: Obramowanie pierwszego przycisku (#767676) ma odpowiedni kontrast z tłem (#f8f8f8). Drugi przycisk używa wypełnienia o wysokim kontraście, co dodatkowo ułatwia jego rozpoznanie.

Zalety tego rozwiązania:

  • Obramowanie przycisku "Anuluj" ma współczynnik kontrastu około 4.5:1 w stosunku do tła, co przekracza wymagane minimum 3:1
  • Przycisk "Zapisz" używa koloru wypełnienia (#0056b3), który ma wysoki kontrast z tłem
  • Elementy interaktywne są wyraźnie odróżnione od tła, co ułatwia ich identyfikację
  • Przyciski pozostają widoczne nawet dla użytkowników z osłabioną percepcją kontrastu

Zły przykład - Wykres z elementami o niskim kontraście

Wykres kołowy z segmentami o zbyt małym kontraście:

<style>
.chart-container {
    width: 300px;
    height: 300px;
    position: relative;
    margin: 0 auto;
}

.pie-chart {
    width: 100%;
    height: 100%;
    border-radius: 50%;
    background-image: conic-gradient(
        #e6e6e6 0% 25%, /* Bardzo jasny szary */
        #d9d9d9 25% 55%, /* Jasny szary - niski kontrast z poprzednim */
        #cccccc 55% 80%, /* Inny jasny szary - słaby kontrast */
        #f2f2f2 80% 100% /* Jeszcze jaśniejszy szary */
    );
}

.chart-legend {
    margin-top: 20px;
}

.legend-item {
    display: flex;
    align-items: center;
    margin-bottom: 5px;
}

.legend-color {
    width: 20px;
    height: 20px;
    margin-right: 10px;
}
</style>

<div class="chart-container">
    <div class="pie-chart"></div>
</div>

<div class="chart-legend">
    <div class="legend-item">
        <div class="legend-color" style="background-color: #e6e6e6;"></div>
        <span>Produkt A (25%)</span>
    </div>
    <div class="legend-item">
        <div class="legend-color" style="background-color: #d9d9d9;"></div>
        <span>Produkt B (30%)</span>
    </div>
    <div class="legend-item">
        <div class="legend-color" style="background-color: #cccccc;"></div>
        <span>Produkt C (25%)</span>
    </div>
    <div class="legend-item">
        <div class="legend-color" style="background-color: #f2f2f2;"></div>

    </div>
</div>
Podgląd:
Produkt A (25%)
Produkt B (30%)
Produkt C (25%)
Produkt D (20%)

Problem: Segmenty wykresu mają zbyt podobne kolory i niski kontrast między sobą. Różnice między segmentami są trudne do zauważenia, zwłaszcza dla osób z daltonizmem lub osłabioną percepcją kontrastu.

Problemy w tym przykładzie:

  • Kolory segmentów są zbyt podobne do siebie (wszystkie są odcieniami jasnego szarego)
  • Brak wystarczającego kontrastu między sąsiadującymi segmentami wykresu
  • Użytkownicy z daltonizmem lub osłabioną percepcją kontrastu mogą mieć trudności z rozróżnieniem segmentów
  • Wykres nie przekazuje skutecznie informacji, ponieważ segmenty są trudne do rozróżnienia

Dobry przykład - Dostępny wykres z odpowiednim kontrastem

Wykres kołowy z wyraźnie zróżnicowanymi segmentami i dodatkowym oznaczeniem:

<style>
.chart-container {
    width: 300px;
    height: 300px;
    position: relative;
    margin: 0 auto;
}

.pie-chart {
    width: 100%;
    height: 100%;
    border-radius: 50%;
    background-image: conic-gradient(
        #0056b3 0% 25%, /* Ciemny niebieski */
        #e63946 25% 55%, /* Czerwony */
        #2a9d8f 55% 80%, /* Turkusowy */
        #f4a261 80% 100% /* Pomarańczowy */
    );
}

.segment-pattern {
    position: absolute;
    width: 100%;
    height: 100%;
    border-radius: 50%;
    mix-blend-mode: overlay;
}

.chart-legend {
    margin-top: 20px;
}

.legend-item {
    display: flex;
    align-items: center;
    margin-bottom: 8px;
}

.legend-color {
    width: 20px;
    height: 20px;
    margin-right: 10px;
    border: 1px solid #333;
}

    position: relative;
}

    content: "";
    position: absolute;
    top: 0;
    left: 0;
    width: 100%;

</style>

<div class="chart-container">
    <div class="pie-chart"></div>
    <!-- Dodatkowe wzory można by dodać tutaj jako nakładki -->

    <div class="legend-item">
        <div class="legend-color" style="background-color: #0056b3;"></div>
        <span>Produkt A (25%)</span>
    </div>
    <div class="legend-item">
        <div class="legend-color" style="background-color: #e63946;"></div>

    </div>

    </div>

        <div class="legend-color" style="background-color: #f4a261;"></div>

    </div>

Produkt A (25%)
Produkt B (30%)

Rozwiązanie: Wykres używa kolorów o wysokim kontraście między sąsiadującymi segmentami. Dodatkowe obramowanie wokół próbek kolorów w legendzie poprawia ich widoczność.

Zalety tego rozwiązania:

  • Segmenty wykresu mają wyraźnie kontrastujące kolory, które łatwo odróżnić
  • Kolory sąsiadujących segmentów mają współczynnik kontrastu powyżej 3:1
  • Dodatkowe obramowanie wokół elementów legendy poprawia ich widoczność
  • Wykres skutecznie przekazuje informacje nawet użytkownikom z zaburzeniami widzenia kolorów
  • Dla jeszcze lepszej dostępności można dodać różne wzory/tekstury do segmentów (np. kropki, paski) jako dodatkowe rozróżnienie

Pamiętaj, że oprócz kontrastu kolorów, dobrą praktyką jest używanie również innych metod rozróżniania (np. wzory, etykiety), aby zapewnić dostępność dla osób z daltonizmem.

Wskazówki i techniki

  • Testuj kontrast - Używaj narzędzi do sprawdzania kontrastu elementów nietekstowych, np. WebAIM Contrast Checker lub Tanaguru Contrast Finder
  • Wyraźne granice komponentów - Zapewnij wyraźne obramowanie lub wypełnienie komponentów interfejsu w kontrastowym kolorze
  • Fokus i hover - Dodawaj wyraźne wskaźniki stanu :focus i :hover z kontrastem co najmniej 3:1
  • Nie polegaj tylko na kolorze - Oprócz kontrastu kolorów, używaj innych metod rozróżnienia elementów (wzory, ikony, kształty)
  • Grafiki i wykresy - Zapewnij odpowiedni kontrast kluczowych elementów wykresów i grafik
  • Testuj w skali szarości - Sprawdzaj, czy elementy są rozróżnialne po konwersji do skali szarości
  • Ikony i przyciski - Upewnij się, że ikony i przyciski mają odpowiedni kontrast z tłem
  • Pola formularza - Granice pól formularza powinny być wyraźnie widoczne poprzez odpowiedni kontrast z tłem
  • Kontrola użytkownika - Rozważ dodanie opcji zwiększania kontrastu dla użytkowników z wadami wzroku
  • Testuj z różnymi typami daltonizmu - Używaj symulatorów daltonizmu, aby sprawdzić, czy elementy są rozróżnialne dla osób z różnymi zaburzeniami widzenia kolorów
