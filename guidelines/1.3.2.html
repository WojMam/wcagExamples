1.3.2 Zrozumiała kolejność

Poziom A

Oficjalna dokumentacja W3C

Opis kryterium

Jeśli kolejność prezentacji treści ma znaczenie dla jej zrozumienia, to właściwa kolejność odczytu może być określona programowo.

Intencja

Celem tego kryterium jest umożliwienie technologiom wspomagającym, takim jak czytniki ekranu, prezentacji treści w logicznej kolejności, która zachowuje znaczenie. Kolejność treści w kodzie źródłowym powinna odzwierciedlać logiczną kolejność odczytu. Ma to zapewnić, że użytkownicy korzystający z alternatywnych sposobów prezentacji (np. audio) otrzymają informacje w takim samym, sensownym porządku jak użytkownicy widzący.

Przykłady

Dobry przykład - logiczna kolejność w kodzie

Struktura HTML, gdzie kolejność elementów w kodzie odpowiada logicznej kolejności odczytu: 

<article>
    <h2>Jak upiec ciasto czekoladowe</h2>
    
    <section>
        <h3>Składniki</h3>
        <ul>
            <li>200g mąki</li>
            <li>150g cukru</li>
            <li>100g masła</li>
            <li>2 jajka</li>
            <li>50g kakao</li>
        </ul>
    </section>
    
    <section>
        <h3>Instrukcja</h3>
        <ol>
            <li>Wymieszaj składniki suche.</li>
            <li>Dodaj masło i jajka, wymieszaj.</li>
            <li>Piecz w 180°C przez 30 minut.</li>
        </ol>
    </section>
</article>
Podgląd:

Jak upiec ciasto czekoladowe

Składniki

  • 200g mąki
  • 150g cukru
  • 100g masła
  • 2 jajka
  • 50g kakao

Instrukcja

  1. Wymieszaj składniki suche.
  2. Dodaj masło i jajka, wymieszaj.
  3. Piecz w 180°C przez 30 minut.

W tym przykładzie, kolejność elementów w kodzie HTML odzwierciedla logiczną kolejność informacji: nagłówek, składniki, a następnie instrukcje. Czytnik ekranu odczyta informacje w tej samej kolejności, co pozwoli użytkownikowi zrozumieć przepis.

Zły przykład - kolejność zaburzona przez pozycjonowanie CSS

Kod HTML, gdzie wizualna kolejność została zmieniona za pomocą CSS, ale kolejność w kodzie źródłowym pozostaje nieprawidłowa:

<style>
    .recipe-container {
        display: grid;
        grid-template-areas:
            "title title"
            "instructions ingredients";
    }
    .title { grid-area: title; }
    .ingredients { grid-area: ingredients; }
    .instructions { grid-area: instructions; }
</style>

<div class="recipe-container">
    <h2 class="title">Jak upiec ciasto czekoladowe</h2>
    
    <section class="instructions">
        <h3>Instrukcja</h3>
        <ol>
            <li>Wymieszaj składniki suche.</li>
            <li>Dodaj masło i jajka, wymieszaj.</li>
            <li>Piecz w 180°C przez 30 minut.</li>
        </ol>
    </section>
    
    <section class="ingredients">
        <h3>Składniki</h3>
        <ul>
            <li>200g mąki</li>
            <li>150g cukru</li>
            <li>100g masła</li>
            <li>2 jajka</li>
            <li>50g kakao</li>
        </ul>
    </section>
</div>

Jak upiec ciasto czekoladowe

Instrukcja

  1. Wymieszaj składniki suche.
  2. Dodaj masło i jajka, wymieszaj.
  3. Piecz w 180°C przez 30 minut.

Składniki

  • 200g mąki
  • 150g cukru
  • 100g masła
  • 2 jajka

W tym przykładzie, w kodzie HTML instrukcje pojawiają się przed składnikami, ale CSS zmienia ich wizualną kolejność. Czytnik ekranu odczyta instrukcje przed składnikami, co jest nielogiczne i utrudnia zrozumienie przepisu. Prawidłowe podejście to dostosowanie kolejności elementów w kodzie HTML do logicznego porządku, a następnie użycie CSS tylko do stylizacji, a nie do zmiany kolejności treści.

Dobry przykład - wielokolumnowy układ z zachowaniem logicznej kolejności

Strona z wielokolumnowym układem, gdzie kolejność w kodzie zapewnia logiczny odczyt:

<div class="article-wrapper">
    <header>
        <h1>Dostępność cyfrowa</h1>
        <p class="intro">Wprowadzenie do zasad dostępności treści internetowych...</p>
    </header>
    
    <main class="columns-layout">
        <section>
            <h2>Dlaczego dostępność jest ważna?</h2>
            <p>Dostępność cyfrowa zapewnia, że treści internetowe są dostępne dla wszystkich użytkowników...</p>
            
            <h2>Kluczowe zasady dostępności</h2>
            <p>Zasady dostępności opierają się na czterech głównych filarach...</p>
        </section>
        
        <aside>
            <h3>Przydatne linki</h3>
            <ul>
                <li><a href="#">Dokumentacja WCAG</a></li>
                <li><a href="#">Narzędzia do testowania</a></li>
            </ul>
        </aside>
    </main>
</div>

<style>
    .columns-layout {
        display: flex;
    }
    section {
        flex: 3;
        padding-right: 20px;
    }
    aside {
        flex: 1;
    }
</style>

W tym przykładzie, tekst główny oraz boczny panel są wyświetlane obok siebie dzięki układowi flex, ale logiczna kolejność w HTML jest zachowana: najpierw nagłówek, potem treść główna, a na końcu informacje dodatkowe. Czytnik ekranu odczyta tekst w logicznej kolejności, niezależnie od układu wizualnego.

Wyzwania i wskazówki

  • Układy wielokolumnowe - upewnij się, że kolejność treści w kodzie HTML jest logiczna, nawet jeśli wizualnie treść jest prezentowana w wielu kolumnach
  • Tabele danych - używaj tabel tylko do prezentacji danych tabelarycznych, nie do układu strony
  • Float i pozycjonowanie absolutne - bądź ostrożny przy używaniu tych właściwości CSS, gdyż mogą zaburzyć kolejność odczytu
  • Flexbox i Grid - używaj tych nowoczesnych layoutów CSS, ale upewnij się, że kolejność elementów w HTML pozostaje logiczna
  • Testuj z wyłączonym CSS - sprawdź, czy treść nadal ma sens, gdy style są wyłączone
  • Użyj czytnika ekranu - testuj swoją stronę za pomocą czytnika ekranu, aby sprawdzić, czy treść jest odczytywana w logicznej kolejności
  • Atrybut tabindex - unikaj używania wartości dodatnich dla tabindex, ponieważ mogą one zaburzyć naturalną kolejność nawigacji klawiaturą
