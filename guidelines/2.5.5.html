2.5.5 Rozmiar obszaru dotyku

Poziom AAA

Oficjalna dokumentacja W3C

Opis kryterium

Rozmiar celu dla wejścia wskaźnikowego wynosi co najmniej 44 na 44 piksele CSS, z wyjątkiem gdy:

  • Równoważnik - cel jest dostępny poprzez równoważny link lub kontrolkę na tej samej stronie, która ma rozmiar co najmniej 44 na 44 piksele CSS
  • W tekście - cel jest w zdaniu lub bloku tekstu
  • Kontrola user agent - rozmiar celu jest określony przez user agent i nie jest modyfikowany przez autora
  • Niezbędna prezentacja - szczególna prezentacja celu jest niezbędna dla przekazania informacji

Intencja

Celem tego kryterium sukcesu jest zapewnienie, że cele dotyku są wystarczająco duże, aby użytkownicy mogli je łatwo aktywować bez przypadkowego dotknięcia sąsiednich celów. Pomaga to użytkownikom:

  • Z ograniczeniami motorycznymi - którzy mogą mieć trudności z precyzyjnym dotykaniem
  • Z drżeniem rąk - którzy potrzebują większych obszarów dotyku
  • Używającymi grubych palców - większe obszary redukują błędy
  • Starszym - którzy mogą mieć zmniejszoną precyzję dotyku
  • Używającymi urządzeń jedną ręką - trudniejsze precyzyjne pozycjonowanie
  • W ruchu - w pojazdach, podczas chodzenia

Kryterium zapewnia komfort użytkowania i zmniejsza frustrację związaną z przypadkowymi aktywacjami.

Wymagania

Cele dotyku muszą mieć:

  • Minimalny rozmiar 44×44 piksele CSS - dla wszystkich interaktywnych elementów
  • Odpowiednie odstępy - między sąsiednimi celami dotyku
  • Wyraźne granice - użytkownicy muszą widzieć obszar dotyku
  • Spójność - podobne elementy mają podobne rozmiary

Definicje

  • Cel dotyku - obszar interfejsu, który reaguje na wejście wskaźnikowe
  • Piksele CSS - jednostka miary niezależna od urządzenia
  • Wejście wskaźnikowe - dotyk, kliknięcie myszą, rysik
  • Równoważny cel - alternatywny sposób wykonania tej samej akcji

Przykłady

Zły przykład - za małe obszary dotyku

Cele dotyku poniżej zalecanego minimum 44×44 piksele:

❌ Problematyczne rozmiary:
1. Małe przyciski bez odstępów

❌ Rozmiar: 20×20px | Odstępy: 2px
Za małe dla precyzyjnego dotykania

2. Zagęszczone linki w menu
Home About Products Contact

❌ Wysokość linków: ~18px | Padding: 2px
Trudne do trafienia na urządzeniach dotykowych

3. Miniaturowe kontrolki formularza

❌ Rozmiar checkboxów/radio: 12×12px
Za małe dla wygodnego zaznaczania

4. Bardzo małe ikony działań
Dokument.pdf

❌ Rozmiar ikon: 16×16px | Odstępy: 1px
Ryzyko przypadkowych aktywacji

Dlaczego to problematyczne?
  • Trudność trafienia - małe cele wymagają większej precyzji
  • Przypadkowe aktywacje - łatwe pomylenie sąsiednich elementów
  • Frustracja użytkowników - wielokrotne próby trafienia
  • Wykluczenie grup - osoby z ograniczeniami motorycznymi
  • Zmęczenie - większy wysiłek przy małych celach

Problem: Cele dotyku są znacznie mniejsze niż zalecane 44×44 piksele, co sprawia, że są trudne do trafienia, szczególnie dla osób z ograniczeniami motorycznymi lub na urządzeniach dotykowych. Małe odstępy zwiększają ryzyko przypadkowych aktywacji.

Zalety odpowiednich rozmiarów:
  • Łatwość użycia - proste trafienie w cel bez wysiłku
  • Mniej błędów - zmniejszone ryzyko przypadkowych aktywacji
  • Większa dostępność - komfort dla osób z ograniczeniami
  • Lepsze doświadczenie - mniej frustracji i zmęczenia
  • Uniwersalność - działa na wszystkich urządzeniach

Rozwiązanie: Wszystkie cele dotyku mają minimum 44×44 piksele z odpowiednimi odstępami między elementami. Etykiety formularzy są kliklane w całości, zwiększając obszar dotyku. Implementacja zapewnia komfort użytkowania na wszystkich urządzeniach.

Testowanie zgodności z 2.5.5

Procedura testowa

  1. Identyfikacja celów dotyku
    • Znajdź wszystkie interaktywne elementy (przyciski, linki, kontrolki)
    • Sprawdź ikony, przyciski nawigacyjne, kontrolki formularzy
    • Zidentyfikuj elementy z małymi obszarami dotyku
  2. Pomiar rozmiarów
    • Użyj narzędzi deweloperskich do pomiaru rzeczywistych wymiarów
    • Sprawdź CSS padding i margin wpływający na obszar dotyku
    • Przetestuj na różnych rozmiarach ekranów
  3. Test odstępów
    • Sprawdź odległości między sąsiednimi celami dotyku
    • Zweryfikuj czy nie ma nakładających się obszarów
    • Przetestuj na urządzeniach dotykowych
  4. Test funkcjonalności
    • Sprawdź czy wszystkie cele są łatwo dostępne
    • Przetestuj z różnymi rozmiarami palców
    • Zweryfikuj brak przypadkowych aktywacji

Narzędzia testowe

  • Chrome DevTools - pomiar elementów w pikselach CSS
  • Firefox Developer Tools - inspector z wymiarami
  • WAVE - sprawdzanie dostępności elementów
  • Accessibility Insights - testy rozmiaru celów
  • Lighthouse - audit dostępności
  • Rzeczywiste urządzenia - telefony, tablety

Pomiar w narzędziach deweloperskich

// Sprawdzenie rozmiarów elementów w konsoli
function checkTargetSizes() {
    const interactiveElements = document.querySelectorAll(
        'button, a, input[type="checkbox"], input[type="radio"], ' +
        'input[type="range"], select, [role="button"], [onclick]'
    );
    
    const minSize = 44; // minimum w pikselach CSS
    const issues = [];
    
    interactiveElements.forEach((element, index) => {
        const rect = element.getBoundingClientRect();
        const computedStyle = window.getComputedStyle(element);
        
        // Uwzględnij padding w rozmiarze dotyku
        const totalWidth = rect.width;
        const totalHeight = rect.height;
        
        if (totalWidth < minSize || totalHeight < minSize) {
            issues.push({
                element: element,
                index: index,
                width: totalWidth,
                height: totalHeight,
                tagName: element.tagName,
                classes: element.className
            });
        }
    });
    
    if (issues.length > 0) {
        console.warn(`Znaleziono ${issues.length} elementów poniżej 44x44px:`);
        console.table(issues);
    } else {
        console.log('✅ Wszystkie cele dotyku spełniają wymóg 44x44px');
    }
    
    return issues;
}

// Uruchom test
checkTargetSizes();

// Sprawdź konkretny element
function checkElementSize(selector) {
    const element = document.querySelector(selector);
    if (element) {
        const rect = element.getBoundingClientRect();
        console.log(`Element ${selector}:`, {
            width: rect.width,
            height: rect.height,
            meetsRequirement: rect.width >= 44 && rect.height >= 44
        });
    }
}

// Przykład użycia
checkElementSize('button');

Checklist zgodności

✅ Kryteria do spełnienia:
  • ☐ Wszystkie cele dotyku mają minimum 44×44 piksele CSS
  • ☐ Odpowiednie odstępy między sąsiednimi celami (min 8px)
  • ☐ Obszary dotyku obejmują padding i margin elementów
  • ☐ Elementy nie nakładają się na siebie
  • ☐ Rozmiary zachowane na różnych urządzeniach
  • ☐ Etykiety formularzy są kliklane (zwiększają obszar dotyku)
  • ☐ Ikony mają wystarczający obszar dotyku wokół siebie
  • ☐ Menu i nawigacja łatwe do użycia na dotykowych ekranach

Wytyczne implementacyjne

Najlepsze praktyki dla rozmiarów celów dotyku:

  • Minimum 44×44 piksele - nigdy nie schodź poniżej tego rozmiaru
  • Preferuj 48×48 piksele - na urządzeniach mobilnych
  • Uwzględnij padding - padding liczy się do obszaru dotyku
  • Zachowaj odstępy - minimum 8px między celami
  • Testuj na urządzeniach - rzeczywiste testy dotykowe
  • Używaj względnych jednostek - em, rem, % dla responsywności

Wzorce implementacji:

Podstawowe rozmiary przycisków
.touch-target {
    min-width: 44px;
    min-height: 44px;
    padding: 12px 16px;
    margin: 4px;
}
Dostępne kontrolki formularza

.form-control-label {
    display: flex;
    align-items: center;
    min-height: 44px;
    padding: 8px;
    cursor: pointer;
    gap: 12px;

.form-control-label input {
    width: 20px;
    height: 20px;
}
Responsywne rozmiary dotyku
/* Desktop */
.touch-target {
    min-width: 44px;
    min-height: 44px;
}

/* Mobile - większe rozmiary */
@media (max-width: 768px) {
    .touch-target {
        min-width: 48px;
        min-height: 48px;
        padding: 14px 18px;
    }
}

Implementacja krok po kroku:

  1. Audit obecnych rozmiarów - znajdź elementy poniżej 44px
  2. Ustaw minimalne rozmiary - min-width, min-height w CSS
  3. Dodaj odpowiedni padding - zwiększ obszar dotyku
  4. Sprawdź odstępy - margin/gap między elementami
  5. Przetestuj na urządzeniach - rzeczywiste testy dotykowe
  6. Zoptymalizuj dla mobile - większe rozmiary na małych ekranach
  7. Dokumentuj standardy - wytyczne dla zespołu

Specjalne przypadki i wyjątki:

  • Linki w tekście - mogą być mniejsze, jeśli są częścią zdania
  • Kontrolki systemu - domyślne style przeglądarki (ale lepiej je dostosować)
  • Równoważne cele - jeśli istnieje większa alternatywa na tej samej stronie
  • Niezbędna prezentacja - gdy rozmiar ma znaczenie semantyczne (np. mapa)

/* Przyciski */

    touch-action: manipulation; /* zapobiega zoom na double-tap */
}

/* Linki nawigacyjne */
nav a {
    display: inline-block;
    min-height: 44px;
    padding: 12px 16px;
    line-height: 20px;
}

/* Kontrolki formularza - zwiększenie przez etykiety */
.checkbox-wrapper,
.radio-wrapper {

    align-items: center;
    padding: 8px;
}

/* Ikony z odpowiednim obszarem dotyku */

    min-width: 44px;
    min-height: 44px;
    display: inline-flex;
    align-items: center;
    justify-content: center;
    padding: 8px;
}

/* Sliders */
input[type="range"] {
    min-height: 44px;
    cursor: pointer;
}

/* Dropdown */
select {
    min-height: 44px;
    padding: 12px;
}

Częste błędy do uniknięcia:

  • Liczenie tylko contentu - padding i border też liczą się do obszaru dotyku
  • Za małe odstępy - sąsiednie elementy za blisko siebie
  • Ignorowanie mobile - małe cele na urządzeniach dotykowych
  • Nakładające się obszary - konflikty między sąsiednimi celami
  • Nieresponzywne rozmiary - stałe piksele zamiast względnych jednostek
  • Tylko width bez height - oba wymiary muszą spełniać minimum
