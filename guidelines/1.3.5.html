1.3.5 Określenie pożądanej wartości

Poziom AA

Oficjalna dokumentacja W3C

Opis kryterium

Cel danych wejściowych może zostać określony programowo, gdy:

  • Pola formularza zbierające informacje o użytkowniku mają określony cel zgodnie z sekcją Input Purposes for User Interface Components

Intencja

Celem tego kryterium jest zapewnienie, że pola formularzy zbierających dane użytkownika (np. dane osobowe, informacje kontaktowe) są odpowiednio oznaczone, aby przeglądarki i technologie wspomagające mogły automatycznie rozpoznać ich cel. Dzięki temu:

  • Przeglądarki mogą automatycznie wypełniać formularze zapisanymi wcześniej danymi
  • Użytkownicy z zaburzeniami poznawczymi mają wsparcie przy wypełnianiu formularzy
  • Technologie wspomagające mogą prezentować pola w bardziej zrozumiały sposób, np. dodając ikony ułatwiające identyfikację
  • Użytkownicy z ograniczoną motoryką muszą wykonać mniej czynności, aby wypełnić formularze

Przykłady

Zły przykład - brak określenia celu pól formularza

Formularz bez atrybutów określających cel pól:

<form>
    <div>
        <label for="name">Imię i nazwisko:</label>
        <input type="text" id="name" name="name">
    </div>
    <div>
        <label for="email">Email:</label>
        <input type="email" id="email" name="email">
    </div>
    <div>
        <label for="tel">Telefon:</label>
        <input type="tel" id="tel" name="tel">
    </div>
    <div>
        <label for="address">Adres:</label>
        <input type="text" id="address" name="address">
    </div>
    <div>
        <label for="city">Miasto:</label>
        <input type="text" id="city" name="city">
    </div>
    <div>
        <label for="postal">Kod pocztowy:</label>
        <input type="text" id="postal" name="postal">
    </div>
    <button type="submit">Wyślij</button>
</form>
Podgląd:

Ten przykład jest problemem z dostępnością, ponieważ:

  • Pola nie zawierają atrybutu autocomplete określającego ich cel
  • Przeglądarki nie mogą rozpoznać konkretnego przeznaczenia pól
  • Automatyczne wypełnianie formularza może działać nieprawidłowo
  • Technologie wspomagające nie otrzymują dodatkowych informacji o celu pól
  • Osoby z niepełnosprawnościami poznawczymi muszą włożyć więcej wysiłku w zrozumienie i wypełnienie formularza

Dobry przykład - określenie celu pól formularza za pomocą autocomplete

Formularz z atrybutami autocomplete określającymi cel pól:

<form>
    <div>
        <label for="name">Imię i nazwisko:</label>
        <input type="text" id="name" name="name" autocomplete="name">
    </div>
    <div>
        <label for="email">Email:</label>
        <input type="email" id="email" name="email" autocomplete="email">
    </div>
    <div>
        <label for="tel">Telefon:</label>
        <input type="tel" id="tel" name="tel" autocomplete="tel">
    </div>
    <div>
        <label for="street-address">Adres:</label>
        <input type="text" id="street-address" name="street-address" autocomplete="street-address">
    </div>
    <div>
        <label for="address-level2">Miasto:</label>
        <input type="text" id="address-level2" name="city" autocomplete="address-level2">
    </div>
    <div>
        <label for="postal-code">Kod pocztowy:</label>
        <input type="text" id="postal-code" name="postal" autocomplete="postal-code">
    </div>
    <button type="submit">Wyślij</button>
</form>
Podgląd:

Ten przykład jest dostępny, ponieważ:

  • Każde pole ma odpowiedni atrybut autocomplete zgodny ze specyfikacją
  • Przeglądarki mogą automatycznie rozpoznać cel pól i oferować wypełnienie danymi
  • Wartości autocomplete są zgodne ze standardem Input Purposes for User Interface Components
  • Technologie wspomagające mogą dostarczyć dodatkowe wskazówki wizualne lub dźwiękowe
  • Użytkownicy z zaburzeniami poznawczymi mogą łatwiej zidentyfikować przeznaczenie pól
  • Użytkownicy z ograniczoną motoryką wykonują mniej czynności dzięki automatycznemu wypełnianiu

Dobry przykład - złożony formularz z sekcjami i pełnym zestawem atrybutów autocomplete

Bardziej rozbudowany formularz z atrybutami autocomplete:

<form>
    <h3>Dane osobowe</h3>
    <div>
        <label for="given-name">Imię:</label>
        <input type="text" id="given-name" name="given-name" autocomplete="given-name" required>
    </div>
    <div>
        <label for="family-name">Nazwisko:</label>
        <input type="text" id="family-name" name="family-name" autocomplete="family-name" required>
    </div>
    <div>
        <label for="bday">Data urodzenia:</label>
        <input type="date" id="bday" name="bday" autocomplete="bday">
    </div>
    
    <h3>Dane kontaktowe</h3>
    <div>
        <label for="email">Email:</label>
        <input type="email" id="email" name="email" autocomplete="email" required>
    </div>
    <div>
        <label for="tel">Telefon:</label>
        <input type="tel" id="tel" name="tel" autocomplete="tel">
    </div>
    
    <h3>Adres dostawy</h3>
    <div>
        <label for="shipping-street-address">Ulica i numer:</label>
        <input type="text" id="shipping-street-address" name="shipping-street-address" autocomplete="shipping street-address" required>
    </div>
    <div>
        <label for="shipping-address-level2">Miasto:</label>
        <input type="text" id="shipping-address-level2" name="shipping-city" autocomplete="shipping address-level2" required>
    </div>
    <div>
        <label for="shipping-postal-code">Kod pocztowy:</label>
        <input type="text" id="shipping-postal-code" name="shipping-postal" autocomplete="shipping postal-code" required>
    </div>
    <div>
        <label for="shipping-country">Kraj:</label>
        <select id="shipping-country" name="shipping-country" autocomplete="shipping country" required>
            <option value="PL">Polska</option>
            <option value="DE">Niemcy</option>
            <option value="CZ">Czechy</option>
            <option value="SK">Słowacja</option>
        </select>
    </div>
    
    <h3>Dane karty płatniczej</h3>
    <div>
        <label for="cc-name">Imię i nazwisko na karcie:</label>
        <input type="text" id="cc-name" name="cc-name" autocomplete="cc-name">
    </div>
    <div>
        <label for="cc-number">Numer karty:</label>
        <input type="text" id="cc-number" name="cc-number" autocomplete="cc-number">
    </div>
    <div>
        <label for="cc-exp">Data ważności:</label>
        <input type="text" id="cc-exp" name="cc-exp" autocomplete="cc-exp" placeholder="MM/RR">
    </div>
    <div>
        <label for="cc-csc">Kod bezpieczeństwa:</label>
        <input type="text" id="cc-csc" name="cc-csc" autocomplete="cc-csc">
    </div>
    
    <button type="submit">Zamów i zapłać</button>
</form>

Ten przykład jest dostępny, ponieważ:

  • Każde pole ma odpowiedni i dokładny atrybut autocomplete
  • Formularz używa precyzyjnych wartości, np. given-name i family-name zamiast ogólnego name
  • Prefiks shipping jest używany dla pól adresu dostawy, co pozwala przeglądarkom rozróżnić adresy dostawy od adresów rozliczeniowych
  • Dane karty płatniczej mają prefiks cc- zgodnie ze standardem
  • Formularz jest podzielony na sekcje z nagłówkami, co dodatkowo poprawia strukturę i czytelność
  • Wymagane pola są oznaczone atrybutem required, co stanowi dodatkową informację dla użytkownika
Podgląd:

Dane osobowe

autocomplete="given-name"
autocomplete="family-name"
autocomplete="bday"

Dane kontaktowe

autocomplete="email"
autocomplete="tel"

Adres dostawy

autocomplete="shipping street-address"
autocomplete="shipping address-level2"
autocomplete="shipping postal-code"
autocomplete="shipping country"

Dane karty płatniczej

autocomplete="cc-name"
autocomplete="cc-number"
autocomplete="cc-exp"
autocomplete="cc-csc"

Wyzwania i wskazówki

  • Zidentyfikuj odpowiednie pola - to kryterium dotyczy przede wszystkim pól zbierających dane osobowe, nie wszystkie pola formularzy
  • Używaj standardowych wartości autocomplete - aby spełnić to kryterium, stosuj wartości z oficjalnej listy WCAG
  • Grupuj powiązane pola - używaj prefiksów takich jak shipping, billing czy cc- dla pól należących do tych samych kategorii
  • Zachowaj spójność - używaj tych samych wartości autocomplete dla podobnych pól w całej witrynie
  • Nie nadużywaj autocomplete - używaj go tylko dla standardowych typów danych opisanych w specyfikacji
  • Pamiętaj o prywatności - dla pól z danymi wrażliwymi możesz rozważyć użycie autocomplete="off", ale tylko gdy jest to uzasadnione
  • Testuj w różnych przeglądarkach - zachowanie funkcji autouzupełniania może się różnić między przeglądarkami

Lista najczęściej używanych wartości autocomplete:

Wartość Opis
name Pełne imię i nazwisko
given-name Imię
family-name Nazwisko
email Adres e-mail
tel Numer telefonu
street-address Adres - ulica i numer
address-level2 Miasto
postal-code Kod pocztowy
country Kraj
bday Data urodzenia (pełna)
username Nazwa użytkownika
current-password Aktualne hasło
new-password Nowe hasło
cc-name Imię i nazwisko na karcie płatniczej
cc-number Numer karty płatniczej
cc-exp Data ważności karty
