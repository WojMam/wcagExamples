Przejdź do treści głównej

2.2.2 Wstrzymywanie (pauza), zatrzymywanie, ukrywanie

Poziom A

Oficjalna dokumentacja W3C

Opis kryterium

Dla ruchomych, migających, przewijanych lub automatycznie aktualizujących się informacji spełnione są wszystkie poniższe warunki:

  • Ruch, miganie, przewijanie: Dla każdej ruchomej, migającej lub przewijanej informacji, która (1) uruchamia się automatycznie, (2) trwa dłużej niż pięć sekund i (3) jest prezentowana równolegle z inną treścią, istnieje mechanizm umożliwiający użytkownikowi wstrzymanie, zatrzymanie lub ukrycie tej informacji, chyba że ruch, miganie lub przewijanie jest częścią aktywności, gdzie jest to niezbędne;
  • Automatyczna aktualizacja: Dla każdej automatycznie aktualizującej się informacji, która (1) uruchamia się automatycznie i (2) jest prezentowana równolegle z inną treścią, istnieje mechanizm umożliwiający użytkownikowi wstrzymanie, zatrzymanie lub ukrycie tej informacji albo kontrolowanie częstotliwości aktualizacji, chyba że automatyczna aktualizacja jest częścią aktywności, gdzie jest to niezbędne.

Intencja

Celem tego kryterium sukcesu jest uniknięcie rozpraszania użytkowników podczas ich interakcji z treścią internetową. Ruchoma treść może stanowić poważne rozproszenie dla niektórych osób, w szczególności dla tych z zaburzeniami uwagi.

Treść, która się porusza lub automatycznie aktualizuje, może stanowić barierę dla każdego, kto ma trudności z szybkim czytaniem tekstu stacjonarnego oraz dla osób, które mają problemy ze śledzeniem poruszających się obiektów. Może to również powodować problemy dla osób korzystających z czytników ekranu.

Pięć sekund wybrano, ponieważ jest to czas wystarczająco długi, aby zwrócić uwagę użytkownika, ale nie na tyle długi, aby użytkownik nie mógł poczekać na zakończenie rozpraszającego elementu, jeśli jest to konieczne do korzystania ze strony.

Przykłady

Zły przykład - karuzelowy baner bez kontrolek

Strona główna witryny zawiera animowany baner z automatycznie zmieniającymi się obrazami co 10 sekund:

<div class="carousel">
    <img src="banner1.jpg" alt="Promocja letnia: 20% rabatu na wszystkie produkty">
    <!-- Kolejne obrazy będą pokazywane automatycznie -->
</div>

<script>
    // Kod JavaScript automatycznie zmieniający obrazy co 10 sekund
    const images = ["banner1.jpg", "banner2.jpg", "banner3.jpg"];
    let currentIndex = 0;
    const carouselImg = document.querySelector('.carousel img');
    
    setInterval(function() {
        currentIndex = (currentIndex + 1) % images.length;
        carouselImg.src = images[currentIndex];
        // Tutaj można również zmieniać atrybut alt w zależności od obrazu
    }, 10000); // 10 sekund
</script>

Problem: Treść automatycznie się zmienia, trwa dłużej niż 5 sekund i nie ma mechanizmu umożliwiającego użytkownikowi zatrzymanie lub wstrzymanie karuzeli. Osoby z zaburzeniami poznawczymi lub wolno czytające mogą nie zdążyć przeczytać informacji przed zmianą obrazu.

Dobry przykład - karuzelowy baner z kontrolkami

Ta sama karuzela z dodanymi elementami sterującymi:

<div class="carousel-container">
    <div class="carousel">
        <img src="banner1.jpg" alt="Promocja letnia: 20% rabatu na wszystkie produkty">
        <!-- Kolejne obrazy będą pokazywane automatycznie -->
    </div>
    
    <div class="carousel-controls">
        <button id="carousel-pause" aria-label="Wstrzymaj karuzelę">⏸️ Pauza</button>
        <button id="carousel-play" aria-label="Wznów karuzelę" style="display: none;">▶️ Odtwórz</button>
        <button id="carousel-prev" aria-label="Poprzedni slajd">< Poprzedni</button>
        <button id="carousel-next" aria-label="Następny slajd">Następny ></button>
    </div>
</div>

<script>
    const images = ["banner1.jpg", "banner2.jpg", "banner3.jpg"];
    const altTexts = [
        "Promocja letnia: 20% rabatu na wszystkie produkty", 
        "Nowa kolekcja jesień 2023 już dostępna", 
        "Darmowa dostawa dla zamówień powyżej 200 zł"
    ];
    let currentIndex = 0;
    const carouselImg = document.querySelector('.carousel img');
    const pauseBtn = document.getElementById('carousel-pause');
    const playBtn = document.getElementById('carousel-play');
    const prevBtn = document.getElementById('carousel-prev');
    const nextBtn = document.getElementById('carousel-next');
    
    let intervalId = startCarousel();
    
    function startCarousel() {
        return setInterval(function() {
            showNextSlide();
        }, 10000); // 10 sekund
    }
    
    function showNextSlide() {
        currentIndex = (currentIndex + 1) % images.length;
        updateSlide();
    }
    
    function showPrevSlide() {
        currentIndex = (currentIndex - 1 + images.length) % images.length;
        updateSlide();
    }
    
    function updateSlide() {
        carouselImg.src = images[currentIndex];
        carouselImg.alt = altTexts[currentIndex];
    }
    
    pauseBtn.addEventListener('click', function() {
        clearInterval(intervalId);
        pauseBtn.style.display = 'none';
        playBtn.style.display = 'inline-block';
    });
    
    playBtn.addEventListener('click', function() {
        intervalId = startCarousel();
        playBtn.style.display = 'none';
        pauseBtn.style.display = 'inline-block';
    });
    
    prevBtn.addEventListener('click', function() {
        clearInterval(intervalId);
        showPrevSlide();
        intervalId = startCarousel();
    });
    
    nextBtn.addEventListener('click', function() {
        clearInterval(intervalId);
        showNextSlide();
        intervalId = startCarousel();
    });
</script>
Podgląd:
[Obraz karuzeli]

Rozwiązanie: Karuzela zawiera przyciski umożliwiające użytkownikowi wstrzymanie, wznowienie oraz ręczną nawigację między slajdami. Dzięki temu użytkownicy mają pełną kontrolę nad tempem zmiany treści i mogą ją dostosować do swoich potrzeb, co jest szczególnie ważne dla osób z zaburzeniami poznawczymi, wolno czytających lub korzystających z technologii wspomagających.

Zły przykład - automatycznie odświeżająca się lista wiadomości

Strona z listą aktualności, która odświeża się automatycznie co minutę:

<div id="news-feed">
    <h3>Najnowsze wiadomości</h3>
    <ul>
        <li>14:35 - Nowe rozporządzenie w sprawie dostępności cyfrowej</li>
        <li>14:28 - Ministerstwo ogłasza konkurs dla startupów</li>
        <li>14:15 - Konferencja na temat sztucznej inteligencji już jutro</li>
    </ul>
</div>

<script>
    // Automatyczne odświeżanie co minutę
    setInterval(function() {
        fetch('/api/najnowsze-wiadomosci')
            .then(response => response.json())
            .then(data => {
                const newsList = document.querySelector('#news-feed ul');
                newsList.innerHTML = ''; // Usuń istniejące wiadomości
                
                // Dodaj nowe wiadomości
                data.forEach(item => {
                    const li = document.createElement('li');
                    li.textContent = `${item.time} - ${item.title}`;
                    newsList.appendChild(li);
                });
            });
    }, 60000); // 60 sekund
</script>

Problem: Lista wiadomości odświeża się automatycznie bez możliwości kontroli przez użytkownika. Użytkownik może właśnie czytać wiadomość, która nagle zniknie i zostanie zastąpiona nową treścią, co może być dezorientujące i uciążliwe, szczególnie dla osób z niepełnosprawnościami poznawczymi.

Dobry przykład - lista wiadomości z kontrolą aktualizacji

Ta sama lista wiadomości z dodanymi elementami sterującymi:

<div id="news-feed">
    <div class="news-header">
        <h3>Najnowsze wiadomości</h3>
        <div class="news-controls">
            <button id="auto-update-toggle" aria-pressed="true">Automatyczne odświeżanie: Włączone</button>
            <button id="manual-update">Odśwież teraz</button>
            <div class="update-frequency">
                <label for="update-interval">Częstotliwość odświeżania:</label>
                <select id="update-interval">
                    <option value="30000">30 sekund</option>
                    <option value="60000" selected>1 minuta</option>
                    <option value="300000">5 minut</option>
                    <option value="600000">10 minut</option>
                </select>
            </div>
        </div>
    </div>
    
    <ul>
        <li>14:35 - Nowe rozporządzenie w sprawie dostępności cyfrowej</li>
        <li>14:28 - Ministerstwo ogłasza konkurs dla startupów</li>
        <li>14:15 - Konferencja na temat sztucznej inteligencji już jutro</li>
    </ul>
    
    <div class="news-footer">
        <p>Ostatnia aktualizacja: <span id="last-update-time">14:35</span></p>
    </div>
</div>

<script>
    let updateIntervalId;
    const toggleBtn = document.getElementById('auto-update-toggle');
    const manualBtn = document.getElementById('manual-update');
    const intervalSelect = document.getElementById('update-interval');
    const lastUpdateSpan = document.getElementById('last-update-time');
    
    // Początkowe ustawienie interwału
    let updateInterval = parseInt(intervalSelect.value);
    
    // Funkcja odświeżająca wiadomości
    function updateNews() {
        fetch('/api/najnowsze-wiadomosci')
            .then(response => response.json())
            .then(data => {
                const newsList = document.querySelector('#news-feed ul');
                newsList.innerHTML = ''; // Usuń istniejące wiadomości
                
                // Dodaj nowe wiadomości
                data.forEach(item => {
                    const li = document.createElement('li');
                    li.textContent = `${item.time} - ${item.title}`;
                    newsList.appendChild(li);
                });
                
                // Aktualizuj czas ostatniej aktualizacji
                const now = new Date();
                lastUpdateSpan.textContent = `${now.getHours()}:${String(now.getMinutes()).padStart(2, '0')}`;
            });
    }
    
    // Funkcja włączająca automatyczne odświeżanie
    function startAutoUpdate() {
        updateIntervalId = setInterval(updateNews, updateInterval);
        toggleBtn.setAttribute('aria-pressed', 'true');
        toggleBtn.textContent = 'Automatyczne odświeżanie: Włączone';
    }
    
    // Funkcja wyłączająca automatyczne odświeżanie
    function stopAutoUpdate() {
        clearInterval(updateIntervalId);
        toggleBtn.setAttribute('aria-pressed', 'false');
        toggleBtn.textContent = 'Automatyczne odświeżanie: Wyłączone';
    }
    
    // Inicjalizacja automatycznego odświeżania
    startAutoUpdate();
    
    // Obsługa przełączania automatycznego odświeżania
    toggleBtn.addEventListener('click', function() {
        if (toggleBtn.getAttribute('aria-pressed') === 'true') {
            stopAutoUpdate();
        } else {
            startAutoUpdate();
        }
    });
    
    // Obsługa ręcznego odświeżania
    manualBtn.addEventListener('click', updateNews);
    
    // Obsługa zmiany częstotliwości odświeżania
    intervalSelect.addEventListener('change', function() {
        updateInterval = parseInt(this.value);
        
        if (toggleBtn.getAttribute('aria-pressed') === 'true') {
            stopAutoUpdate();
            startAutoUpdate();
        }
    });
</script>
Podgląd:

Najnowsze wiadomości

  • 14:35 - Nowe rozporządzenie w sprawie dostępności cyfrowej
  • 14:28 - Ministerstwo ogłasza konkurs dla startupów
  • 14:15 - Konferencja na temat sztucznej inteligencji już jutro

Ostatnia aktualizacja: 14:35

Rozwiązanie: Strona zawiera kontrolki umożliwiające użytkownikowi:

  • Włączanie/wyłączanie automatycznego odświeżania
  • Ręczne odświeżanie treści
  • Ustawienie częstotliwości aktualizacji

Dodatkowo, informacja o ostatniej aktualizacji pomaga użytkownikowi zorientować się, kiedy treść została zmieniona. Dzięki tym funkcjom użytkownicy mają pełną kontrolę nad tempem prezentowania nowych informacji.

Dopuszczalny przykład - animacja ładowania

Strona z animacją ładowania, która jest niezbędnym elementem:

<div class="loading-screen">
    <div class="spinner"></div>
    <p>Trwa ładowanie aplikacji. Proszę czekać...</p>
    <p>To może potrwać do 30 sekund przy wolnym połączeniu.</p>
</div>

<style>
    .loading-screen {
        position: fixed;
        top: 0;
        left: 0;
        width: 100%;
        height: 100%;
        background-color: rgba(255, 255, 255, 0.9);
        display: flex;
        flex-direction: column;
        justify-content: center;
        align-items: center;
        z-index: 1000;
    }
    
    .spinner {
        width: 50px;
        height: 50px;
        border: 5px solid #f3f3f3;
        border-top: 5px solid #3498db;
        border-radius: 50%;
        animation: spin 1s linear infinite;
        margin-bottom: 20px;
    }
    
    @keyframes spin {
        0% { transform: rotate(0deg); }
        100% { transform: rotate(360deg); }
    }
</style>

<script>
    // Kod inicjujący ładowanie aplikacji
    window.addEventListener('load', function() {
        // Symulacja ładowania zasobów
        setTimeout(function() {
            document.querySelector('.loading-screen').style.display = 'none';
            // Tutaj pokazałoby się właściwe UI aplikacji
        }, 3000); // Symulacja 3-sekundowego ładowania
    });
</script>

Wyjaśnienie: Animacja ładowania jest uważana za niezbędną, ponieważ:

  • Jest jedyną treścią na ekranie (nie jest prezentowana równolegle z inną treścią)
  • Informuje użytkownika o postępie ładowania aplikacji
  • Zapobiega dezorientacji użytkownika, który mógłby w przeciwnym razie pomyśleć, że strona zawiesiła się lub nie działa
  • Znika automatycznie po zakończeniu ładowania

W tym przypadku, zgodnie z "Uwagą 4" z oficjalnej dokumentacji WCAG, animacja ładowania może być uznana za niezbędną i nie wymaga dostarczenia mechanizmu do jej zatrzymania.

Dodatkowe zasoby

  • Zrozumieć WCAG: Wstrzymywanie (pauza), zatrzymywanie, ukrywanie
