2.2.3 Bez ograniczeń czasowych

Poziom AAA

Oficjalna dokumentacja W3C

Opis kryterium

Czas nie jest istotną częścią zdarzenia lub aktywności prezentowanej w treści, z wyjątkiem nieinteraktywnych zsynchronizowanych mediów i zdarzeń czasu rzeczywistego.

Intencja

Celem tego kryterium jest zminimalizowanie występowania treści, które wymagają interakcji opartej na ograniczeniach czasowych. Umożliwia to osobom niewidomym, słabowidzącym, osobom z ograniczeniami poznawczymi lub ruchowymi swobodną interakcję z treścią. To kryterium różni się od kryterium na poziomie A (2.2.1) tym, że jedynym dopuszczalnym wyjątkiem są wydarzenia w czasie rzeczywistym.

Wiele osób z niepełnosprawnościami potrzebuje więcej czasu na wykonanie różnych zadań. Mogą one wolniej reagować fizycznie, wolniej czytać i rozumieć informacje, lub korzystać z technologii wspomagających, które wymagają dodatkowego czasu. Kryterium to zapewnia, że użytkownicy nie będą ograniczeni limitami czasowymi podczas korzystania z treści cyfrowych.

Uwaga: Treści wideo (np. w języku migowym) są objęte wytyczną 1.1.

Przykłady

Zły przykład - test z ograniczeniem czasowym

Test online z limitem czasowym na ukończenie:

<div class="test-container">
    <div class="test-header">
        <h2>Test wiedzy z dostępności cyfrowej</h2>
        <div class="timer">Pozostały czas: <span id="countdown">15:00</span></div>
    </div>
    
    <form id="quiz-form">
        <!-- Pytania testowe -->
        <div class="question">
            <p>1. Które kryterium WCAG dotyczy kontrastu tekstu?</p>
            <div class="options">
                <label><input type="radio" name="q1" value="a"> 1.1.1</label>
                <label><input type="radio" name="q1" value="b"> 1.4.3</label>
                <label><input type="radio" name="q1" value="c"> 2.4.4</label>
                <label><input type="radio" name="q1" value="d"> 3.3.2</label>
            </div>
        </div>
        
        <!-- Więcej pytań... -->
        
        <button type="submit">Zakończ test</button>
    </form>
</div>

<script>
    // Ustawienie limitu czasu na 15 minut
    let timeLeft = 15 * 60; // 15 minut w sekundach
    const countdownEl = document.getElementById('countdown');
    const quizForm = document.getElementById('quiz-form');
    
    // Funkcja odliczająca czas
    const timer = setInterval(function() {
        timeLeft--;
        
        // Aktualizacja wyświetlanego czasu
        const minutes = Math.floor(timeLeft / 60);
        const seconds = timeLeft % 60;
        countdownEl.textContent = `${minutes}:${seconds < 10 ? '0' + seconds : seconds}`;
        
        // Gdy czas się skończy, automatycznie wyślij formularz
        if (timeLeft <= 0) {
            clearInterval(timer);
            alert('Czas minął! Twój test zostanie automatycznie zakończony.');
            quizForm.submit();
        }
    }, 1000); // Aktualizacja co sekundę
</script>

Problem: Test zawiera sztywny limit czasowy, który nie może być dostosowany przez użytkownika. Osoby z różnymi niepełnosprawnościami, takimi jak zaburzenia poznawcze, wzrokowe czy ruchowe, mogą potrzebować więcej czasu na ukończenie testu. Bez możliwości dostosowania czasu, test może być dla nich niemożliwy do ukończenia, co stawia ich w niekorzystnej sytuacji.

Dobry przykład - test bez ograniczeń czasowych

Ten sam test bez narzuconego limitu czasu:

<div class="test-container">
    <div class="test-header">
        <h2>Test wiedzy z dostępności cyfrowej</h2>
        <div class="test-info">
            <p>Brak limitu czasowego. Możesz wypełnić test w swoim tempie.</p>
            <p>Czas rozpoczęcia: <span id="start-time"></span></p>
        </div>
    </div>
    
    <form id="quiz-form">
        <!-- Pytania testowe -->
        <div class="question">
            <p>1. Które kryterium WCAG dotyczy kontrastu tekstu?</p>
            <div class="options">
                <label><input type="radio" name="q1" value="a"> 1.1.1</label>
                <label><input type="radio" name="q1" value="b"> 1.4.3</label>
                <label><input type="radio" name="q1" value="c"> 2.4.4</label>
                <label><input type="radio" name="q1" value="d"> 3.3.2</label>
            </div>
        </div>
        
        <!-- Więcej pytań... -->
        
        <button type="submit">Zakończ test</button>
    </form>
</div>

<script>
    // Wyświetlenie czasu rozpoczęcia testu
    const startTimeEl = document.getElementById('start-time');
    const now = new Date();
    startTimeEl.textContent = now.toLocaleTimeString();
    
    // Opcjonalnie: Zapisanie czasu ukończenia przy wysyłaniu formularza
    document.getElementById('quiz-form').addEventListener('submit', function(e) {
        const endTime = new Date();
        const duration = Math.round((endTime - now) / 1000); // Czas w sekundach
        
        // Dodanie ukrytego pola z czasem wykonania
        const timeInput = document.createElement('input');
        timeInput.type = 'hidden';
        timeInput.name = 'duration';
        timeInput.value = duration;
        this.appendChild(timeInput);
    });
</script>
Podgląd:

Test wiedzy z dostępności cyfrowej

Brak limitu czasowego. Możesz wypełnić test w swoim tempie.

Czas rozpoczęcia: 10:15:32

1. Które kryterium WCAG dotyczy kontrastu tekstu?

Rozwiązanie: Ta wersja testu nie nakłada żadnych ograniczeń czasowych na użytkownika. Rejestrowany jest jedynie czas rozpoczęcia i ukończenia testu, ale nie ma to wpływu na możliwość wykonania zadania. Dzięki temu:

  • Każdy użytkownik może pracować we własnym tempie
  • Osoby z niepełnosprawnościami nie są dyskryminowane
  • Użytkownicy mogą zrobić przerwę, jeśli tego potrzebują
  • Wyniki testu lepiej odzwierciedlają faktyczną wiedzę, a nie zdolność do szybkiego wykonania zadania

Zły przykład - sesja logowania z automatycznym wylogowaniem

Strona z automatycznym wylogowaniem po okresie nieaktywności, bez ostrzeżenia:

<script>
    // Automatyczne wylogowanie po 30 minutach nieaktywności
    let inactivityTime = 30 * 60 * 1000; // 30 minut w milisekundach
    let logoutTimer;
    
    function resetTimer() {
        clearTimeout(logoutTimer);
        logoutTimer = setTimeout(logout, inactivityTime);
    }
    
    function logout() {
        // Bezpośrednie przekierowanie na stronę logowania
        window.location.href = 'login.html?message=session-expired';
    }
    
    // Resetowanie timera przy aktywności użytkownika
    window.addEventListener('mousemove', resetTimer);
    window.addEventListener('keypress', resetTimer);
    window.addEventListener('click', resetTimer);
    window.addEventListener('scroll', resetTimer);
    
    // Inicjalizacja timera
    resetTimer();
</script>

Problem: Strona automatycznie wylogowuje użytkownika po 30 minutach nieaktywności, bez wcześniejszego ostrzeżenia i możliwości wydłużenia sesji. Takie zachowanie może być bardzo problematyczne dla osób z niepełnosprawnościami, które mogą potrzebować więcej czasu na wykonanie zadań na stronie. Wylogowanie bez ostrzeżenia może skutkować utratą niezapisanych danych i koniecznością powtórnego logowania oraz wykonania tych samych czynności od początku.

Dobry przykład - brak automatycznego wylogowania

Strona, która nie wymusza automatycznego wylogowania, ale oferuje funkcję ręcznego wylogowania:

<header class="site-header">
    <nav>
        <ul class="nav-links">
            <li><a href="dashboard.html">Pulpit</a></li>
            <li><a href="profile.html">Mój profil</a></li>
            <li><a href="settings.html">Ustawienia</a></li>
            <li><a href="#" id="logout-btn">Wyloguj się</a></li>
        </ul>
    </nav>
</header>

<main>
    <!-- Treść strony... -->
</main>

<script>
    // Funkcja wylogowania uruchamiana tylko na żądanie użytkownika
    document.getElementById('logout-btn').addEventListener('click', function(e) {
        e.preventDefault();
        
        // Wyświetlenie potwierdzenia, jeśli są niezapisane dane
        if (hasUnsavedChanges()) {
            if (confirm('Masz niezapisane zmiany. Czy na pewno chcesz się wylogować?')) {
                performLogout();
            }
        } else {
            performLogout();
        }
    });
    
    function hasUnsavedChanges() {
        // Logika sprawdzająca, czy na stronie są niezapisane zmiany
        return false; // Przykładowo, brak zmian
    }
    
    function performLogout() {
        // Wylogowanie użytkownika
        window.location.href = 'login.html?action=logout';
    }
</script>

  • Pulpit
  • Mój profil
  • Ustawienia
  • Wyloguj się

Treść strony...

Rozwiązanie: Strona nie zawiera automatycznego wylogowania po upływie czasu, co jest zgodne z kryterium 2.2.3. Zamiast tego:

  • Użytkownik ma możliwość samodzielnego wylogowania, gdy zakończy pracę
  • Przed wylogowaniem system sprawdza, czy istnieją niezapisane zmiany i ostrzega użytkownika
  • Sesja pozostaje aktywna tak długo, jak potrzebuje tego użytkownik

Warto zaznaczyć, że w niektórych przypadkach (np. aplikacje bankowe czy medyczne) ze względów bezpieczeństwa może być konieczne wprowadzenie limitów czasowych. W takich sytuacjach należy zastosować techniki z kryterium 2.2.1, takie jak ostrzeganie przed wygaśnięciem sesji i umożliwienie jej przedłużenia.

Wyjątek - zdarzenie czasu rzeczywistego

Platforma aukcyjna, gdzie czas jest nieodłącznym elementem wydarzenia:

<div class="auction-item">
    <div class="item-details">
        <h2>Antyczny zegarek kieszonkowy, XIX wiek</h2>
        <img src="pocket-watch.jpg" alt="Antyczny zegarek kieszonkowy z grawerowaną kopertą">
        <p class="description">Unikatowy zegarek kieszonkowy z XIX wieku, z grawerowaną złotą kopertą i oryginalnym mechanizmem.</p>
    </div>
    
    <div class="auction-status">
        <div class="timer">
            <h3>Czas do zakończenia aukcji</h3>
            <div class="countdown" id="auction-countdown">00:05:23</div>
        </div>
        
        <div class="bidding">
            <h3>Aktualna cena: <span id="current-price">5200 zł</span></h3>
            <p>Najwyższa oferta złożona przez: <span id="highest-bidder">Jan K.</span></p>
            
            <form id="bid-form">
                <label for="bid-amount">Twoja oferta (min. 5300 zł):</label>
                <input type="number" id="bid-amount" min="5300" step="100" value="5300">
                <button type="submit">Złóż ofertę</button>
            </form>
        </div>
    </div>
</div>

<script>
    // Aktualizacja odliczania co sekundę
    const countdown = document.getElementById('auction-countdown');
    let timeLeft = 5 * 60 + 23; // 5 minut 23 sekundy w sekundach
    
    const timer = setInterval(function() {
        timeLeft--;
        
        // Aktualizacja wyświetlanego czasu
        const hours = Math.floor(timeLeft / 3600);
        const minutes = Math.floor((timeLeft % 3600) / 60);
        const seconds = timeLeft % 60;
        
        countdown.textContent = `${String(hours).padStart(2, '0')}:${String(minutes).padStart(2, '0')}:${String(seconds).padStart(2, '0')}`;
        
        // Zakończenie aukcji po upływie czasu
        if (timeLeft <= 0) {
            clearInterval(timer);
            endAuction();
        }
    }, 1000);
    
    function endAuction() {
        // Logika zakończenia aukcji
        countdown.textContent = "Aukcja zakończona";
        document.getElementById('bid-form').innerHTML = '<p>Aukcja została zakończona.</p>';
    }
</script>

Wyjaśnienie: W tym przypadku ograniczenie czasowe jest dopuszczalne, ponieważ:

  • Aukcja jest przykładem "zdarzenia czasu rzeczywistego" (real-time event), które jest explicite wymienione jako wyjątek w kryterium 2.2.3
  • Czas jest nieodłącznym elementem aukcji - wszyscy uczestnicy muszą mieć ten sam limit czasowy, aby zachować uczciwość procesu
  • Nie ma alternatywnego sposobu przeprowadzenia aukcji bez limitu czasowego, który zachowałby jej istotny charakter

Warto jednak pamiętać, że nawet w przypadku takich wyjątków, należy zapewnić, że interfejs jest jak najbardziej dostępny w innych aspektach - na przykład poprzez wyraźne komunikaty, przejrzysty sposób składania ofert i odpowiedni kontrast elementów.

Szczegóły techniczne

Techniki wystarczające do spełnienia kryterium:

  • G5: Umożliwienie użytkownikom ukończenia aktywności bez żadnego limitu czasowego

Korzyści

  • Osoby z niepełnosprawnościami fizycznymi często potrzebują więcej czasu na reakcję, pisanie i wykonywanie czynności
  • Osoby słabowidzące potrzebują więcej czasu na zlokalizowanie elementów na ekranie i ich przeczytanie
  • Osoby niewidome korzystające z czytników ekranu potrzebują więcej czasu na zrozumienie układu strony, znalezienie informacji i obsługę kontrolek
  • Osoby z ograniczeniami poznawczymi lub językowymi potrzebują więcej czasu na czytanie i zrozumienie tekstu
  • Osoby głuche komunikujące się w języku migowym mogą potrzebować więcej czasu na przeczytanie informacji tekstowych (które mogą być dla niektórych drugim językiem)
