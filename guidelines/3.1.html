Przejdź do treści głównej

3.1 Czytelność

Wytyczna ta zapewnia, że treść tekstowa jest czytelna i zrozumiała dla użytkowników.

Opis wytycznej

Czytelność treści to fundament dostępności cyfrowej. Użytkownicy muszą być w stanie nie tylko odczytać treść, ale także ją zrozumieć. Wytyczna 3.1 koncentruje się na:

  • Identyfikacji języka - strony i jej części
  • Wyjaśnianiu trudnych pojęć - nietypowe słowa, skróty, żargon
  • Dostosowaniu poziomu trudności - do możliwości odbiorców
  • Pomocy w wymowie - dla słów o nieoczywistej wymowie

Te elementy są szczególnie istotne dla osób:

  • Z dysleksją - które mają trudności z czytaniem
  • Z ograniczeniami poznawczymi - które potrzebują prostego języka
  • Używających czytników ekranu - które potrzebują informacji o języku
  • Uczących się języka - które nie znają zaawansowanego słownictwa
  • Starszych - które mogą mieć trudności z nowoczesnym żargonem
  • Z ograniczoną edukacją - które preferują prosty język

Kryteria sukcesu

Poziom A (podstawowy)

  • 3.1.1 Język strony (A) - określenie głównego języka strony

Poziom AA (standardowy)

  • 3.1.2 Język części (AA) - oznaczenie fragmentów w innych językach

Poziom AAA (zaawansowany)

  • 3.1.3 Nietypowe słowa (AAA) - wyjaśnianie specjalistycznych terminów i żargonu
  • 3.1.4 Skróty (AAA) - mechanizmy wyjaśniania skrótów i akronimów
  • 3.1.5 Poziom trudności czytania (AAA) - dostosowanie poziomu tekstu do umiejętności czytelniczych
  • 3.1.6 Wymowa (AAA) - mechanizmy określania wymowy dla słów o niejednoznacznym znaczeniu

Kluczowe zasady czytelności

🌍 Identyfikacja języka

Strona i jej części muszą mieć:

  • Określony główny język strony
  • Oznaczone fragmenty w innych językach
  • Prawidłowe atrybuty lang w HTML
  • Wsparcie dla technologii wspomagających

📚 Jasność komunikacji

Treść powinna być:

  • Napisana prostym, zrozumiałym językiem
  • Wyjaśniać trudne lub specjalistyczne terminy
  • Definiować skróty i akronimy
  • Dostosowana do grupy odbiorców

🎯 Dostosowanie poziomu

Trudność tekstu powinna:

  • Odpowiadać możliwościom odbiorców
  • Uwzględniać kontekst i cel treści
  • Oferować alternatywy dla trudnych fragmentów
  • Zapewniać dodatkowe wyjaśnienia

Elementy czytelności w web

1. Oznaczanie języka

<!-- Główny język strony -->
<html lang="pl">

<!-- Fragment w innym języku -->
<p>Hasło firmy to <span lang="en">"Innovation for everyone"</span>.</p>

<!-- Cytat w obcym języku -->
<blockquote lang="fr">
    <p>L'art de vivre</p>
</blockquote>

2. Wyjaśnianie terminów

<!-- Definicja terminu -->
<p>System <dfn>CMS</dfn> (Content Management System) to oprogramowanie
do zarządzania treścią strony internetowej.</p>

<!-- Skrót z rozwinięciem -->
<p>Organizacja <abbr title="World Wide Web Consortium">W3C</abbr> 
ustala standardy internetowe.</p>

<!-- Słowniczek terminów -->
<dl>
    <dt>API</dt>
    <dd>Application Programming Interface - interfejs programistyczny aplikacji</dd>
</dl>

3. Wskazówki wymowy

<!-- Wskazówka wymowy -->
<p>Miasto <span title="wymowa: Szcze-cin">Szczecin</span> leży nad Odrą.</p>

<!-- Transkrypcja fonetyczna -->
<p>Słowo <i lang="fr">rendezvous</i> 
<span class="pronunciation">[ˈrɑ̃devu]</span> pochodzi z francuskiego.</p>

Techniki poprawy czytelności

Struktura tekstu

  • Nagłówki hierarchiczne - h1, h2, h3 dla organizacji treści
  • Krótkie akapity - łatwiejsze do przyswojenia
  • Listy punktowane - dla lepszej przejrzystości
  • Białe przestrzenie - dla odpoczynku wzroku

Język i słownictwo

  • Proste zdania - unikanie nadmiernie złożonych konstrukcji
  • Aktywna strona - zamiast biernej
  • Konkretne słowa - zamiast abstrakcyjnych
  • Znane terminy - lub wyjaśnione specjalistyczne

Wsparcie wizualne

  • Ikony i symbole - wspomagające zrozumienie
  • Ilustracje - wyjaśniające trudne koncepty
  • Tabele i wykresy - dla danych liczbowych
  • Podkreślenia - dla ważnych informacji

Narzędzia analizy czytelności

Automatyczne testy

  • Indeks mglistości Gunninga - poziom trudności tekstu
  • Test Flesch-Kinaid - wymagany poziom edukacji
  • SMOG Index - lata edukacji potrzebne do zrozumienia
  • ARI (Automated Readability Index) - wiek czytelnika

Narzędzia online

Walidatory HTML

// Sprawdzenie atrybutu lang
function checkLanguageAttributes() {
    const htmlElement = document.documentElement;
    const langAttr = htmlElement.getAttribute('lang');
    
    if (!langAttr) {
        console.warn('Brak atrybutu lang w elemencie html');
        return false;
    }
    
    console.log(`Główny język strony: ${langAttr}`);
    
    // Sprawdź fragmenty w innych językach
    const foreignLangElements = document.querySelectorAll('[lang]:not(html)');
    foreignLangElements.forEach(element => {
        console.log(`Fragment w języku: ${element.getAttribute('lang')}`);
    });
    
    return true;
}

// Analiza trudnych słów
function analyzeComplexWords() {
    const textContent = document.body.textContent;
    const words = textContent.split(/\s+/);
    const complexWords = words.filter(word => word.length > 12);
    
    console.log(`Znaleziono ${complexWords.length} potencjalnie trudnych słów`);
    return complexWords;
}

Testowanie czytelności

Testy manualne

  • Czytanie na głos - czy tekst brzmi naturalnie
  • Test z użytkownikami - czy odbiorcy rozumieją treść
  • Sprawdzanie skrótów - czy wszystkie są wyjaśnione
  • Weryfikacja języka - czy fragmenty są prawidłowo oznaczone

Testy techniczne

  • Walidacja HTML - sprawdzenie atrybutów lang
  • Czytniki ekranu - test z różnymi językami
  • Narzędzia czytelności - analiza poziomu trudności
  • Tłumaczenia automatyczne - test jakości dla fragmentów obcojęzycznych

Checklist czytelności

✅ Sprawdź czy:

  • ☐ Strona ma określony główny język (lang="pl")
  • ☐ Fragmenty w innych językach są oznaczone
  • ☐ Skróty i akronimy są wyjaśnione
  • ☐ Trudne terminy mają definicje
  • ☐ Język jest dostosowany do odbiorców
  • ☐ Struktura tekstu jest logiczna
  • ☐ Zdania nie są nadmiernie długie
  • ☐ Słownictwo jest zrozumiałe
