3.3.5 Pomoc (AAA)

Dostępna jest pomoc kontekstowa.

Opis kryterium

To kryterium poziomu AAA ma na celu pomóc użytkownikom uniknąć błędów poprzez zapewnienie dostępu do pomocy kontekstowej. Osoby z niepełnosprawnościami mogą być bardziej skłonne do popełniania błędów niż użytkownicy bez niepełnosprawności. Dzięki pomocy kontekstowej użytkownicy dowiadują się, jak wykonać operację, nie tracąc z oczu tego, co robią.

Charakterystyka pomocy kontekstowej

  • Kontekstowa - związana z aktualnie wykonywaną funkcją
  • Dostępna na żądanie - użytkownik może ją uzyskać, kiedy jej potrzebuje
  • Oczywista w istnieniu - użytkownik musi wiedzieć, że pomoc jest dostępna
  • Nie przerwająca - nie zmusza do opuszczenia bieżącego kontekstu
  • Wsparcie dla etykiet - uzupełnia etykiety, gdy są niewystarczające

Kiedy wymagana jest pomoc kontekstowa

Pomoc kontekstowa jest wymagana tylko wtedy, gdy sama etykieta nie wystarcza do opisania wszystkich funkcji. Główne sytuacje to:

  • Formularze wymagające wprowadzania tekstu
  • Pola z oczekiwanym formatem danych
  • Złożone operacje lub procedury
  • Specjalistyczne terminy lub koncepcie
  • Nietypowe sposoby interakcji

Kto odnosi korzyść

  • Osoby z trudnościami w pisaniu - otrzymują wsparcie przy wprowadzaniu tekstu
  • Osoby z ograniczeniami intelektualnymi - lepiej rozumieją wymagania
  • Osoby z trudnościami w czytaniu - otrzymują dodatkowe wyjaśnienia
  • Seniorzy - mają dostęp do dodatkowych instrukcji
  • Osoby z ograniczeniami poznawczymi - otrzymują wsparcie w podejmowaniu decyzji
  • Wszyscy użytkownicy - mogą lepiej zrozumieć złożone funkcje

Definicje

Pomoc kontekstowa
Tekst pomocy, który dostarcza informacji związanych z funkcją aktualnie wykonywaną
Etykieta
Tekst lub inne komponenty prezentowane użytkownikowi w celu identyfikacji składnika treści internetowych
Funkcja
Działanie lub operacja, którą użytkownik może wykonać za pomocą elementu interfejsu
Kontekst
Otoczenie lub sytuacja, w której użytkownik wykonuje określoną czynność

<script>
function toggleHelp(helpId) {
    const helpElement = document.getElementById(helpId);
    const isHidden = helpElement.hidden;
    
    // Ukryj wszystkie inne pomocs
    document.querySelectorAll('.help-text').forEach(help => {
        help.hidden = true;
    });
    
    // Pokaż/ukryj aktualną pomoc
    helpElement.hidden = !isHidden;
    helpElement.focus();
}
</script>

<script>
function showInlineHelp(helpId) {
    const helpElement = document.getElementById(helpId);
    
    // Ukryj wszystkie inne pomoce inline
    document.querySelectorAll('.inline-help').forEach(help => {
        if (help.id !== helpId) {
            help.hidden = true;
        }
    });
    
    // Toggle aktualnej pomocy
    helpElement.hidden = !helpElement.hidden;
    
    if (!helpElement.hidden) {
        helpElement.scrollIntoView({ behavior: 'smooth' });
    }
}

function showGeneralHelp() {
    document.getElementById('general-help-modal').hidden = false;
}

function closeGeneralHelp() {
    document.getElementById('general-help-modal').hidden = true;
}
</script>

function updateContextualHelp(fieldName) {
    const assistantContent = document.getElementById('assistant-content');
    const fieldValue = document.getElementById(fieldName).value;
    
    let helpContent = '';
    
    if (fieldName === 'income-type' && fieldValue && contextualHelp[fieldName][fieldValue]) {
        const help = contextualHelp[fieldName][fieldValue];
        helpContent = `
            <div class="assistant-message">
                <h4>💡 ${help.title}</h4>
                ${help.content}
            </div>
        `;
    } else if (contextualHelp[fieldName]) {
        const help = contextualHelp[fieldName];
        helpContent = `
            <div class="assistant-message">
                <h4>💡 ${help.title}</h4>
                ${help.content}
            </div>
        `;
    }
    
    if (helpContent) {
        assistantContent.innerHTML = helpContent;
    }
}

function updateWordCount(textarea) {
    const text = textarea.value.trim();
    const wordCount = text ? text.split(/\s+/).length : 0;
    const counter = document.getElementById('word-count');
    const container = document.getElementById('description-counter');
    
    counter.textContent = wordCount;
    
    if (wordCount >= 100) {
        container.className = 'word-counter sufficient';
    } else if (wordCount >= 50) {
        container.className = 'word-counter partial';
    } else {
        container.className = 'word-counter insufficient';
    }

function checkSpelling(textarea) {
    const text = textarea.value.toLowerCase();
    const suggestionContainer = document.getElementById('spelling-suggestions');
    
    // Symulowane częste błędy
    const commonMistakes = {
        'javascript': 'JavaScript',
        'react': 'React',
        'node.js': 'Node.js',
        'html': 'HTML',
        'css': 'CSS',
        'responsywny': 'responsywny (sprawdź pisownię)',
        'technologi': 'technologie',
        'skillow': 'umiejętności'
    };
    
    const suggestions = [];
    
    for (const [mistake, correction] of Object.entries(commonMistakes)) {
        if (text.includes(mistake) && mistake !== correction.toLowerCase()) {
            suggestions.push(`"${mistake}" → "${correction}"`);
        }
    }
    
    if (suggestions.length > 0) {
        suggestionContainer.innerHTML = `
            <h4>💡 Sugestie poprawek:</h4>
            <ul>
                ${suggestions.map(suggestion => `<li>${suggestion}</li>`).join('')}
            </ul>
            <p><small>To są tylko sugestie - sprawdź dokładnie przed wprowadzeniem zmian.</small></p>
        `;
        suggestionContainer.hidden = false;
    } else {
        suggestionContainer.hidden = true;
    }
}

  • Nie przerywa procesu
  • Łatwa implementacja

Charakterystyka

Pomoc wyświetlana po kliknięciu ikony lub linku pomocy

Zastosowanie

  • Szczegółowe wyjaśnienia
  • Złożone instrukcje
  • Przykłady i wzorce

  • Nie zajmuje miejsca na ekranie
  • Pozwala na obszerną treść
  • Użytkownik kontroluje wyświetlanie

Pomoc dostosowująca się do działań użytkownika

Zastosowanie

  • Asystenci wirtualni
  • Kontekstowe rekomendacje

Zalety

  • Najwyższa użyteczność
  • Personalizowane doświadczenie
  • Proaktywne wsparcie

Charakterystyka

Instrukcje tekstowe z konkretnymi przykładami

class ContextualHelp {
    constructor() {
        this.activeHelp = null;
        this.helpData = new Map();
        this.init();
    }
    
    init() {
        // Inicjalizacja eventów
        document.addEventListener('click', this.handleHelpClick.bind(this));
        document.addEventListener('focus', this.handleFieldFocus.bind(this), true);
        document.addEventListener('keydown', this.handleKeyDown.bind(this));
    }
    
    // Rejestracja pomocy dla pola
    registerHelp(fieldId, helpConfig) {
        this.helpData.set(fieldId, {
            content: helpConfig.content,
            trigger: helpConfig.trigger || 'click',
            type: helpConfig.type || 'tooltip',
            position: helpConfig.position || 'below'
        });
    }
    
    // Pokazanie pomocy
    showHelp(fieldId, triggerElement) {
        const helpConfig = this.helpData.get(fieldId);
        if (!helpConfig) return;
        
        // Ukryj aktualną pomoc
        this.hideCurrentHelp();
        
        // Utwórz element pomocy
        const helpElement = this.createHelpElement(fieldId, helpConfig);
        
        // Umieść pomoc w odpowiednim miejscu
        this.positionHelp(helpElement, triggerElement, helpConfig.position);
        
        // Pokaż pomoc
        this.displayHelp(helpElement, helpConfig.type);
        
        this.activeHelp = helpElement;
    }
    
    createHelpElement(fieldId, config) {
        const helpElement = document.createElement('div');
        helpElement.className = `contextual-help contextual-help--${config.type}`;
        helpElement.id = `help-${fieldId}-${Date.now()}`;
        helpElement.setAttribute('role', 'tooltip');
        helpElement.setAttribute('aria-live', 'polite');
        
        // Dodaj przycisk zamknięcia
        helpElement.innerHTML = `
            <div class="help-content">
                <button type="button" class="help-close" aria-label="Zamknij pomoc">×</button>
                ${config.content}
            </div>
        `;
        
        // Event dla zamknięcia
        helpElement.querySelector('.help-close').addEventListener('click', () => {
            this.hideHelp(helpElement);
        });
        
        return helpElement;
    }
    
    positionHelp(helpElement, triggerElement, position) {
        document.body.appendChild(helpElement);
        
        const triggerRect = triggerElement.getBoundingClientRect();
        const helpRect = helpElement.getBoundingClientRect();
        const viewport = {
            width: window.innerWidth,
            height: window.innerHeight
        };
        
        let top, left;
        
        switch (position) {
            case 'above':
                top = triggerRect.top - helpRect.height - 10;
                left = triggerRect.left;
                break;
            case 'below':
                top = triggerRect.bottom + 10;
                left = triggerRect.left;
                break;
            case 'left':
                top = triggerRect.top;
                left = triggerRect.left - helpRect.width - 10;
                break;
            case 'right':
                top = triggerRect.top;
                left = triggerRect.right + 10;
                break;
            default:
                top = triggerRect.bottom + 10;
                left = triggerRect.left;
        }
        
        // Sprawdź czy pomoc mieści się w viewport
        if (left + helpRect.width > viewport.width) {
            left = viewport.width - helpRect.width - 20;
        }
        if (left < 20) {
            left = 20;
        }
        if (top + helpRect.height > viewport.height) {
            top = triggerRect.top - helpRect.height - 10;
        }
        if (top < 20) {
            top = 20;
        }
        
        helpElement.style.position = 'fixed';
        helpElement.style.top = `${top}px`;
        helpElement.style.left = `${left}px`;
        helpElement.style.zIndex = '1000';
    }
    
    displayHelp(helpElement, type) {
        switch (type) {
            case 'tooltip':
                helpElement.classList.add('help-tooltip');
                break;
            case 'modal':
                helpElement.classList.add('help-modal');
                this.createOverlay();
                break;
            case 'inline':
                helpElement.classList.add('help-inline');
                break;
        }
        
        helpElement.hidden = false;
        
        // Animacja wejścia
        requestAnimationFrame(() => {
            helpElement.classList.add('help-visible');
        });
        
        // Focus na pierwszym focusable element
        const firstFocusable = helpElement.querySelector('button, a, input, [tabindex]');
        if (firstFocusable) {
            firstFocusable.focus();
        }
    }
    
    hideCurrentHelp() {
        if (this.activeHelp) {
            this.hideHelp(this.activeHelp);
        }
    }
    
    hideHelp(helpElement) {
        if (!helpElement) return;
        
        helpElement.classList.remove('help-visible');
        
        setTimeout(() => {
            if (helpElement.parentNode) {
                helpElement.parentNode.removeChild(helpElement);
            }
            if (this.activeHelp === helpElement) {
                this.activeHelp = null;
            }
        }, 300);
        
        this.removeOverlay();
    }
    
    createOverlay() {
        if (document.querySelector('.help-overlay')) return;
        
        const overlay = document.createElement('div');
        overlay.className = 'help-overlay';
        overlay.addEventListener('click', () => this.hideCurrentHelp());
        document.body.appendChild(overlay);
    }
    
    removeOverlay() {
        const overlay = document.querySelector('.help-overlay');
        if (overlay) {
            overlay.parentNode.removeChild(overlay);
        }
    }
    
    handleHelpClick(event) {
        if (event.target.matches('.help-trigger, .help-trigger *')) {
            event.preventDefault();
            const trigger = event.target.closest('.help-trigger');
            const fieldId = trigger.getAttribute('aria-describedby')?.replace('-help', '');
            if (fieldId) {
                this.showHelp(fieldId, trigger);
            }
        }
    }
    
    handleFieldFocus(event) {
        if (event.target.matches('input, select, textarea')) {
            const fieldId = event.target.id;
            const helpConfig = this.helpData.get(fieldId);
            if (helpConfig && helpConfig.trigger === 'focus') {
                this.showHelp(fieldId, event.target);
            }
        }
    }
    
    handleKeyDown(event) {
        if (event.key === 'Escape' && this.activeHelp) {
            this.hideCurrentHelp();
        }
    }
}

Najlepsze praktyki

Projektowanie pomocy

  • Uczynić pomoc oczywistą - ikony, linki, przyciski
  • Umieszczać pomoc blisko pól, których dotyczy
  • Używać jasnego, zrozumiałego języka
  • Podawać konkretne przykłady i wzorce
  • Testować z prawdziwymi użytkownikami

Treść pomocy

  • Być zwięzłym ale kompletnym
  • Skupić się na działaniu, nie na technologii
  • Używać pozytywnego języka
  • Podawać alternatywne rozwiązania
  • Aktualizować regularnie

Implementacja techniczna

  • Zapewnić dostępność dla czytników ekranu
  • Implementować obsługę klawiatury
  • Optymalizować dla urządzeń mobilnych
  • Używać semantycznego HTML
  • Testować wydajność

Testowanie

Metody testowania

  • Test użyteczności - czy użytkownicy potrafią znaleźć i używać pomocy
  • Test dostępności - czy pomoc działa z technologiami asystującymi
  • Test z różnymi grupami - osoby z różnymi niepełnosprawnościami
  • Test mobilny - czy pomoc działa na urządzeniach dotykowych
  • Test wydajności - czy pomoc nie spowalnia strony

Lista kontrolna

  • Czy pomoc jest dostępna tam, gdzie etykiety są niewystarczające?
  • Czy istnienie pomocy jest oczywiste dla użytkownika?
  • Czy pomoc można uzyskać bez opuszczania kontekstu?
  • Czy pomoc jest dostępna za pomocą klawiatury?
  • Czy pomoc jest czytelna dla czytników ekranu?
  • Czy treść pomocy jest jasna i użyteczna?
  • Czy pomoc działa na urządzeniach mobilnych?

Narzędzia pomocne w testowaniu

  • NVDA/JAWS - testowanie dostępności pomocy
  • Keyboard navigation - test obsługi klawiatury
  • Mobile testing - test na różnych urządzeniach
  • User testing - obserwacja rzeczywistego użytkowania
  • Analytics - analiza korzystania z pomocy
