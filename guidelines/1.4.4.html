1.4.4 Zmiana rozmiaru tekstu

Poziom AA

Oficjalna dokumentacja W3C

Opis kryterium

Oprócz napisów rozszerzonych oraz tekstu w postaci grafiki, rozmiar tekstu może zostać powiększony do 200% bez użycia technologii wspomagających oraz bez utraty treści lub funkcjonalności.

Intencja

Celem tego kryterium jest zapewnienie, że tekst wyświetlany wizualnie można skalować z zachowaniem czytelności. Osoby z zaburzeniami widzenia często potrzebują powiększonego tekstu, aby go odczytać. Użytkownicy powinni mieć możliwość zwiększenia rozmiaru tekstu na stronie (np. za pomocą narzędzi przeglądarki) bez utraty funkcjonalności.

To kryterium jest szczególnie istotne dla osób ze słabym wzrokiem, które nie korzystają z lupy ekranowej czy innych technologii wspomagających, ale potrzebują większego tekstu dla czytelności.

Przykłady

Zły przykład - Układ, który łamie się przy powiększeniu

Strona z układem opartym na stałych wymiarach pikselowych, który łamie się przy powiększeniu tekstu:

<style>
.container {
    width: 800px;
    margin: 0 auto;
}

.sidebar {
    width: 200px;
    float: left;
    font-size: 14px;
}

.content {
    width: 600px;
    float: left;
    font-size: 16px;
}

.navigation {
    width: 100%;
    height: 50px;
    overflow: hidden; /* obcina zawartość przy powiększeniu */
}
</style>

<div class="container">
    <div class="navigation">
        <a href="#">Strona główna</a>
        <a href="#">O nas</a>
        <a href="#">Usługi</a>
        <a href="#">Kontakt</a>
    </div>
    
    <div class="sidebar">
        <h3>Menu boczne</h3>
        <ul>
            <li><a href="#">Link 1</a></li>
            <li><a href="#">Link 2</a></li>
            <li><a href="#">Link 3</a></li>
        </ul>
    </div>
    
    <div class="content">
        <h2>Nasza oferta</h2>
        <p>Treść strony...</p>
    </div>
</div>
Podgląd przy normalnym rozmiarze tekstu:
Strona główna O nas Usługi Kontakt

Menu boczne

  • Link 1
  • Link 2
  • Link 3

Nasza oferta

Treść strony...

Podgląd przy powiększeniu tekstu o 200%:
Strona główna O nas Usługi Kontakt

Menu boczne

  • Link 1
  • Link 2
  • Link 3

Nasza oferta

Treść strony...

Problem: Część tekstu jest obcięta, linki w nawigacji są niedostępne, a układ jest zaburzony przy powiększeniu o 200%.

Problem: Ten układ ma kilka problemów związanych z powiększaniem tekstu:

  • Elementy mają stałe szerokości w pikselach, co nie pozwala na elastyczne dostosowanie do większego tekstu
  • Używanie overflow: hidden powoduje, że tekst jest obcinany przy powiększeniu
  • Brak zawijania tekstu w nawigacji powoduje, że część linków staje się niedostępna
  • Użycie float do układu bez odpowiedniej elastyczności powoduje problemy przy powiększeniu

Dobry przykład - Elastyczny układ responsywny

Strona z elastycznym układem, który dostosowuje się do powiększonego tekstu:

<style>
.container {
    max-width: 1200px;
    margin: 0 auto;
    width: 100%;
}

.navigation {
    display: flex;
    flex-wrap: wrap;
    margin-bottom: 1rem;
}

.navigation a {
    margin-right: 1rem;
    margin-bottom: 0.5rem;
}

.main-content {
    display: flex;
    flex-wrap: wrap;
}

.sidebar {
    flex: 1;
    min-width: 200px;
    padding-right: 1rem;
}

.content {
    flex: 3;
    min-width: 300px;
}

@media (max-width: 768px) {
    .main-content {
        flex-direction: column;
    }
    
    .sidebar, .content {
        width: 100%;
    }
}
</style>

<div class="container">
    <nav class="navigation">
        <a href="#">Strona główna</a>
        <a href="#">O nas</a>
        <a href="#">Usługi</a>
        <a href="#">Kontakt</a>
    </nav>
    
    <div class="main-content">
        <aside class="sidebar">
            <h3>Menu boczne</h3>
            <ul>
                <li><a href="#">Link 1</a></li>
                <li><a href="#">Link 2</a></li>
                <li><a href="#">Link 3</a></li>
            </ul>
        </aside>
        
        <main class="content">
            <h2>Nasza oferta</h2>
            <p>Treść strony...</p>
        </main>
    </div>
</div>
Podgląd przy normalnym rozmiarze tekstu:
Strona główna O nas Usługi Kontakt

Menu boczne

  • Link 1
  • Link 2
  • Link 3

Nasza oferta

Treść strony...

Podgląd przy powiększeniu tekstu o 200%:
Strona główna O nas Usługi Kontakt

Menu boczne

  • Link 1
  • Link 2
  • Link 3

Nasza oferta

Treść strony...

Rozwiązanie: Układ elastycznie dostosowuje się do powiększonego tekstu. Nawigacja zawija się, a elementy zmieniają pozycję, ale cała treść pozostaje dostępna i czytelna.

Rozwiązanie: Ten układ używa kilku technik, które zapewniają dostępność przy powiększeniu tekstu:

  • Stosuje max-width i procentowe szerokości zamiast stałych wartości pikselowych
  • Używa Flexbox do elastycznego układu, który może się zawijać (flex-wrap: wrap)
  • Zawiera media queries, które dostosowują układ na mniejszych ekranach (co również pomaga przy powiększeniu tekstu)
  • Nie używa overflow: hidden do obcinania zawartości
  • Używa jednostek względnych (rem) do definiowania marginesów

Zły przykład - Tekst w obrazach

Strona, która używa obrazów zawierających tekst, których nie można powiększyć:

<!-- Tekst jako część obrazu, nie można go powiększyć -->
<div class="hero-banner">
    <img src="banner-with-text.jpg" alt="Witamy na naszej stronie - najlepsze rozwiązania dla Twojej firmy" width="1200" height="400">
</div>

<div class="content">
    <h2>Nasza oferta</h2>
    <p>Treść normalna, którą można powiększyć...</p>
</div>
Podgląd:
[Obraz z tekstem, którego nie można powiększyć]
Witamy na naszej stronie - najlepsze rozwiązania dla Twojej firmy

Nasza oferta

Treść normalna, którą można powiększyć...

Problem: Tekst w obrazie banera nie może być powiększony przez mechanizmy przeglądarki. Przy powiększeniu strony o 200%, tylko zwykły tekst się powiększy, a tekst w banerze pozostanie niezmieniony.

Problem: Używanie tekstu jako części obrazu uniemożliwia jego powiększenie za pomocą standardowych narzędzi przeglądarki. Ta praktyka nie tylko narusza kryterium 1.4.4, ale także utrudnia dostęp do treści osobom ze słabym wzrokiem.

Dobry przykład - Tekst zamiast obrazów z tekstem

Strona używająca rzeczywistego tekstu z tłem, który można powiększyć:

<style>
.hero-banner {
    background-image: url('banner-background.jpg');
    background-size: cover;
    background-position: center;
    padding: 3rem 2rem;
    margin-bottom: 2rem;
    text-align: center;
    color: white;
}

.hero-banner h1 {
    font-size: 2.5rem;
    margin-bottom: 1rem;
    text-shadow: 0 0 10px rgba(0, 0, 0, 0.7); /* dla lepszej czytelności na tle */
}

.hero-banner p {
    font-size: 1.25rem;
    max-width: 800px;
    margin: 0 auto;
    text-shadow: 0 0 5px rgba(0, 0, 0, 0.7);
}
</style>

<div class="hero-banner">
    <h1>Witamy na naszej stronie</h1>
    <p>Najlepsze rozwiązania dla Twojej firmy</p>
</div>

<div class="content">
    <h2>Nasza oferta</h2>
    <p>Treść normalna, którą można powiększyć...</p>
</div>
Podgląd przy normalnym rozmiarze tekstu:

Witamy na naszej stronie

Najlepsze rozwiązania dla Twojej firmy

Nasza oferta

Treść normalna, którą można powiększyć...

Podgląd przy powiększeniu tekstu o 200%:

Witamy na naszej stronie

Najlepsze rozwiązania dla Twojej firmy

Nasza oferta

Treść normalna, którą można powiększyć...

Rozwiązanie: Cały tekst na stronie, w tym tekst w banerze, można powiększyć o 200% i nadal pozostaje on w pełni czytelny.

Rozwiązanie: Zamiast umieszczać tekst jako część obrazu, lepiej jest używać rzeczywistego tekstu HTML nałożonego na tło:

  • Używa tekstu HTML z obrazem jako tłem, co pozwala na powiększanie tekstu
  • Stosuje jednostki względne (rem) dla rozmiaru czcionki, co ułatwia skalowanie
  • Dodaje cień tekstu dla lepszej czytelności na różnych tłach
  • Cały tekst na stronie skaluje się jednocześnie i proporcjonalnie

Wskazówki i techniki

  • Używaj jednostek względnych - Stosuj jednostki względne (em, rem, %) zamiast stałych jednostek (px) dla czcionek, marginesów, paddingów
  • Projektuj elastyczne układy - Używaj flexbox, grid lub układów responsywnych, które dostosowują się do różnych rozmiarów treści
  • Unikaj stałych wymiarów kontenerów - Używaj max-width zamiast width, aby umożliwić kontenerom rozszerzanie się
  • Testuj powiększenie - Regularnie testuj swoją stronę z tekstem powiększonym do 200%
  • Unikaj obcinania tekstu - Nie używaj overflow: hidden, chyba że masz dobry mechanizm przewijania
  • Używaj media queries - Media queries mogą pomóc w reorganizacji układu przy większym tekście
  • Zapewnij wystarczającą interlinię - Większy tekst potrzebuje większej interlinii dla zachowania czytelności
  • Unikaj tekstu w obrazach - Zamiast tego używaj rzeczywistego tekstu HTML
  • Sprawdzaj widok mobilny - Strony dobrze działające na urządzeniach mobilnych często dobrze radzą sobie również z powiększonym tekstem
  • Zapewniaj wystarczające kontrasty - Przy powiększeniu, dobry kontrast jest jeszcze bardziej istotny
