1.4.9 Obrazy tekstu (bez wyjątków)

Poziom AAA

Oficjalna dokumentacja W3C

Opis kryterium

Obrazy tekstu są używane tylko jako czysta dekoracja lub gdy określona prezentacja tekstu jest niezbędna dla przekazywanej informacji.

Uwaga: Logotypy (tekst, który jest częścią logo lub nazwy własnej marki) są uznawane za niezbędne.

Intencja

Celem tego kryterium jest wyeliminowanie obrazów tekstu w sytuacjach, gdy ta sama prezentacja wizualna może zostać osiągnięta za pomocą zwykłego tekstu HTML z CSS. W przeciwieństwie do kryterium 1.4.5 Obrazy tekstu (poziom AA), które dopuszcza obrazy tekstu, jeśli mogą być dostosowane wizualnie, to kryterium (poziom AAA) ogranicza użycie obrazów tekstu tylko do specyficznych sytuacji.

Prawdziwy tekst HTML (w przeciwieństwie do obrazów tekstu) zapewnia wiele korzyści:

  • Może być skalowany bez utraty jakości
  • Można dostosować jego wygląd do potrzeb użytkownika
  • Jest dostępny dla technologii wspomagających
  • Można go wyszukiwać i kopiować
  • Lepiej się renderuje na różnych urządzeniach i przy różnych ustawieniach

Przykłady

Zły przykład - Niepotrzebne użycie obrazu tekstu

Nagłówek zaprojektowany jako obraz, mimo że mógłby być zwykłym tekstem:

<!-- Nagłówek sekcji jako obraz tekstu -->
<div class="section-header">
    <img src="header-our-services.png" alt="Nasze usługi" width="300" height="60">
</div>

<p>Oferujemy szeroki zakres usług w dziedzinie marketingu cyfrowego, 
w tym optymalizację dla wyszukiwarek, marketing w mediach 
społecznościowych oraz kampanie reklamowe.</p>
Podgląd:
[Obraz: Nasze usługi]

Oferujemy szeroki zakres usług w dziedzinie marketingu cyfrowego, w tym optymalizację dla wyszukiwarek, marketing w mediach społecznościowych oraz kampanie reklamowe.

Problem: Nagłówek "Nasze usługi" jest obrazem, mimo że ten sam efekt wizualny można osiągnąć za pomocą zwykłego tekstu HTML z formatowaniem CSS.

Problemy w tym przykładzie:

  • Nagłówek jako obraz tekstu nie jest niezbędny do przekazania informacji
  • Ten sam efekt wizualny można łatwo osiągnąć za pomocą HTML i CSS
  • Obraz tekstu nie może być dostosowany do potrzeb użytkownika (np. zmiana rozmiaru, kontrastu)
  • Obraz może tracić na jakości przy powiększeniu strony
  • Tekst na obrazie nie może być wyszukiwany ani kopiowany

Dobry przykład - Użycie HTML i CSS zamiast obrazu tekstu

Ten sam nagłówek zaimplementowany jako tekst HTML z formatowaniem CSS:

<style>
.section-header {
    text-align: center;
    margin-bottom: 20px;
}

.section-title {
    display: inline-block;
    background-color: #3f51b5;
    padding: 15px 30px;
    color: white;
    font-size: 24px;
    font-weight: bold;
    border-radius: 4px;
    box-shadow: 0 3px 5px rgba(0,0,0,0.2);
    margin: 0;
}
</style>

<div class="section-header">
    <h2 class="section-title">Nasze usługi</h2>
</div>

<p>Oferujemy szeroki zakres usług w dziedzinie marketingu cyfrowego, 
w tym optymalizację dla wyszukiwarek, marketing w mediach 
społecznościowych oraz kampanie reklamowe.</p>
Podgląd:

Nasze usługi

Oferujemy szeroki zakres usług w dziedzinie marketingu cyfrowego, w tym optymalizację dla wyszukiwarek, marketing w mediach społecznościowych oraz kampanie reklamowe.

Rozwiązanie: Nagłówek jest teraz tekstem HTML, który można sformatować za pomocą CSS, aby wyglądał tak samo jak obraz, ale z zachowaniem wszystkich zalet prawdziwego tekstu.

Zalety tego rozwiązania:

  • Nagłówek jest teraz pełnoprawnym tekstem HTML, który zachowuje się jak każdy inny tekst na stronie
  • Użytkownik może skalować stronę bez utraty jakości nagłówka
  • Tekst może być dostosowany przez użytkownika (np. poprzez własne arkusze stylów)
  • Technologie wspomagające mogą go odczytać bez polegania na tekście alternatywnym
  • Wygląd można łatwo zmienić, modyfikując CSS, bez konieczności tworzenia nowego obrazu
  • Wizualna prezentacja jest praktycznie identyczna z wersją obrazkową

Dopuszczalny przykład - Logo jako obraz tekstu

Zgodnie ze standardem, logo firmy może być obrazem tekstu:

<header>
    <div class="logo">
        <img src="company-logo.png" alt="TechFirm - Innowacyjne rozwiązania technologiczne" width="200" height="60">
    </div>
    <nav>
        <ul>
            <li><a href="#">Strona główna</a></li>
            <li><a href="#">O nas</a></li>
            <li><a href="#">Usługi</a></li>
            <li><a href="#">Kontakt</a></li>
        </ul>
    </nav>
</header>
Podgląd:
[Logo: TechFirm]
  • Strona główna
  • O nas
  • Usługi
  • Kontakt

Dopuszczalne: Logo firmy jest uznawane za niezbędną prezentację tekstu, więc może być obrazem nawet według kryterium 1.4.9.

Wyjaśnienie:

  • Logo firmy zawiera tekst, który jest częścią identyfikacji wizualnej marki
  • Standard WCAG 2.1 wyraźnie wymienia logotypy jako wyjątek od zasady unikania obrazów tekstu
  • Specyficzna prezentacja wizualna logo (krój pisma, układ, efekty) jest uznawana za niezbędną dla przekazania informacji o marce
  • Ważne jest jednak, aby logo miało odpowiedni tekst alternatywny dla technologii wspomagających

Zły przykład - Tekstura na tekście jako obraz

Użycie obrazu tekstu tylko dla efektu tekstury:

<!-- Nagłówek z efektem tekstury jako obraz -->
<div class="textured-heading">
    <img src="textured-heading.png" alt="Nasze projekty" width="400" height="80">
</div>
Podgląd:
[Obraz: Nasze projekty]

Problem: Użyto obrazu tekstu tylko dla uzyskania efektu tekstury na tekście, mimo że podobny efekt można osiągnąć za pomocą CSS.

Problemy w tym przykładzie:

  • Obraz tekstu został użyty wyłącznie w celu dodania efektu tekstury do tekstu
  • Podobny efekt można uzyskać za pomocą nowszych technik CSS, takich jak background-clip: text
  • Taki obraz nie jest niezbędny do przekazania informacji - sama treść "Nasze projekty" mogłaby być przekazana jako zwykły tekst
  • Obraz tekstu ogranicza dostępność i możliwość dostosowania przez użytkownika

Dobry przykład - Tekstura na tekście za pomocą CSS

Osiągnięcie podobnego efektu za pomocą nowszych technik CSS:

<style>
.textured-heading {
    text-align: center;
    margin: 20px 0;
}

.textured-text {
    font-size: 36px;
    font-weight: bold;
    background: url('texture.jpg') center;
    background-clip: text;
    -webkit-background-clip: text;
    color: transparent;
    text-shadow: 2px 2px 3px rgba(0,0,0,0.3);
    padding: 10px;
}
</style>

<div class="textured-heading">
    <h2 class="textured-text">Nasze projekty</h2>
</div>
Podgląd:

Nasze projekty

Rozwiązanie: Uzyskano podobny efekt tekstury za pomocą właściwości CSS background-clip: text, zachowując zalety prawdziwego tekstu HTML.

Zalety tego rozwiązania:

  • Zastosowano właściwość CSS background-clip: text, aby nadać teksturę tekstowi
  • Tekst pozostaje prawdziwym tekstem HTML, a nie obrazem
  • Użytkownik może skalować stronę bez utraty jakości tekstu
  • Technologie wspomagające mogą odczytać tekst bezpośrednio
  • Wygląd można łatwo zmodyfikować, zmieniając CSS
  • Podobny efekt wizualny, ale z zachowaniem dostępności

Uwaga: Właściwość background-clip: text może wymagać prefiksu -webkit- dla maksymalnej zgodności z przeglądarkami.

Wskazówki i techniki

  • Używaj tekstu HTML z CSS - Zawsze, gdy to możliwe, stosuj rzeczywisty tekst HTML z formatowaniem CSS zamiast obrazów tekstu
  • Web Fonts - Wykorzystuj czcionki internetowe (Google Fonts, Adobe Fonts itp.) dla specjalnych krojów pisma zamiast obrazów
  • Nowoczesne techniki CSS - Stosuj zaawansowane techniki CSS (text-shadow, gradients, transforms, filters) do uzyskania efektów specjalnych na tekście
  • Obrazy tekstu tylko gdy konieczne - Używaj obrazów tekstu tylko dla logo lub gdy jest to absolutnie niezbędne
  • SVG dla tekstów - Rozważ użycie SVG dla tekstów, które wymagają specjalnego formatowania - SVG jest skalowalne i może być dostępne
  • Alternatywy dla obrazów tekstu - Jeśli musisz użyć obrazu tekstu, zawsze zapewnij odpowiedni tekst alternatywny
  • Testuj dostępność - Sprawdzaj, czy strona pozostaje dostępna dla technologii wspomagających
  • Dbaj o responsywność - Upewnij się, że rozwiązanie działa dobrze na różnych urządzeniach i przy różnych rozmiarach ekranu
