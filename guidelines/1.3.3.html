1.3.3 Właściwości zmysłowe

Poziom A

Oficjalna dokumentacja W3C

Opis kryterium

Instrukcje dotyczące zrozumienia i obsługi treści nie opierają się wyłącznie na właściwościach zmysłowych, takich jak kształt, rozmiar, wzrokowa lokalizacja, orientacja w przestrzeni czy dźwięk.

Intencja

Celem tego kryterium jest zapewnienie, że wszyscy użytkownicy, w tym osoby z niepełnosprawnościami wzroku lub słuchu, mogą zrozumieć i obsługiwać treści cyfrowe. Instrukcje nie powinny opierać się wyłącznie na cechach, które mogą być niedostępne dla niektórych użytkowników, jak kolor, kształt, położenie, dźwięk czy orientacja. Zamiast tego, należy dostarczyć instrukcje, które mogą być zrozumiane na wiele sposobów.

Przykłady

Zły przykład - instrukcje oparte na lokalizacji wizualnej

Instrukcje, które polegają wyłącznie na lokalizacji wizualnej:

<p>Aby kontynuować, kliknij niebieski przycisk w prawym górnym rogu ekranu.</p>

<p>Więcej informacji znajduje się w sekcji po lewej stronie.</p>

<p>Formularz kontaktowy dostępny jest na dole strony.</p>
Podgląd:

Aby kontynuować, kliknij niebieski przycisk w prawym górnym rogu ekranu.

Więcej informacji znajduje się w sekcji po lewej stronie.

Formularz kontaktowy dostępny jest na dole strony.

Te instrukcje są problematyczne, ponieważ polegają wyłącznie na lokalizacji wizualnej. Osoby niewidome lub słabowidzące, które korzystają z czytników ekranu, nie widzą, gdzie znajdują się elementy. Co więcej, układ strony może się zmieniać na różnych urządzeniach, co czyni takie instrukcje nieprecyzyjnymi.

Dobry przykład - instrukcje z identyfikatorami tekstowymi

Te same instrukcje, ale z dodanymi identyfikatorami tekstowymi i opisami:

<p>Aby kontynuować, kliknij przycisk <strong>"Następny krok"</strong> 
(niebieski przycisk w prawym górnym rogu ekranu).</p>

<p>Więcej informacji znajduje się w sekcji <strong>"Dodatkowe zasoby"</strong> 
(po lewej stronie).</p>

<p>Formularz kontaktowy dostępny jest w sekcji <strong>"Skontaktuj się z nami"</strong> 
(na dole strony).</p>
Podgląd:

Aby kontynuować, kliknij przycisk "Następny krok" (niebieski przycisk w prawym górnym rogu ekranu).

Więcej informacji znajduje się w sekcji "Dodatkowe zasoby" (po lewej stronie).

Formularz kontaktowy dostępny jest w sekcji "Skontaktuj się z nami" (na dole strony).

W tym przykładzie, choć nadal zawiera odniesienia do lokalizacji wizualnej, dodano również identyfikatory tekstowe, które pozwalają użytkownikom znaleźć odpowiednie elementy bez polegania wyłącznie na ich lokalizacji. Użytkownicy czytników ekranu mogą wyszukać elementy po ich nazwach.

Zły przykład - instrukcje oparte na kształcie i kolorze

Instrukcje opierające się wyłącznie na kształcie i kolorze:

<p>Kliknij czerwony kwadrat, aby zapisać swoje zmiany.</p>

<p>Pola oznaczone na zielono są obowiązkowe.</p>

<div class="icon-buttons">
    <div class="square red-bg"></div>
    <div class="circle blue-bg"></div>
    <div class="triangle green-bg"></div>
</div>

<style>
    .icon-buttons div {
        display: inline-block;
        width: 30px;
        height: 30px;
        margin-right: 10px;
    }
    .square { /* kształt kwadratu */ }
    .circle { border-radius: 50%; /* kształt koła */ }
    .triangle { /* kształt trójkąta */ }
    .red-bg { background-color: red; }
    .blue-bg { background-color: blue; }
    .green-bg { background-color: green; }
</style>
Podgląd:

Kliknij czerwony kwadrat, aby zapisać swoje zmiany.

Pola oznaczone na zielono są obowiązkowe.

Ten przykład jest problematyczny z kilku powodów:

  • Odnosi się do kształtu i koloru elementów, co jest niedostępne dla osób niewidomych i osób z daltonizmem
  • Ikony nie mają żadnych etykiet tekstowych
  • Nie ma alternatywnego sposobu identyfikacji elementów

Dobry przykład - ikony z tekstem i ARIA

Ikony z etykietami tekstowymi i odpowiednimi atrybutami ARIA:

<p>Kliknij przycisk <strong>"Zapisz"</strong> (czerwony kwadrat), aby zapisać swoje zmiany.</p>

<p>Pola oznaczone symbolem <strong>*</strong> i dodatkowym tekstem "Wymagane" są obowiązkowe.</p>

<div class="icon-buttons">
    <button class="btn square red-bg" aria-label="Zapisz">
        <span class="icon"></span>
        <span class="btn-text">Zapisz</span>
    </button>
    
    <button class="btn circle blue-bg" aria-label="Pomoc">
        <span class="icon"></span>
        <span class="btn-text">Pomoc</span>
    </button>
    
    <button class="btn triangle green-bg" aria-label="Nowy">
        <span class="icon"></span>
        <span class="btn-text">Nowy</span>
    </button>
</div>

<form>
    <div class="form-group">
        <label for="name">Imię i nazwisko <span class="required">*</span></label>
        <span class="sr-only">Wymagane</span>
        <input type="text" id="name" required aria-required="true">
    </div>
</form>
Podgląd:

Kliknij przycisk "Zapisz" (czerwony kwadrat), aby zapisać swoje zmiany.

Pola oznaczone symbolem * i dodatkowym tekstem "Wymagane" są obowiązkowe.

Wymagane

Ten przykład jest znacznie lepszy, ponieważ:

  • Przyciski mają etykiety tekstowe, które są widoczne dla wszystkich użytkowników
  • Dodano atrybuty ARIA dla lepszej dostępności
  • Obowiązkowe pola są oznaczone zarówno wizualnie (gwiazdką), jak i tekstowo ("Wymagane")
  • Instrukcje odnoszą się głównie do etykiet tekstowych, a odwołania do kształtów i kolorów są tylko dodatkową informacją

Zły przykład - instrukcje oparte na dźwięku

Instrukcje polegające wyłącznie na komunikatach dźwiękowych:

<p>Usłyszysz dźwięk dzwonka, gdy formularz zostanie pomyślnie wysłany.</p>

<p>Kiedy usłyszysz sygnał dźwiękowy, naciśnij przycisk "Dalej".</p>

<audio id="success-sound" src="success.mp3"></audio>
<audio id="alert-sound" src="alert.mp3"></audio>
Podgląd:

Usłyszysz dźwięk dzwonka, gdy formularz zostanie pomyślnie wysłany.

Kiedy usłyszysz sygnał dźwiękowy, naciśnij przycisk "Dalej".

Ten przykład jest niedostępny dla osób niesłyszących lub słabosłyszących, ponieważ opiera się wyłącznie na komunikatach dźwiękowych. Użytkownicy, którzy nie mogą usłyszeć dźwięków, nie będą wiedzieli, kiedy formularz został wysłany lub kiedy mają nacisnąć przycisk.

Dobry przykład - komunikaty dźwiękowe z alternatywami wizualnymi

Komunikaty dźwiękowe z alternatywnymi powiadomieniami wizualnymi:

<p>Po pomyślnym wysłaniu formularza zobaczysz komunikat potwierdzający 
i usłyszysz dźwięk dzwonka (jeśli masz włączony dźwięk).</p>

<p>Gdy pojawi się migający przycisk "Dalej" i usłyszysz sygnał dźwiękowy, 
możesz kontynuować.</p>

<div id="success-message" class="alert alert-success" role="alert" hidden>
    <i class="icon-check"></i> Formularz został pomyślnie wysłany!
</div>

<button id="next-button" disabled>Dalej</button>

<script>
    function onFormSuccess() {
        // Pokaż wizualny komunikat
        document.getElementById('success-message').hidden = false;
        
        // Odtwórz dźwięk (opcjonalnie)
        const sound = document.getElementById('success-sound');
        if (sound) sound.play();
        
        // Aktywuj i podświetl przycisk
        const nextButton = document.getElementById('next-button');
        nextButton.disabled = false;
        nextButton.classList.add('highlight-button');
    }
</script>

<style>
    .highlight-button {
        animation: pulse 2s infinite;
        box-shadow: 0 0 10px blue;
    }
    @keyframes pulse {
        0% { opacity: 1; }
        50% { opacity: 0.8; }
        100% { opacity: 1; }
    }
</style>

<audio id="success-sound" src="success.mp3"></audio>
Podgląd:

Po pomyślnym wysłaniu formularza zobaczysz komunikat potwierdzający i usłyszysz dźwięk dzwonka (jeśli masz włączony dźwięk).

Gdy pojawi się migający przycisk "Dalej" i usłyszysz sygnał dźwiękowy, możesz kontynuować.

✓ Formularz został pomyślnie wysłany!

Ten przykład jest znacznie lepszy, ponieważ:

  • Informuje użytkownika o alternatywnych powiadomieniach (wizualnych i dźwiękowych)
  • Pokazuje widoczny komunikat potwierdzający
  • Wykorzystuje animację wizualną dla przycisku "Dalej", co jest pomocne dla osób niesłyszących
  • Wyjaśnia, że dźwięk jest dodatkiem, a nie jedynym wskaźnikiem
  • Używa atrybutu role="alert", co oznacza, że komunikat może być odczytany przez czytniki ekranu

Wyzwania i wskazówki

  • Unikaj instrukcji opartych wyłącznie na:
    • Kolorze - "kliknij czerwony przycisk"
    • Kształcie - "wybierz opcję w okrągłym polu"
    • Lokalizacji - "menu na górze strony"
    • Orientacji - "poziomy pasek narzędzi"
    • Dźwięku - "poczekaj na sygnał dźwiękowy"
  • Zawsze dodawaj identyfikatory tekstowe:
    • Etykiety przycisków i formularzy
    • Nazwy sekcji i obszarów strony
    • Tekstowe opisy ikon i symboli
  • Łącz wskazówki zmysłowe z tekstowymi - "kliknij przycisk 'Zapisz' (niebieski przycisk z ikoną dyskietki)"
  • Używaj atrybutów ARIA dla lepszej dostępności komponentów interfejsu
  • Zapewniaj alternatywy dla wszystkich typów powiadomień - wizualne odpowiedniki dla dźwięków i odwrotnie
  • Testuj z wyłączonymi:
    • Kolorami (tryb monochromatyczny)
    • Dźwiękiem
    • Grafikami (tylko tekst)
